## 翻译规则

1. 对于每个 .md 文件，读取其内容，将正文部分从英文翻译为中文
2. YAML frontmatter（文件开头 `---` 包裹的部分）中的字段：
   - `description` 字段翻译为中文
   - `name` 字段保持英文原名不变
   - 其他字段（如 `phase`、`lesson` 等）保持原值不变
3. Markdown 标题（`#`、`##`、`###` 等）翻译为中文
4. Markdown 列表项内容翻译为中文
5. 代码块（用 ``` 包裹的内容）中的代码保持原样不翻译，但代码块内的英文注释可以翻译
6. 行内代码（用反引号 `` ` `` 包裹的内容，如命令、变量名、文件名）保持原样不翻译
7. 链接的 URL 保持原样，链接文字翻译为中文
8. 专业术语按业界通用译法翻译，首次出现时可保留英文原文加括号注释，如「反向传播（Backpropagation）」
9. 保持原文的所有格式标记（加粗、斜体、列表层级等）不变

## 输出规则

1. 在每个 .md 文件的同级目录（即同一个 outputs 文件夹内）生成翻译后的文件
2. 文件名格式：`{原文件名（不含 .md 后缀）}+zh.md`
   - 例如：`prompt-env-check.md` → `prompt-env-check+zh.md`
   - 例如：`skill-architecture-picker.md` → `skill-architecture-picker+zh.md`
3. 如果目标文件已存在，则覆盖它
