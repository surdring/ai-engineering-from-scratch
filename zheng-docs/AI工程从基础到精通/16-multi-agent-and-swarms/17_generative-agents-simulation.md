# 生成式代理与涌现模拟

> Park 等人 2023（UIST '23，arXiv:2304.03442）用三部分架构填充了 **Smallville**——一个 25 个代理的沙盒：**记忆流（Memory Stream）**（自然语言日志）、**反思（Reflection）**（代理关于自身流生成的高阶综合）和**计划（Plan）**（日级行为，然后是子计划）。里程碑式结果是情人节派对涌现：一个代理被植入「想举办 Valentine's Day 派对」，在没有进一步脚本的情况下，产生了在人群中传播的邀请、协调日期，派对发生了——来自 24 个开始时对此一无所知的代理。消融实验表明所有三个组件都是可信度（Believability）所必需的。记录在案的失败是空间规范错误（进入关闭的商店、共享单人卫生间）。这是 2026 年代理模拟和多 Agent 社会评估的参考架构。

## 问题

大多数多 Agent 系统是紧密脚本化的团队：规划者规划、程序员编码、审查者审查。这对定义明确的任务有效。它不能捕捉当代理有记忆、优先级和开放世界时产生的涌现、非脚本化行为。研究、社会模拟和越来越多的游戏 AI 需要这第二种。

Smallville 架构是其基准。在 Park 2023 之前，最好的代理模拟是肤浅的脚本追随者；之后，该模式成为开放世界中生成式代理的默认。如果你在 2026 年构建代理模拟，你要么使用 Smallville 的三个组件，要么显式地解释为什么不用。

## 概念

### 三个组件

**记忆流。** 一个仅追加的观察、行动、反思和计划日志。每个条目有时间戳、类型、描述（自然语言）和派生元数据：**新近度（Recency）**、**重要性（Importance）**（代理自我评分 1-10）和**相关性（Relevance）**（与当前查询的余弦相似度）。

```
[2026-02-14 09:12:03] observation: Isabella Rodriguez 问我是否喜欢爵士乐
[2026-02-14 09:14:22] reflection:   我享受关于音乐的长时间对话
[2026-02-14 10:05:00] plan:         参加 Isabella 今晚的情人节派对
```

记忆检索结合三个分数：`score = w_recency * e^(-decay * age) + w_importance * importance + w_relevance * cos_sim`。Top-k 条目进入当前提示。

**反思。** 周期性（每 N 条记忆或在重要事件时），代理从最近记忆生成高阶综合。反思条目回到流中，像任何其他记忆一样可检索。这是代理构建「理解」的方式——架构中等价于长期信念的东西。

**计划。** 自上而下的分解。首先是粗略的日级计划（「去上班，和 Klaus 吃晚饭」）。然后是小时级计划。然后是行动级计划。计划是可修订的：当观察与计划矛盾时，代理重新规划受影响的部分。

### 为什么三个都重要（消融）

Park 等人运行了分别去掉观察、反思和计划的消融实验。每个消融都损害可信度：

- 没有**观察**，代理错过上下文并根据过时信念行动。
- 没有**反思**，代理无法形成高阶信念；交互保持肤浅。
- 没有**计划**，行为变成反应性噪音；目标消散。

来自人类评分者的可信度分数在全部三个都有的情况下最高；去掉任何一个都会产生可测量的退化。

### 情人节涌现

一个代理，Isabella Rodriguez，被植入目标「想在 2 月 14 日下午 5 点在 Hobbs Cafe 举办情人节派对」。其他 24 个代理没有收到这样的种子。在模拟的日子里：

1. Isabella 的计划包括邀请人们。
2. 每个邀请成为邻居记忆流中的一个观察。
3. 那个邻居的反思产生信念：「Isabella 在举办派对。」
4. 邻居的计划纳入「2 月 14 日参加派对。」
5. 邻居告诉其他邻居。邀请在没有中央协调的情况下传播。
6. 2 月 14 日下午 5 点，多个代理聚集在 Hobbs Cafe。

这是技术意义上的涌现：系统级行为（派对）从局部交互（双边邀请 + 个人规划）中产生，没有中央编排者。

### 记录在案的失败模式

Park 等人明确记录了：

- **空间规范错误。** 代理走进关闭的商店。代理试图使用同一个单人卫生间。代理在不打算吃饭的房间吃饭。模型不会仅从环境推断社会-物理规范。
- **记忆溢出。** 深度模拟运行导致记忆检索成本增长。实际补救措施：周期性记忆压缩（摘要并修剪）和低重要性条目的衰减。
- **反思幻觉。** 反思可以发明记忆流中不存在的关系。缓解措施：在反思提示中包含来源记忆 ID，并在检索时验证。

这些是生产相关的失败模式：任何 2026 年代理模拟都会继承它们。

### 三组件实现规则

1. **记忆是仅追加的。** 永远不要修改记忆条目。更正是新条目。
2. **重要性分数廉价。** 在写入时调用 LLM 评分重要性 1-10。缓存分数。
3. **检索是排名的，而非过滤的。** 按组合分数的 Top-k；不要使用硬过滤器（会丢失上下文）。
4. **反思周期性运行。** 当未处理记忆的重要性之和超过阈值（例如 150）时触发。
5. **计划是可修订的。** 当新观察与计划矛盾时，只重新生成受影响的部分，而非整个计划。

### 超越 Smallville 的生成式代理

2024-2026 年的后续文献扩展了该架构：

- **用于政策 / 市场研究的多 Agent 社会模拟。** 类似 Smallville 的人群模拟用户对功能的反应。比 A/B 测试更快；准确性有争议。
- **游戏的 NPC AI。** 带有 Smallville 代理的 RPG 产生涌现的故事情节，而非脚本化的任务。
- **生成式代理评估基准。** 度量标准变成可信度 + 长时间运行的行为连贯性，而非任务准确性。

该架构是参考。扩展替换了组件（用于记忆的向量存储、检索增强反思、神经符号计划），但保持三部分结构。

### 为什么这对多 Agent 工程很重要

Smallville 是概念验证，证明当组件正确时多 Agent 涌现是廉价的。该架构现在已在开源模型上复现（较小的 LLM 优雅地失去可信度，而非急剧下降）。任何需要**涌现社会行为**的生产系统都使用这种形状。任何需要**紧密任务执行**的系统使用本阶段前面的监督者 / 角色 / 原语模式。

## 构建

`code/main.py` 在标准库 Python 中用脚本化代理策略（无真实 LLM）实现了三个组件。演示在微型中复现情人节派对涌现：

- `MemoryStream`——带新近度/重要性/相关性检索的仅追加日志。
- `reflect(stream)`——对最近高重要性记忆的脚本化反思。
- `plan(agent_state)`——基于当前信念的日级和小时级计划。
- 场景：5 个代理。Agent 1 从「下午 5 点举办派对」开始。在模拟的时间步中，邀请传播，代理聚集。

运行：

```
python3 code/main.py
```

预期输出：逐时间步追踪。到最后时间步，5 个代理中至少有 3 个将派对显示在其计划中，并且它们聚集在派对地点。单个种子在没有编排者的情况下产生了协调到达。

## 交付

生产模拟规则：

- **记忆是数据库。** 在规模上选择真正的存储（向量数据库、Postgres）。内存中标准库用于原型。
- **记录检索追踪。** 对于每个行动，记录驱动它的 top-k 记忆。这是你的调试能力。
- **每个代理的 token 预算。** 每个代理每时间步的 retrieve + reflect + plan 是 O(k) 次 LLM 调用。N 个代理 × T 时间步 × 每时间步调用数可能使预算相形见绌。
- **周期性压缩记忆。** 摘要并修剪低重要性条目。保留策略是设计决策，而非细节。
- **显式检测空间 / 社会规范违规。** 架构不学习它们。

## 练习

1. 运行 `code/main.py`。确认 3+ 个代理聚集在派对上。将代理增加到 10——涌现仍然发生吗？
2. 移除反思步骤。行为看起来像什么？映射到 Park 2023 中的消融发现。
3. 引入一个竞争的种子目标（「Klaus 想在下午 5 点做一个研究演讲」）。代理是分裂，还是一个目标占主导？什么决定它？
4. 添加空间约束：Hobbs Cafe 最多容纳 4 个代理。模拟是优雅处理溢出，还是命中「单人卫生间」失败模式？
5. 阅读 Park 等人（arXiv:2304.03442）第 6 节（涌现行为实验）。识别一项在微型中不可复现的行为。你需要增强架构的哪个组件？

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|------|---------|---------|
| Memory stream | 「代理的日记」 | 观察、行动、反思、计划的仅追加日志。 |
| Recency | 「记忆有多新」 | 按年龄的指数衰减分数。 |
| Importance | 「代理有多在乎」 | 写入时自我评分 1-10。缓存。 |
| Relevance | 「与当前查询的相关程度」 | 余弦相似度（基于嵌入）。 |
| Reflection | 「高阶信念」 | 从最近记忆生成的综合，作为新记忆重新摄入。 |
| Plan | 「日/小时/行动分解」 | 自上而下的计划树。当观察矛盾时可修订。 |
| Smallville | 「Park 2023 的沙盒」 | 产生情人节涌现的 25 代理模拟。 |
| Believability | 「质量度量」 | 人类评分者对行为是否像可信代理的评分。 |
