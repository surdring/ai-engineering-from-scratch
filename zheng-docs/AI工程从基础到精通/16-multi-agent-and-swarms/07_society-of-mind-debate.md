# 心智社会与多 Agent 辩论

> Minsky 1986 年的前提——智能是一个专家的社会——每十年被重新发现一次。2023 年 Du 等人将其转化为一个具体算法：多个 LLM 实例提出答案，阅读彼此的答案，进行批判，并更新。经过 N 轮后，它们收敛到一个共识，在六个推理和事实性任务上超越了零样本 CoT 和反思。两个发现很重要：**多个代理**和**多轮**都有独立贡献。社会胜过单代理独白；多轮交换胜过一次性投票。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置知识：** 第 16 阶段 · 04（原语模型）
**时间：** 约 60 分钟

## 问题

自一致性（Self-Consistency）——对一个模型多次采样并取多数答案——是你能附加的最便宜的推理改进。它有效，但很快饱和。你可以将样本加倍，却看不到另一个有意义的跳跃。

辩论打破了饱和。不是来自一个模型的 N 个独立样本，而是 N 个代理阅读彼此的推理并修订。样本之间的相关性下降（它们不再是独立同分布的），收敛点通常在独立同分布投票自信出错的地方是正确的。

## 概念

### Du 等人 2023 年算法

来自 arXiv:2305.14325（ICML 2024）：

1. 每个 N 个代理对问题产生一个初始答案。
2. 对于第 r = 2..R 轮：每个代理被展示其他代理的第 r-1 轮答案，并被问及「考虑这些，给出你更新后的答案。」
3. 经过 R 轮后，对最终答案进行多数投票。

论文在 MMLU、GSM8K、传记、MATH 和事实性基准上进行了测试。辩论始终击败 CoT 和自我反思。

### 两个独立的旋钮

同一论文的消融实验：

- **仅代理数量**（1 轮，N 个答案的多数投票）在大多数任务上胜过单代理，但趋于平台。
- **仅轮数**（1 个代理看到自己先前的推理）几乎没帮助——反思的已知弱点。
- **两者结合**产生大幅跳跃。多个代理之间的多轮交换驱动收益。

### 为什么有效

两个机制：

1. **暴露于分歧。** 当一个代理看到另一个代理的推理链并得出不同结论时，它必须要么辩护要么更新。无论哪种情况，第 r+1 轮的上下文都比第 r 轮更丰富。
2. **减少相关错误。** 在自一致性中，所有样本来自同一模型，因此错误相关——你平均进入一个自信错误的答案。不同模型或不同种子去相关化。不同的*辩论观点*进一步去相关化。

### 异质辩论

A-HMAD 和相关跟进使用*不同的基础模型*给不同的代理。Llama + Claude + GPT 辩论减少了单一文化崩溃（Monoculture Collapse，第 26 课），因为一个模型家族的相关错误不被其他家族共享。

缺点：一个弱模型参与辩论可能将共识拖向其错误答案（见「Should we be going MAD?」，arXiv:2311.17371）。

### NLSOM——129 Agent 扩展

Zhuge 等人（「Mindstorms in Natural Language-Based Societies of Mind，」arXiv:2305.17066）将这一想法扩展到 129 个成员的社会。结果：专业化和自组织随着规模涌现，系统在视觉问答等任务上优于单代理。

### 失败模式

- **谄媚级联（Sycophancy Cascade）。** 所有代理屈从于听起来最自信的那个。辩论崩溃为最响亮的声音。提示给出对抗性角色（「一个代理必须争论反方立场」）有帮助。
- **主题漂移（Topic Drift）。** 多轮辩论偏离原始问题。缓解：每轮重新注入问题。
- **计算爆炸。** N 个代理 × R 轮 = N·R 次 LLM 调用，每次调用具有不断增长的上下文。5 代理、5 轮辩论是 25 次调用，上下文不断增长。每个问题的成本可能超过单次 CoT 调用的 10 倍。

## 构建

`code/main.py` 对一个数学问题运行 3 代理 × 3 轮辩论，每个代理从不同（可能错误的）答案开始。代理是脚本化的——每个「更新」通过对邻居答案求加权平均，权重由脚本化的置信度决定。收敛在逐轮日志中可见。

演示展示了两个关键效果：

- 单轮交换使代理更接近正确答案。
- 第 2 轮之后的额外轮次显示递减回报（匹配 Du 等人的平台）。

运行：

```
python3 code/main.py
```

## 实践

`outputs/skill-debate-configurator.md` 为新任务配置辩论：代理数量、轮数、异质性（同一模型 vs 混合）、角色分配（对称 vs 一个对抗性角色）。它还在运行之前估算 token 成本。

## 交付

如果你交付辩论：

- **轮数上限为 3。** Du 等人显示 3 轮捕获了大部分收益。更多是成本而非质量。
- **代理数量上限为 5。** 超过 5，上下文膨胀和成本占主导。
- **默认异质。** 池中至少两个不同的基础模型。
- **对抗性插槽。** 一个代理被提示无论怎样都要反对。打破谄媚。
- **记录每一轮。** 隐藏中间轮的辩论系统无法调试或审计。

## 练习

1. 运行 `code/main.py`，然后将轮数设为 5，观察递减回报。额外收敛在哪一轮停止？
2. 添加一个带有对抗性角色的第四代理：始终反对当前多数意见。这打破还是改进了收敛？
3. 绘制（打印）每轮的同意分数（多数答案上的代理比例）。它何时达到 1.0，这等同于「正确」吗？
4. 阅读 Du 等人第 4 节消融实验。使用此代码复现「仅代理」vs「仅轮次」vs「两者」的结果。
5. 阅读「Should we be going MAD?」（arXiv:2311.17371）并列出除循环赛之外的两个辩论变体——例如，法官主导、辩论链、对抗性。

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|------|---------|---------|
| Society of Mind | 「Minsky 的想法」 | 智能作为交互的专业化代理；1986 年框架现在通过 LLM 辩论操作化。 |
| Multi-agent debate | 「代理争论」 | N 个代理提出、相互批判、在 R 轮中修订、多数投票。 |
| Consensus | 「他们同意了」 | 不是认识论的真相——只是多数答案上的比例。可能是自信错误的。 |
| Rounds | 「交换步骤」 | 一轮 = 每个代理读取其他代理并更新一次。 |
| Heterogeneous debate | 「混合模型家族」 | 使用不同基础模型来去相关化错误。 |
| Sycophancy cascade | 「每个人都同意最响亮的声音」 | 辩论失败，代理屈从于最自信的代理，无论正确性如何。 |
| NLSOM | 「129 Agent 社会」 | 自然语言心智社会；Zhuge 等人的扩展版。 |
| Correlated error | 「相同模型，相同 Bug」 | 为什么自一致性饱和；跨不同观点的辩论去相关化。 |
