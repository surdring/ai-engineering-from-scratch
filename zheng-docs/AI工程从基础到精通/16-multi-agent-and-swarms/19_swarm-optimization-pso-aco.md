# 面向 LLM 的集群优化（PSO、ACO）

> 仿生优化正在 LLM 中回归。**LMPSO**（arXiv:2504.09247）使用 PSO，其中每个粒子的速度是提示，LLM 生成下一个候选；在结构化序列输出（数学表达式、程序）上效果很好。**Model Swarms**（arXiv:2410.11163）将每个 LLM 专家视为模型权重流形上的 PSO 粒子，并报告在 9 个数据集上仅 200 个实例就比 12 个基线平均提升 **13.3%**。**SwarmPrompt**（ICAART 2025）混合 PSO + 灰狼（Grey Wolf）用于提示优化。**AMRO-S**（arXiv:2603.12933）是 ACO 启发的信息素专员（Pheromone Specialist），用于多 Agent LLM 路由——**4.7 倍加速**、可解释路由证据、质量门控异步更新，将推理与学习解耦。本课在提示参数空间上实现 PSO，在代理路由上实现 ACO，测量为什么这些经典算法适合 LLM 时代，以及什么时候不适合。

## 问题

你有一个在任务评估上得分为 62% 的提示。你想改进它。朴素方法是无梯度的手动调整，扩展性差。强化学习需要奖励信号和足够的 rollouts 来训练。通过提示进行反向传播真的不可能——提示是离散字符串，不是可微分参数。

经典仿生优化——用于连续搜索空间的 PSO、用于路径选择的 ACO——正是为这种机制设计的：无梯度、基于种群、每次评估廉价。将它们与 LLM 配对用于无梯度搜索步骤，你得到一个令人惊讶的实用优化器。

相同的模式适用于多 Agent 系统中的代理*路由*。ACO 风格的信息素轨迹记录哪个代理在哪种任务类型上效果最好，让路由器利用该轨迹，信息素衰减使路由可以被重新发现。

## 概念

### PSO 复习（Kennedy & Eberhart 1995）

粒子群优化（Particle Swarm Optimization）：连续搜索空间中的粒子种群。每个粒子有位置 `x_i` 和速度 `v_i`。每次迭代：

```
v_i <- w * v_i + c1 * r1 * (p_best_i - x_i) + c2 * r2 * (g_best - x_i)
x_i <- x_i + v_i
评估适应度(x_i)
如果有改进则更新 p_best_i
如果是全局最优则更新 g_best
```

其中 `p_best` 是粒子自身最优，`g_best` 是集群最优，`w, c1, c2` 是惯性 + 认知 + 社会权重，`r1, r2` 是随机因子。

### LLM 输出上的 PSO——LMPSO

arXiv:2504.09247 为 LLM 生成的结构化输出（数学表达式、程序）调整了 PSO。每个粒子是候选输出。速度是一个*提示*，描述如何将当前输出向个人/全局最优修改。LLM 从速度提示生成新输出。速度的「惯性」是像「做小的增量改变」这样的提示。

这在以下情况下效果很好：
- 输出是结构化的（可解析、可评估）。
- 适应度是自动的（测试运行、算术评估）。
- 种群小（~10-30 个粒子），因此总 LLM 调用保持可控。

当适应度需要人工审查时效果不好——每次迭代的成本变得禁止。

### Model Swarms

arXiv:2410.11163 将 PSO 从输出层带到*模型*层。每个「粒子」是一个专家 LLM（参数）。集群通过无梯度更新将参数移向集体最优。报告：在 9 个数据集上仅每次迭代 200 个实例的情况下，比 12 个基线平均提升 13.3%。

关键洞察是 LLM 专家模型已经在共享参数流形（Adapter 权重、LoRA 增量）中接近。在这个低维子空间上的 PSO 既廉价又有效。

### ACO 复习（Dorigo 1992）

蚁群优化（Ant Colony Optimization）：蚂蚁在图上穿越；每条路径有信息素轨迹。蚂蚁移动概率按信息素强度加权。完成任务的蚂蚁按解决方案质量沉积信息素。信息素随时间衰减。

### AMRO-S——用于代理路由的 ACO

arXiv:2603.12933 使用 ACO 进行多 Agent 路由。每种任务类型是「目的地」；每个代理是可能的路线。信息素加强产生良好输出的路线。关键贡献：

- **可解释路由证据。** 信息素强度是人类可读的信号。
- **质量门控异步更新。** 信息素仅在质量检查通过后更新，将推理与学习解耦。
- 在多 Agent 路由基准上 **4.7 倍加速**。

质量门很重要：没有它，快但错误的代理积累信息素，系统锁定在坏路线上。

### 什么时候对 LLM 使用 PSO / ACO

**使用 PSO 当：**
- 搜索空间是连续的或映射到连续参数（提示嵌入、LoRA 权重、数值生成参数）。
- 适应度廉价且自动。
- 种群可以小（10-30）。

**使用 ACO 当：**
- 你有路由或路径选择问题。
- 决策随时间加强（相同任务类型再次出现）。
- 你需要路由决策的可解释证据。

**不使用两者当：**
- 适应度需要人工审查（每次迭代太昂贵）。
- 搜索空间是 PSO 不覆盖的离散和组合形式（改用遗传算法）。
- 实时决策需要严格延迟（PSO/ACO 相对于单次启发式收敛慢）。

### 为什么仿生优化仍然获胜

基于梯度的方法需要可微分信号。LLM 输出和路由决策不是简单可微分的。伪梯度方法（强化学习路由、DPO 风格提示调优）有效但需要昂贵训练。

PSO 和 ACO 只需要一个*评估器*函数。如果你可以评分候选输出或路由决策，你就可以在空间上优化。这使得适用门槛低得多。

### 实际限制

- **种群预算。** N 个粒子 × T 次迭代 × 每次评估成本。对于每次调用约 0.02 美元的 LLM 评估，20 粒子的 PSO 运行 50 次迭代花费约 20 美元。相应计划。
- **探索 vs 利用。** 信息素衰减率和 PSO 惯性权衡；衰减太快 → 忘记解决方案；太慢 → 卡在早期局部最优。
- **灾难性漂移。** 如果适应度景观发生变化（新数据分布），两种算法都可以收敛然后发散。监控最优适应度稳定性。

## 构建

`code/main.py` 实现：

- `LMPSO`——对数值提示参数（temperature、top_k 权重）的 PSO。每个粒子的「LLM 生成」被模拟为脚本化的适应度函数。运行算法 30 次迭代并显示 g_best 收敛。
- `AMRO_S`——ACO 风格路由。3 个代理，4 种任务类型，信息素矩阵，100 个路由任务。打印随时间（task_type → agent 选择）分布以显示轨迹形成。
- 比较：相同任务流上的随机路由 vs ACO 路由。测量质量和延迟。

运行：

```
python3 code/main.py
```

预期输出：
- LMPSO：g_best 适应度在 30 次迭代内从随机改善到接近最优。
- AMRO-S：信息素表在每个任务类型上稳定在正确的代理；ACO 路由在质量上超过随机约 30-40%，同时也减少延迟（更少重试）。

## 交付

- **从小开始。** 10-20 粒子，20-50 迭代。仅当收敛曲线显示明确增益时才扩展。
- **记录每次迭代的信息素或 g_best。** 没有轨迹调试集群优化器是痛苦的。
- **质量门控更新。** 尤其对 ACO 路由：快但错误的代理不能积累信息素。
- **在分布变化时重置衰减。** 当你的评估分布改变时，老化的信息素是过时的；重置或暂时加倍衰减率。
- **限制每次迭代成本。** 发出每次迭代成本指标。每次迭代花费 500 美元且获得 0.5% 增益的 PSO 不可交付。

5. 阅读 AMRO-S（arXiv:2603.12933）。实现解耦的「推理快速通道」和异步信息素更新。这如何改变持续负载下的系统延迟？

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|------|---------|---------|
| PSO | 「粒子群优化」 | Kennedy-Eberhart 1995。基于种群的无梯度优化器。 |
| ACO | 「蚁群优化」 | Dorigo 1992。通过信息素轨迹进行路径/路线优化。 |
| LMPSO | 「带 LLM 生成的 PSO」 | arXiv:2504.09247。速度是提示；LLM 产生候选。 |
| Model Swarms | 「专家权重上的 PSO」 | arXiv:2410.11163。模型参数子空间上的无梯度更新。 |
| AMRO-S | 「用于 Agent 路由的 ACO」 | arXiv:2603.12933。任务类型 × Agent 上的信息素矩阵。 |
| p_best / g_best | 「个人 / 全局最优」 | 迄今为止找到的每个粒子和整个集群的最优解决方案。 |
| Pheromone | 「路由记忆」 | 边上的强度；随时间衰减；根据质量沉积。 |
| Quality-gated update | 「只从好运行中学习」 | 信息素沉积以质量检查为条件。 |
| Catastrophic drift | 「分布变化」 | 适应度景观变化；旧的 p_best 和信息素变得陈旧。 |
