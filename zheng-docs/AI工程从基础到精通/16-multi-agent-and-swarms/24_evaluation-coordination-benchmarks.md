# 评估与协调基准

> 五个 2025-2026 年的基准覆盖了多代理评估空间。**MultiAgentBench / MARBLE**（ACL 2025，arXiv:2503.01935）用里程碑 KPI 评估星形/链形/树形/图形拓扑；**图形拓扑在研究方面最优**，认知规划增加约 3% 的里程碑达成率。**COMMA** 评估多模态非对称信息协调；包括 GPT-4o 在内的最先进模型难以超越随机基线。**MedAgentBoard**（arXiv:2505.12371）覆盖四个医疗任务类别，经常发现多代理并不优于单 LLM。**AgentArch**（arXiv:2509.10769）对结合工具使用 + 记忆 + 编排的企业代理架构进行基准测试。**SWE-bench Pro**（[arXiv:2509.16941](https://arxiv.org/abs/2509.16941)）有 1865 个问题，跨 41 个仓库，涵盖商业应用、B2B 服务和开发者工具；前沿模型在 Pro 上得分约 23%，而在 Verified 上得分 70%+ —— 这是对污染（Contamination）的现实检验。Claude Opus 4.7（2026 年 4 月）据报道在 Pro 上达到 **64.3%**，采用显式代理团队协调（尚无 Anthropic 官方来源发布——视为初步数据）；Verdent（代理脚手架）在 Verified 上达到 **76.1% pass@1**（[Verdent 技术报告](https://www.verdent.ai/blog/swe-bench-verified-technical-report)）。**AAAI 2026 Bridge Program WMAC**（https://multiagents.org/2026/）是 2026 年社区焦点。本课基于 MARBLE 的指标，运行拓扑 vs 指标扫描，并确立"仅通过 SWE-bench Verified 不能作为泛化证据"的规则。

**类型：** 学习
**语言：** Python（标准库）
**前置条件：** Phase 16 · 15（投票与辩论拓扑）、Phase 16 · 23（失败模式）
**时间：** ~75 分钟

## 问题

当一篇论文声称"我们的多代理系统更好"时，问题是：比什么好、在什么上好、如何衡量？2023-2024 年的多代理评估时代是混乱的——每个人都选择自己的指标、自己的基线和自己的任务集。2025-2026 年的基准施加了结构。

没有共享基准，你无法有意义地比较两个多代理系统。更糟的是，没有留出基准（Hold-out Benchmarks），前沿模型可能会被污染。到 2025 年中，SWE-bench Verified 部分进入了训练语料库；前沿分数膨胀；Pro 被设计为无污染的现实检验。

本课列举了 2026 年的五个规范基准，说明每个基准衡量什么，并教你以怀疑态度阅读基准声明。

## 概念

### MultiAgentBench (MARBLE) — ACL 2025

arXiv:2503.01935。在科研、编码和规划任务上评估四种协调拓扑（星形、链形、树形、图形）。基于里程碑的 KPI 追踪部分进展，而非仅最终成功。

测量结果：

- **图形**拓扑在科研场景中最优；支持任意对等批判。
- **链形**在逐步细化编码中最优。
- **星形**在快速事实整合中最优。
- **协调税（Coordination Tax）**在图形拓扑上约 4 个代理后出现。
- **认知规划**跨拓扑增加约 3% 的里程碑达成率。

何时使用：当你想要同类比较协调拓扑时。MARBLE 仓库（https://github.com/ulab-uiuc/MARBLE）提供评估器。

### COMMA — 多模态非对称信息

覆盖代理具有不同观察模态且必须在不完全信息共享下协调的任务。报告结果令人不安：包括 GPT-4o 在内的前沿模型在 COMMA 的代理间协作上难以超越**随机基线**。信号是多代理模态训练不足且评估不足——LLM 在单模态合作上表现尚可；多模态协调则崩溃。

何时使用：当你的系统具有多模态或非对称信息协调时。COMMA 的空结果为在声称之前先测量提供了警告。

### MedAgentBoard — 领域压力测试

arXiv:2505.12371。四个医疗任务类别：诊断、治疗方案制定、报告生成、患者沟通。比较多代理 vs 单 LLM vs 传统基于规则的系统。

发现：多代理在大多数类别上并不优于单 LLM。多代理优势狭窄——当子任务明显可分离时任务分解有帮助（诊断 + 治疗）；当协调开销超过专业化收益时则有害（报告生成）。

何时使用：当你的领域有明确的单 LLM 基线时。如果 MedAgentBoard 的经验具有普遍性，许多被提出的多代理系统是过度工程化的。

### AgentArch — 企业架构

arXiv:2509.10769。企业设置，工具使用、记忆和编排叠加在一起。该基准隔离每一层的贡献：添加工具有多大帮助？添加记忆？添加多代理编排？

何时使用：当你正在设计企业代理栈并需要为每一层提供理由时。AgentArch 有助于避免购买你无法衡量其价值的功能。

### SWE-bench Pro — 现实检验

arXiv:2509.16941。1865 个问题，跨 41 个仓库，涵盖商业应用、B2B 服务和开发者工具。设计为在后续训练截止日期下**无污染**。前沿模型在 Pro 上得分约 23%，而在 Verified 上得分 70%+。差距是污染信号。

2026 年 4 月分数：
- Claude Opus 4.7 在 Pro 上：**64.3%**（据报道采用显式代理团队协调；尚无 Anthropic 官方来源发布——视为初步数据）。
- Verdent（代理脚手架）在 Verified 上：**76.1% pass@1**（[技术报告](https://www.verdent.ai/blog/swe-bench-verified-technical-report)）。
- 前沿模型在没有代理脚手架时的 Pro 原始分数：~23-35%（[SWE-bench Pro 论文](https://arxiv.org/abs/2509.16941)）。

启示："我们击败了 SWE-bench Verified"不再是能力的证据。Pro 是当前的门控测试。代理团队脚手架在 Pro 上产生可测量的收益（~30-40 分的增量），这是 2026 年多代理协调最有力的实证论据之一。

### AAAI 2026 WMAC

AAAI 2026 Bridge Program —— 多代理协调研讨会（Workshop on Multi-Agent Coordination）（https://multiagents.org/2026/）。2026 年多代理 AI 研究的社区焦点。接收的论文和研讨会论文集是评估新方法的规范场所；对于生产决策，优先采用 WMAC 接收的声明而非 arXiv 预印本。

### 以怀疑态度阅读基准声明——2026 年检查清单

当有人声称一个多代理结果时：

1. **哪个基准、哪个分割？** SWE-bench Verified vs Pro 差异巨大。在错误的分割上报告的数字毫无价值。
2. **污染检查。** 基准是否在模型训练截止日期之后发布？如果不是，谨慎对待。
3. **基线比较。** 对比单 LLM 基线、对比随机、对比先前的多代理工作。不是"对比同一系统的未调优版本"。
4. **统计显著性。** N 次试验、p 值、置信区间。前沿模型方差高；单次运行具有误导性。
5. **任务多样性。** 一个任务还是多个？泛化性对生产重要。
6. **成本披露。** 每任务的 Token 数、墙钟时间。一个 90% 的解决方案但 20 倍成本是商业决策，而非能力声明。

### 基准未能很好衡量的方面

- **长周期协调。** 数天的墙钟交互。所有当前基准都是短期的。
- **对抗性韧性。** 当一个代理是恶意的或被攻陷时会发生什么？
- **部署下的漂移。** 基准是静态的；生产分布会变化。
- **成本归一化性能。** 大多数基准报告原始准确率，而非每美元准确率。

构建你自己关心的轴上的内部基准通常是正确的做法。

## 构建它

`code/main.py` 是一个非交互式走查：

- 在一个玩具任务上模拟 3 个多代理系统。
- 为每个系统计算 MARBLE 风格的里程碑指标。
- 通过从"训练"集中留出任务来运行污染检查。
- 显式地与随机基线比较。
- 打印基准声明记分卡。

运行：

```bash
python3 code/main.py
```

预期输出：系统记分卡，包含原始准确率、里程碑达成率、每任务成本、与随机基线的增量，以及污染检查说明。

## 使用它

`outputs/skill-benchmark-reader.md` 阅读任何多代理基准声明并应用审查清单。输出：评级和注意事项。

## 交付它

生产评估纪律：

- **构建反映你实际生产分布的内部基准。** 公开基准提供信息但不能替代。
- **在每次比较中包含随机基线。** 如果你在协调任务上不能大幅超越随机，该任务可能定义不当。
- **在准确率旁边报告成本。** Token 成本和墙钟时间。运维团队两者都需要。
- **每季度重建基准。** 生产分布会变化；过时的基准具有误导性。
- **避免公开基准过拟合。** 如果你的团队专门为 SWE-bench Pro 数字优化，你将在生产上退化。

## 练习

1. 运行 `code/main.py`。确定三个模拟系统中哪个具有最佳的每里程碑成本。它是否与原始准确率最高的系统一致？
2. 阅读 MultiAgentBench（arXiv:2503.01935）。为你自己的任务领域，确定 MARBLE 会推荐四种拓扑中的哪一种。根据论文结果进行论证。
3. 阅读 SWE-bench Pro 论文。具体是什么使它具有抗污染性？同样的技术是否可以应用于你关心的其他基准？
4. 阅读 COMMA 关于多模态协调的发现。设计一个你可以添加到内部基准的简单多模态协调任务。什么算作有用的信号？
5. 将基准声明检查清单应用于一篇近期多代理论文的标题结果。你会给该声明什么评级？
