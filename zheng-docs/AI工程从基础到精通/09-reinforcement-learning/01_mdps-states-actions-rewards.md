# MDP、状态、动作与奖励

> 马尔可夫决策过程是五件事：状态、动作、转移、奖励、折扣。RL 中的一切——Q-learning、PPO、DPO、GRPO——都在这个形状上优化。学一次，免费阅读强化学习的其余部分。

**类型：** 学习
**语言：** Python
**前置要求：** 第 1 阶段 · 06（概率与分布），第 2 阶段 · 01（机器学习分类法）
**时间：** 约 45 分钟

## 问题

你在写一个国际象棋机器人。或者一个库存规划器。或者一个交易代理。或者训练推理模型的 PPO 循环。四个不同领域，一个惊人的事实：所有四个都坍缩为相同的数学对象。

监督学习给你 `(x, y)` 对并要求你拟合一个函数。强化学习不给你标签——只有状态流、你采取的动作和一个标量奖励。这步棋赢了吗？补货决定省钱了吗？交易盈利了吗？LLM 刚产生的标记是否导致裁判给出更高奖励？

在形式化之前你无法从这个流中学习。"我看到了什么"、"我做了什么"、"接下来发生了什么"、"这有多好"——每个都必须成为你可以推理的对象。这种形式化就是马尔可夫决策过程。本阶段的每个 RL 算法，包括最后的 RLHF 和 GRPO 循环，都在这个形状上优化。

## 概念

![马尔可夫决策过程：状态、动作、转移、奖励、折扣](../assets/mdp.svg)

**五个对象。**

- **状态** `S`。代理决策所需的一切。在 GridWorld 中，是格子。在国际象棋中，是棋盘。在 LLM 中，是上下文窗口加任何记忆。
- **动作** `A`。选择。上/下/左/右移动。走一步棋。发出一个标记。
- **转移** `P(s' | s, a)`。给定状态 `s` 和动作 `a`，下一状态的分布。国际象棋中确定性，库存中随机，LLM 解码中几乎确定性。
- **奖励** `R(s, a, s')`。标量信号。赢 = +1，输 = -1。收入减成本。GRPO 中的对数似然比项。
- **折扣** `γ ∈ [0, 1)`。未来奖励相对于现在的权重。`γ = 0.99` 约 100 步视野；`γ = 0.9` 约 10 步。

**马尔可夫性质** `P(s_{t+1} | s_t, a_t) = P(s_{t+1} | s_0, a_0, …, s_t, a_t)`。未来仅依赖于当前状态。如果不成立，状态表示不完整——不是方法的失败，是状态的失败。

**策略与回报。** 策略 `π(a | s)` 将状态映射为动作分布。回报 `G_t = r_t + γ r_{t+1} + γ² r_{t+2} + …` 是未来奖励的折扣和。值 `V^π(s) = E[G_t | s_t = s]` 是在策略 `π` 下从 `s` 开始的期望回报。Q 值 `Q^π(s, a) = E[G_t | s_t = s, a_t = a]` 是以特定动作开始的期望回报。每个 RL 算法估计这两者之一，然后相应改进 `π`。

**Bellman 方程。** 本阶段一切使用的定点方程：

`V^π(s) = Σ_a π(a|s) Σ_{s', r} P(s', r | s, a) [r + γ V^π(s')]`
`Q^π(s, a) = Σ_{s', r} P(s', r | s, a) [r + γ Σ_{a'} π(a'|s') Q^π(s', a')]`

这些将期望回报分解为"这一步骤的奖励"加"你到达之处的折扣值"。递归的。第 9 阶段的每个算法要么将这个方程迭代到收敛（动态规划），要么从中采样（蒙特卡洛），要么自举一步（时序差分）。

## 构建

### 步骤 1：一个微型确定性 MDP

4×4 GridWorld。代理从左上角开始，终点在右下角，每步奖励 -1，动作 `{上, 下, 左, 右}`。见 `code/main.py`。

```python
GRID = 4
TERMINAL = (3, 3)
ACTIONS = {"up": (-1, 0), "down": (1, 0), "left": (0, -1), "right": (0, 1)}

def step(state, action):
    if state == TERMINAL:
        return state, 0.0, True
    dr, dc = ACTIONS[action]
    r, c = state
    nr = min(max(r + dr, 0), GRID - 1)
    nc = min(max(c + dc, 0), GRID - 1)
    return (nr, nc), -1.0, (nr, nc) == TERMINAL
```

五行。这就是整个环境。确定性转移，恒定步惩罚，吸收终止状态。

### 步骤 2：展开策略

策略是从状态到动作分布的函数。最简单：均匀随机。

```python
def uniform_policy(state):
    return {a: 0.25 for a in ACTIONS}

def rollout(policy, max_steps=200):
    s, total, steps = (0, 0), 0.0, 0
    for _ in range(max_steps):
        a = sample(policy(s))
        s, r, done = step(s, a)
        total += r
        steps += 1
        if done:
            break
    return total, steps
```

运行随机策略 1000 次。这个 4×4 棋盘的平均回报约为 -60 到 -80。最优回报是 -6（右下直线路径）。缩小这个差距是第 9 阶段的一切。

### 步骤 3：通过 Bellman 方程精确计算 `V^π`

对于小型 MDP，Bellman 方程是线性系统。枚举状态，应用期望，迭代直到值停止变化。

```python
def policy_evaluation(policy, gamma=0.99, tol=1e-6):
    V = {s: 0.0 for s in all_states()}
    while True:
        delta = 0.0
        for s in all_states():
            if s == TERMINAL:
                continue
            v = 0.0
            for a, pi_a in policy(s).items():
                s_next, r, _ = step(s, a)
                v += pi_a * (r + gamma * V[s_next])
            delta = max(delta, abs(v - V[s]))
            V[s] = v
        if delta < tol:
            return V
```

这是迭代策略评估。它是 Sutton & Barto 中的第一个算法，也是后续每个 RL 方法的理论基础。

### 步骤 4：`γ` 是有物理意义的超参数

有效视野约为 `1 / (1 - γ)`。`γ = 0.9` → 10 步。`γ = 0.99` → 100 步。`γ = 0.999` → 1000 步。

太低则代理行为短视。太高则信用分配变嘈杂，因为许多早期步骤共同负责远期奖励。LLM RLHF 通常使用 `γ = 1`，因为 episode 短且有界。控制任务使用 `0.95–0.99`。长视野策略游戏使用 `0.999`。

## 陷阱

- **非马尔可夫状态。** 如果你需要最近三个观察来决定，"状态"不仅仅是当前观察。修复：堆叠帧（Atari 上的 DQN 堆叠 4 帧）或使用循环状态（LSTM/GRU 处理观察序列）。
- **稀疏奖励。** 仅有胜负奖励在大状态空间中几乎不可能学习。塑造奖励（中间信号）或通过模仿引导（第 9 阶段 · 09）。
- **奖励攻击。** 优化代理奖励经常产生病态行为。OpenAI 的划船代理永远绕圈收集能量道具而不是完成比赛。始终从目标结果定义奖励，而非代理。
- **折扣误设。** 无限视野任务上的 `γ = 1` 使每个值无限。始终用有限视野或 `γ < 1` 限制。
- **奖励尺度。** `{+100, -100}` vs `{+1, -1}` 的奖励给出相同的最优策略但梯度幅度差异巨大。在插入 PPO/DQN 之前归一化到约 `[-1, 1]`。

## 使用

2026 年的技术栈在触及代码之前将每个 RL 流水线化简为 MDP：

| 情境 | 状态 | 动作 | 奖励 | γ |
|-----------|-------|--------|--------|---|
| 控制（运动、操作） | 关节角度 + 速度 | 连续力矩 | 任务特定塑造 | 0.99 |
| 游戏（国际象棋、围棋、扑克） | 棋盘 + 历史 | 合法走法 | 赢=+1/输=-1 | 1.0（有限） |
| 库存 / 定价 | 库存 + 需求 | 订货数量 | 收入 - 成本 | 0.95 |
| LLM 的 RLHF | 上下文标记 | 下一标记 | 结束时奖励模型分数 | 1.0（episode 约 200 个标记） |
| GRPO 用于推理 | 提示 + 部分响应 | 下一标记 | 结束时验证器 0/1 | 1.0 |

在写任何训练循环之前写出五个元组。大多数"RL 不工作"的错误报告追溯到一个在纸上就被破坏的 MDP 公式。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------------|-----------------------|
| MDP | "马尔可夫决策过程" | `(S, A, P, R, γ)` 元组，形式化任何顺序决策问题。 |
| 回报 | "G_t" | 从时间 t 开始的折扣未来奖励和。 |
| 值函数 | "V(s)" | 从状态 s 出发的期望回报。 |
| Q 函数 | "动作值" | 在 s 中以 a 开始的期望回报。 |
| 策略 | "π(a|s)" | 代理选择动作的分布。 |
| Bellman 方程 | "定点方程" | V 和 Q 的递归一致性条件。 |
| 折扣因子 | "γ" | 未来奖励 vs 现在的乘数。 |
| 马尔可夫性质 | "无记忆" | 未来仅依赖于当前状态。 |
