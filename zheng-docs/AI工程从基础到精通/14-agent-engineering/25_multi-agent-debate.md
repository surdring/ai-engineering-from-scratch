# 多代理辩论与协作

> Du 等人（ICML 2024，"Society of Minds"）运行 N 个模型实例独立提出答案，然后在 R 轮内迭代互相批判以达成收敛。提高了事实性、规则遵循和推理能力。在 Token 成本上，稀疏拓扑优于全连接网格。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置条件：** Phase 14 · 12（工作流模式），Phase 14 · 05（自我修正与 CRITIC）
**时间：** ~60 分钟

## 学习目标

- 解释辩论协议（Debate Protocol）：N 个提案者，R 轮，收敛到共享答案。
- 描述为什么辩论能提高事实性、规则遵循和推理能力。
- 解释稀疏拓扑（Sparse Topology）：不是每个辩论者都需要看到所有其他人。
- 在脚本化 LLM 上实现标准库辩论，包含全连接网格和稀疏变体；测量 Token 成本 vs 准确率。

## 问题

自我修正（Self-Refine，第 05 课）是一个模型自我批判——存在群体思维（Groupthink）风险。CRITIC（第 05 课）通过外部工具为批判提供事实依据——不是总能使用。辩论引入了第三种模式：多个实例，交叉批判，通过分歧达成收敛。

## 概念

### Society of Minds（Du 等人，ICML 2024）

- N 个模型实例独立对同一问题提出答案。
- 在 R 轮内，每个模型阅读他人的提案并加以批判。
- 模型根据批判更新自己的答案。
- 在 R 轮后，返回收敛答案。

原始实验因成本考虑使用 N=3，R=2。在困难问题（MMLU、GSM8K、国际象棋走法有效性、传记生成）上，准确性随更多代理和更多轮次提高。

跨模型组合优于单模型辩论：ChatGPT + Bard 一起 > 任何单独一个。

### 稀疏拓扑

"Improving Multi-Agent Debate with Sparse Communication Topology"（arXiv:2406.11776，2024-2025）表明全连接网格辩论并不总是最优。稀疏拓扑（星型、环形、轮辐型 Hub-and-Spoke）可以在更低的 Token 成本下匹配准确率。每个辩论者只看到一部分同伴。

影响：

- 全连接网格 N=5，R=3 = 5 × 3 = 15 个提案，每个阅读 4 个同伴 = 60 次批判操作。
- 星型 N=5，R=3（一个中心节点 + 4 个辐条）= 15 个提案，辐条只读中心节点 = 12 次批判操作。

### 辩论何时有帮助

- **事实性。** N 个独立提案，交叉检查减少幻觉。
- **规则遵循。** 国际象棋走法有效性——一个模型漏掉一条规则，其他模型发现。
- **开放式推理。** 多种框架方式缩小到正确答案。

### 辩论何时有害

- **延迟敏感 UX。** N × R 轮串行是你可能负担不起的延迟。
- **成本敏感的规模。** 每个问题 N × R 个 Token。
- **简单事实查询。** 一次查找比五次辩论更便宜。

### 2026 年实际实例化

- **Anthropic 编排器-工作器**（第 12 课）— 带综合步骤的辩论变体。
- **LangGraph 监督器**（第 13 课）— 中心路由 + 专家代理可将辩论实现为一个节点。
- **OpenAI Agents SDK**（第 16 课）— 代理之间来回交接进行迭代批判。
- **多代理评估** — 配对辩论 + 评估器-优化器以获取评估信号。

### 这种模式的陷阱

- **收敛崩溃。** 所有代理收敛到第一个错误答案。通过强制分歧轮次来缓解。
- **中心节点故障。** 在星型拓扑中，一个坏的中心节点会破坏所有人。轮换或使用多个中心节点。
- **提示同质化。** 所有代理使用相同提示；产生相同答案。使用多样化的提示和/或模型。

## 构建

`code/main.py` 实现标准库辩论：

- `Debater` 类（脚本化 LLM，每个辩论者独立意见漂移）。
- `FullMeshDebate` 和 `SparseDebate` 运行器。
- 三个问题：一个事实问题、一个规则问题、一个推理问题。
- 指标：收敛答案、到收敛的轮次、总批判操作数。

运行方式：

```
python3 code/main.py
```

输出：每种协议的准确率和成本；稀疏在 2/3 的问题上以更低成本匹配全连接网格。

## 使用场景

- **Anthropic 编排器-工作器** — 简单的 2-3 个工作器辩论。
- **LangGraph** — 带检查点的有状态多轮辩论。
- **自定义** — 用于研究或专门的正确性保证。

## 部署

`outputs/skill-debate.md` 搭建一个多代理辩论脚手架，包含可配置的拓扑结构、N、R 和收敛规则。

## 练习

1. 实现"强制分歧"规则：在第 1 轮中，每个辩论者必须给出不同提案。测量对收敛速度的影响。
2. 添加置信度加权聚合：辩论者返回（答案，置信度）；聚合器按置信度加权。有帮助吗？
3. 将一个"代理"替换为具有不同意见的不同脚本化 LLM。异构性能提高准确率吗？
4. 测量全连接网格 vs 稀疏在你的 3 个问题上的 Token 成本。绘制成本 vs 准确率图。
5. 阅读 Society of Minds 论文。将你的玩具移植到 N=5，R=3。什么会出问题？什么会更好？

## 关键术语

| 术语 | 人们常说的 | 实际含义 |
|------|-----------|---------|
| 辩论（Debate） | "多代理批判" | N 个提案者，R 轮交叉批判，收敛 |
| 全连接网格（Full Mesh） | "每个人都读所有人" | 每个辩论者每轮读取所有同伴 |
| 稀疏拓扑（Sparse Topology） | "有限同伴视野" | 辩论者只读取一部分同伴 |
| 轮辐型（Hub-and-Spoke） | "星型拓扑" | 一个中心辩论者，N-1 个辐条只读中心节点 |
| 收敛（Convergence） | "一致" | 辩论者收敛到共享答案 |
| Society of Minds | "Du 等人辩论论文" | ICML 2024 多代理辩论方法 |
