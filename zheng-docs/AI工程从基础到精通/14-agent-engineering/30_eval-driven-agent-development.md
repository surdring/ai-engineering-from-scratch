# 评估驱动的代理开发

> Anthropic 的指导："从简单提示开始，通过全面评估进行优化，仅在有需要时才添加多步骤代理系统。"评估不是最后一步。它是驱动 Phase 14 中每一个其他选择的外部循环。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置条件：** Phase 14 全部。
**时间：** ~60 分钟

## 学习目标

- 列举三个评估层——静态基准测试、自定义离线评估、在线生产评估——以及每个层的用途。
- 解释评估器-优化器（Evaluator-Optimizer）紧密循环。
- 描述 2026 年最佳实践：评估与代码同存，在 CI 中运行，门控 PR 合并。
- 将每个 Phase 14 的课程连接到它产生的评估用例。

## 问题

代理在演示中通过。它们在 demo 无法预测的方式下在生产中失败。基准测试回答"这个模型广泛上是否有能力？"而不是"这个代理是否为我产品交付正确的补丁？"答案：在三层评估中，持续运行，每个护栏和学到的规则都映射到一个评估用例。

## 概念

### 三个评估层

1. **静态基准测试** — SWE-bench Verified 用于代码（第 19 课），WebArena/OSWorld 用于浏览/桌面（第 20 课），GAIA 用于通用（第 19 课），BFCL V4 用于工具使用（第 06 课）。用于跨模型比较和回归门控。污染是真实的：SWE-bench+ 发现 32.67% 解决方案泄漏。始终报告 Verified / +-审计分数。

2. **自定义离线评估** — 你产品的形态：
   - LLM-as-Judge（Langfuse、Phoenix、Opik — 第 24 课）。
   - 基于执行（运行补丁，检查测试）。
   - 基于轨迹（将操作序列与黄金轨迹比较；OSWorld-Human 显示顶级代理比黄金轨迹多 1.4-2.7 倍步骤）。

3. **在线评估** — 生产：
   - 会话回放（Langfuse）。
   - 护栏触发告警（第 16 课、第 21 课）。
   - 每步成本/延迟追踪（第 23 课 OTel Span）。

### 评估器-优化器（Anthropic）

紧密循环：

1. 提案者生成输出。
2. 评估器评判。
3. 精炼直到评估器通过。

这是自我修正（Self-Refine，第 05 课）的泛化。任何你关心的代理流程都可以包装在评估器-优化器中以提高可靠性。

### 2026 年最佳实践

- 评估与代码同存。
- 在每个 PR 上通过 CI 运行。
- 通过评估分数门控合并（例如"与 main 相比无 >5% 回归"）。
- 每个护栏映射到一个评估用例。
- 每个学到的规则（Reflexion、pro-workflow learn-rule）映射到一个失败用例。

### 评估驱动开发的失败之处

- **没有基线。** 没有已知良好状态的评估不可读。存储基线。
- **LLM-as-Judge 没有事实依据。** 评判者也会幻觉。CRITIC 模式（第 05 课）— 评判者基于外部工具。
- **过度拟合评估。** 优化评估会偏离生产实用性。轮换用例。
- **不稳定的评估。** 非确定性用例导致误报。固定随机种子，快照状态。

## 构建

`code/main.py` 是一个标准库评估工具链：

- 用例注册表，按类别（基准测试、自定义、在线）分类。
- 一个被测脚本化代理。
- 评估器-优化器循环：提案、评判、精炼直到通过或达到最大轮次。
- CI 门控：聚合通过率 + 对基线的回归。

运行方式：

```
python3 code/main.py
```

输出：每个用例的通过/失败、回归标志、CI 门控裁决。

## 使用场景

- 在与代理代码相同的仓库中编写评估用例。
- 在每次 PR 上通过 CI 运行它们。
- 出现回归时使构建失败。
- 随时间追踪通过率。
- 将每个生产失败关联到新用例。

## 部署

`outputs/skill-eval-suite.md` 为代理产品构建三层评估套件，包含 CI 门控和回归追踪。

## 练习

1. 取你的一次生产失败。编写一个复现它的评估用例。你的代理现在能通过吗？
2. 为你领域构建 LLM-as-Judge 评分标准，包含三个维度（事实、语气、范围）。评分 50 个会话。
3. 将评估套件接入 CI。在 >=5% 回归时使构建失败。
4. 添加轨迹效率指标：代理比黄金轨迹多用了多少步骤？
5. 将每个 Phase 14 课程映射到你的套件中的一个评估用例。有缺失的吗？那就是需要填补的空白。

## 关键术语

| 术语 | 人们常说的 | 实际含义 |
|------|-----------|---------|
| 静态基准测试 | "现成评估" | SWE-bench、GAIA、AgentBench、WebArena、OSWorld |
| 自定义离线评估 | "领域评估" | 基于你产品形态的 LLM-as-Judge / 执行 / 轨迹 |
| 在线评估 | "生产评估" | 会话回放、护栏告警、成本/延迟追踪 |
| 评估器-优化器 | "提案-评判-精炼" | 迭代直到评判者通过 |
| CI 门控 | "合并阻断器" | 在评估回归时使构建失败 |
| 基线 | "已知良好状态" | 用于检测回归的参考分数 |
| 轨迹效率 | "步骤数/黄金轨迹" | 代理步骤数除以人类专家最小值 |
