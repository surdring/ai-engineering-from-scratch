# 多 Token 预测（MTP）

> 从 GPT-2 到 Llama 3，每个自回归 LLM 在训练时每个位置使用一个损失：预测下一个 token。DeepSeek-V3 在每个位置添加了第二个损失：预测再下一个 token。在 671B 模型上的额外 14B 参数通过梯度流蒸馏回主模型，并且训练好的 MTP 头在推理时被重新用作接受率 80%+ 的推测解码草稿器。1.8× 的生成吞吐量免费获得。本课从 DeepSeek 技术报告构建顺序 MTP 模块，计算损失和共享头参数布局，并解释为什么 MTP 保持因果链而 Gloeckle 等人的原始并行 MTP 破坏了它。

- 陈述 MTP 训练目标并推导各预测深度的联合损失。

## 问题

下一个 token 预测是标准的 LLM 训练目标。每个隐藏状态被监督来预测恰好一件事：紧随其后的 token。这是一个令人惊讶的弱信号。序列中的大部分信息延伸超过一个 token——结构、连贯性、事实性、算术流。模型必须通过在数万亿 token 上累积许多单 token 信号来学习这些。

MTP 提出的问题是：如果每个隐藏状态被监督来一次预测多个未来 token 会怎样？Gloeckle 等人（Meta，2024）展示了这有帮助。他们的实现在骨干网络之上放置了若干个独立输出头，每个预测不同的偏移。并行，简单，但这些头看到相同的隐藏状态而没有任何层次化精炼——而且预测不以因果方式链式连接，因此它们不能用于推测解码。

DeepSeek-V3（2024 年 12 月）将 MTP 重新设计为在每个预测深度保持因果链的顺序模块。模型从 `h_i^(0)` 预测 `t+1`，然后从结合了 `h_i^(0)` 和 `E(t+1)` 嵌入的新隐藏状态 `h_i^(1)` 预测 `t+2`，以此类推。每个深度是其自己的小型 transformer 块。共享嵌入和共享输出头保持参数开销适度。在 DeepSeek-V3 的规模下，MTP 模块在 671B 主模型权重之上有 14B 额外参数。那 2% 的开销换来了更密集的训练信号和一个在推理时随时可用的推测解码草稿。

本课从头构建单个 MTP 模块和 D 深度损失。数学简洁。实现约 150 行。

## 核心概念

### 顺序 MTP 配方

DeepSeek-V3 在主模型之上添加 `D` 个 MTP 模块。每个模块 `k`（`k = 1..D`）预测深度 `k` 处的 token——即给定通过位置 `i` 的前缀，预测 `t_{i+k}`。

模块 `k` 包括：

- 具有自己的注意力和 MLP 的 transformer 块 `T_k`。
- 一个投影矩阵 `M_k`，将前一深度的隐藏状态与下一深度真实 token 的嵌入组合。
- 共享嵌入 `E`（与主模型相同）。
- 共享输出头 `Out`（与主模型相同）。

训练时，对于通过位置 `i` 的前缀，逐深度隐藏状态为：

```
h_i^(0) = 位置 i 处的主模型骨干
h_i^(k) = T_k( M_k * concat(RMSNorm(h_i^(k-1)), RMSNorm(E(t_{i+k}))) )   当 k >= 1 时
```

逐深度预测为：

```
logits_{i+k} = Out(h_i^(k-1))   当 k = 1..D 时
```

逐深度损失是对真实 `t_{i+k}` 的交叉熵：

```
L_k = CE(logits_{i+k}, t_{i+k})
```

跨深度的联合损失：

```
L_MTP = (lambda / D) * sum_{k=1..D} L_k
```

`lambda` 是一个小的加权因子——DeepSeek-V3 在训练的前 10% 使用 0.3，之后使用 0.1。总训练损失为 `L_main + L_MTP`。

### 为什么是顺序的，而非并行的

Gloeckle 的原始并行 MTP 有 D 个输出头，每个直接应用于 `h_i^(0)`。每个头从相同的骨干隐藏状态预测 `t_{i+k}`。这训练起来没问题，但预测不互为条件。你不能使用 `head_1` 的输出来帮助 `head_2`——这些头并行触发。

DeepSeek-V3 的顺序设计从 `h_i^(k-1)` 加上实际的下一个 token 嵌入 `E(t_{i+k})` 构建 `h_i^(k)`。这保持了因果链：为了预测 `t_{i+k+1}`，深度 `k+1` 处的模块看到 `t_{i+k}` 处的内容。这在结构上与自回归解码器消费自身输出的方式相同——使 MTP 模块可直接用作推测解码草稿器。

推理时：将 `h_i^(k-1)` 和草拟的 `t_{i+k}` 送入模块 `k+1`，获得对 `t_{i+k+1}` 的预测。重复。这正是 EAGLE 风格的草稿，使用训练好的 MTP 模块作为草稿网络。DeepSeek-V3 报告在第一个 MTP 模块上接受率超过 80%，加速约 1.8×。

### 参数核算

对于隐藏维度 `h` 和词汇量 `V` 的模型：

- 主模型：数十亿参数，加上大小为 `V * h` 的一个输出头。
- 共享输出头：重用主模型的输出头。无额外参数。
- 共享嵌入：重用主模型的嵌入。无额外参数。
- 每个 MTP 模块：
  - 投影 `M_k`：`(2h) * h = 2h^2`。
  - Transformer 块 `T_k`：注意力（MHA 为 `4h^2`）加上 MLP（SwiGLU 比率为 8/3 时通常 `8h^2`）。每个块约 `12h^2`。

每个模块总计额外参数：`~14h^2`。对于 DeepSeek-V3 的 `h = 7168`，D = 1 个模块：`~14 * 7168^2 = ~720M` 参数（纸面值）。DeepSeek-V3 报告 14B——差异主要在于 MTP 模块中的专家层也是 MoE。

### 推测解码回报

在预训练期间，MTP 模块减慢训练约 10%（更多前向计算，额外损失）。回报是双重的：

1. 更密集的训练信号。每个隐藏状态看到 D+1 个监督目标。DeepSeek-V3 的消融实验显示在 MMLU、GSM8K、MATH、HumanEval 上的测量效果：一致的几个百分点提升。

2. 推理时免费获得推测解码草稿。MTP 模块已经被训练来预测后续几个 token。重新用作草稿网络，它提供 80%+ 的接受率。在该水平，N=3 或 N=5 推测解码提供 1.8× 吞吐量。10% 的训练时间成本在第一次运行推理时就已回本。

### 与 EAGLE 的关系

EAGLE 在预训练之后单独训练一个小草稿模型。MTP 将草稿融入到预训练中。两种方法在相似的接受率上汇聚，但通过不同的流水线：

| 维度 | EAGLE-3 | MTP（DeepSeek-V3） |
|-----------|---------|------------------|
| 训练时机 | 预训练后 | 预训练期间 |
| 与已有权重后向兼容 | 是 | 否（需要重新训练） |
| 草稿参数 | 1-2 transformer 层 | 1 transformer 块 + 投影 |
| 接受率 | 0.88-0.92 | 深度 1 处 0.80+ |
| 超越加速的收益 | 仅推测解码 | 更密集的训练信号 + 加速 |

## 构建实现

`code/main.py` 端到端构建单个 MTP 模块：共享嵌入、投影、transformer 块、共享输出头。然后计算在短合成序列上的逐深度交叉熵损失，并按组件打印参数计数。32 token 的玩具词汇量保持数字可读。

### 步骤 1：共享嵌入表

单个 `vocab_size x hidden` 表被主模型和每个深度的每个 MTP 模块使用。不是第二份副本——字面上是同一个张量。

### 步骤 2：逐深度组合

```python
def combine(prev_hidden, next_token_embed, M_k):
    # 沿特征维度拼接，然后投影回 hidden
    concat = rms_norm(prev_hidden) + rms_norm(next_token_embed)  # 向量加法代替
    projected = matvec(M_k, concat)
    return projected
```

真实 DeepSeek-V3 将两个 RMSNorm 后的向量拼接为 `[2h]` 并用 `h x 2h` 矩阵投影。玩具使用向量加法以简化标准库实现。

### 步骤 3：深度 k 处的 transformer 块

自注意力加 MLP。在玩具中，一个单层线性注意力块和一个 SwiGLU MLP 使结构在没有 numpy 的情况下可见。

### 步骤 4：共享输出头

重用主模型的输出投影。在词汇表上的 logits。

### 步骤 5：逐深度损失

softmax(logits) 对偏移 `k` 处的真实 token 的交叉熵。用 `lambda / D` 缩放因子在深度间聚合。

### 步骤 6：参数核算

打印总参数数量、共享（嵌入、输出头）数量和每个模块的额外数量。显示 MTP 额外参数与主模型大小的比率。

## 使用方式

MTP 已集成到 DeepSeek-V3（2024 年 12 月）和 DeepSeek-R1 系列中。推理时：

- DeepSeek 自身的服务栈开箱即用地将 MTP 模块作为推测解码器消费。
- 截至 2026 年 4 月，vLLM 和 SGLang 有 DeepSeek-V3 MTP 的集成路径。
- AMD 的 ROCm SGLang 教程展示了一个具体的 MTP 推测解码配置，在 V3 检查点上测得 1.8× 加速。

何时在新的预训练运行中使用 MTP：

- 你控制完整的预训练流水线，并希望储备更密集的训练信号。
- 你知道你将大规模服务模型，并希望免费获得推测解码。
- 你的隐藏维度至少为 4096。在 1B 规模下开销比收益更大。

何时不使用：

- 微调已有预训练稠密模型。MTP 模块未训练。
- 你想要一个干净基线来比较的研究模型。MTP 改变了架构。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|----------------|------------------------|
| MTP 模块 | "额外损失块" | 一个小型 transformer 块加上投影，预测主模型之前 k 个位置的 token |
| 预测深度 | "哪个偏移" | 整数 `k`，使得模块 `k` 从通过位置 `i` 的前缀预测 `t_{i+k}` |
| 并行 MTP | "Gloeckle 风格" | 在相同骨干隐藏状态上的 D 个独立头，无条件链 |
| 顺序 MTP | "DeepSeek-V3 风格" | 每个模块以前一深度隐藏状态加上下一个 token 的嵌入为条件；保持因果链 |
| 共享输出头 | "重用主输出头" | MTP 模块调用主模型的 LM 头，而非单独的输出投影 |
| 共享嵌入 | "重用主嵌入表" | 相同的词汇嵌入表处处使用；无重复参数 |
| 投影矩阵 M_k | "组合隐藏 + 下一个 token" | 将前一隐藏状态和目标 token 嵌入折叠到下一深度输入的 `h x 2h` 线性层 |
| 联合损失 L_MTP | "平均额外损失" | 逐深度交叉熵损失的算术平均，按 `lambda` 缩放 |
| 深度 1 接受率 | "MTP 草稿正确的频率" | D=1 MTP 模块的 top-1 预测等于主模型 top-1 预测的比率；DeepSeek-V3 上为 80%+ |
| Lambda 加权 | "额外损失重要性" | 逐深度缩放因子；DeepSeek-V3 上训练开始时为 0.3，之后为 0.1 |
