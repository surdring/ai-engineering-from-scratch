# DPO：直接偏好优化

> RLHF 有效。但它也需要训练三个模型（SFT、奖励模型、策略），管理 PPO 的不稳定性，并调整 KL 惩罚。DPO 问：如果你可以跳过所有这些呢？DPO 直接在偏好对上优化语言模型。没有奖励模型。没有 PPO。一个训练循环。相同的结果。

**类型：** 构建
**语言：** Python（使用 numpy）
**前置要求：** 第 10 阶段，第 07 课（RLHF）
**时间：** 约 90 分钟

## 学习目标

- 实现 DPO 训练，直接在偏好对上优化语言模型，无需单独的奖励模型
- 推导 DPO 损失函数并解释它如何通过策略的对数概率隐式表示奖励模型
- 比较 DPO 和 RLHF 在训练稳定性、计算成本和所需模型数量方面的差异
- 调整 beta 参数以控制训练后策略偏离参考模型的程度

## 问题

你在第 07 课中构建了一个 RLHF 管道。三个阶段。三个模型。SFT 模型、奖励模型和用 PPO 优化的策略模型。仅奖励模型就需要数千个人类偏好对和一个单独的训练循环。PPO 需要仔细调整 KL 系数、学习率、裁剪比率和 epoch 数量。

在实践中，PPO 训练是出了名的不稳定。小的超参数变化导致训练发散。奖励模型是人类偏好的不完美代理，策略会找到利用其弱点的方法。KL 惩罚有帮助但需要自己的调整——太低你会得到奖励破解，太高模型几乎学不到东西。

这种复杂性是为什么大多数开源模型在 InstructGPT 发布后的几年里都在 RLHF 上挣扎。三阶段管道是脆弱的。每个阶段都有自己的失败模式，错误会累积。

2023 年 5 月，斯坦福的 Rafael Rafailov、Archit Sharma 及其同事发表了"直接偏好优化：你的语言模型秘而不宣地是一个奖励模型"。关键洞察：你不需要一个单独的奖励模型。最优奖励函数由语言模型自身的 token 概率数学确定。你可以完全跳过奖励模型，直接优化语言模型在偏好对上。

DPO 将 RLHF 简化为一个单独的监督学习步骤。一个模型。一个损失函数。一个训练循环。没有强化学习。Zephyr-7B 是首批大规模使用 DPO 的模型之一，在多个基准上匹配或超越了用完整 RLHF 训练的模型。Meta 使用 DPO 作为 Llama 3 对齐管道的一部分。Anthropic 在其对齐研究中引用了 DPO 式方法。

## 概念

### 关键洞察

RLHF 优化此目标：

```
最大化：E[R(x, y)] - beta * KL(pi || pi_ref)
```

其中 R 是奖励模型，pi 是策略，pi_ref 是参考模型，beta 是 KL 系数。

DPO 论文表明这个目标有一个闭式最优解。对于任何奖励函数 R，最优策略是：

```
pi*(y | x) = pi_ref(y | x) * exp(R(x, y) / beta) / Z(x)
```

其中 Z(x) 是归一化常数。重排：

```
R(x, y) = beta * log(pi*(y | x) / pi_ref(y | x)) + beta * log Z(x)
```

这就是突破。奖励完全用策略模型的概率和参考模型的概率表示。你不需要训练单独的奖励模型。奖励在概率比率中*隐式*存在。

将其代入 Bradley-Terry 偏好模型：

```
P(y_w > y_l | x) = sigmoid(R(x, y_w) - R(x, y_l))
                  = sigmoid(beta * (log pi(y_w|x)/pi_ref(y_w|x) - log pi(y_l|x)/pi_ref(y_l|x)))
```

Z(x) 项抵消了，因为两个响应以相同的提示 x 为条件。剩下的只是策略模型的对数概率和参考模型在偏好和被拒绝响应上的对数概率的函数。

### DPO 损失

```
L_DPO = -log(sigmoid(beta * (log pi(y_w|x)/pi_ref(y_w|x) - log pi(y_l|x)/pi_ref(y_l|x))))
```

让我们拆解每个部分：

- **y_w** = 偏好（赢）响应
- **y_l** = 被拒绝（输）响应
- **x** = 提示
- **pi** = 当前模型（正在训练）
- **pi_ref** = 参考模型（冻结的 SFT 检查点）
- **beta** = 控制偏离参考程度的温度参数（通常 0.1 到 0.5）

比率 `log pi(y|x) / pi_ref(y|x)` 是对数概率比率。当这个比率为正时，当前模型给响应 y 分配比参考更高的概率。为负时，当前模型分配更低概率。

DPO 损失推动模型增加偏好响应的对数概率比率，减少被拒绝响应的对数概率比率。beta 参数控制模型可以多激进地偏离参考——小 beta 意味着允许大偏离，大 beta 保持模型接近参考。

### DPO vs RLHF：并排比较

| 方面 | RLHF | DPO |
|------|------|-----|
| 所需模型 | 3（SFT + 奖励模型 + 策略） | 2（SFT + 策略；无单独奖励模型） |
| 训练步骤 | 3（SFT → RM → PPO） | 2（SFT → DPO） |
| 奖励信号 | 奖励模型预测 | 从对数概率比率隐式 |
| 稳定性 | 低：PPO 对超参数敏感 | 高：标准分类损失，无 RL |
| 奖励破解风险 | 高（RM 是单独的有盲点的代理） | 低（无单独奖励模型可被利用） |
| 每次迭代计算 | 策略前向 + 奖励模型前向 + PPO 裁剪 | 仅对数概率（一次前向） |
| 开源采用 | 需要基础设施；最初难以复现 | 单脚本可实现；广泛采用 |

### DPO 何时失败

DPO 不是 RLHF 的严格替代品。它在某些模式下失败：

**分布外偏好。** DPO 根据参考模型下偏好响应的概率加权训练。如果偏好数据分布与参考模型分布相差太远——例如人类标注者偏好冗长、引文丰富的 STEM 响应，而参考模型默认生成简洁摘要——DPO 相对于 RLHF 是低效的。RLHF 可以通过奖励模型学习捕捉这种偏好差异，奖励模型可以从头开始专门在偏好标签上训练。

**迭代训练。** 在实际中，RLHF 经常以迭代方式部署：收集偏好数据 → 训练 RM → 做 PPO → 部署 → 收集更多偏好数据 → 重新训练 RM → 做更多 PPO。DPO 每次迭代需要一个参考模型，这在你有 10 轮在线数据时变成后勤噩梦。在线 RLHF（带滚动策略的数据收集）改进了离线 RLHF，但不清楚如何在标准 DPO 中实现等效的在线数据收集，除了一次又一次地重新合成数据集。

**多轮对话。** DPO 原生优化单轮（提示，响应）对。将 DPO 扩展到多轮是一个活跃的研究领域。实际模型通常为多轮使用 SFT，仅对最后一轮响应使用 DPO。

### 实践中的 Beta

Beta 控制模型可以偏离参考多远。它的行为类似于 RLHF 中的 KL 惩罚系数：

- **Beta = 0.1（小）：** 模型大量偏离参考。学得更快。如果偏好标签噪声大，风险更高。
- **Beta = 0.5（大）：** 模型保持接近参考。更安全。如果偏好信号弱，学习更慢。

Llama 3 使用 beta 值约为 0.1。Zephyr 使用 0.01。选择还取决于你的偏好对有多干净——更高质量的标签允许更小的 beta（更强的学习），而更多噪声标签需要更大的 beta（更保守）。

## 练习

1. **简单。** 为在 SFT 和 RLHF 中使用的相同 5 个提示创建偏好对。在玩具参考模型上运行 DPO，验证所选响应获得更高概率。
2. **中等。** 在相同偏好对上用 beta = 0.01、0.1、0.5 比较 DPO。测量训练后偏好响应和被拒绝响应之间的对数概率差距。哪个 beta 产生最大的差距而不破坏连贯性？
3. **困难。** 解释为什么 DPO 在某些情况下可以胜过 RLHF，而在其他情况下失败。在在线 RLHF 的背景下分析 DPO 的数学推导——"离线"约束将什么假设烘焙到优化中？

## 关键术语

| 术语 | 含义 |
|------|------|
| DPO | 直接偏好优化：直接通过 log p 在偏好对上优化。 |
| 对数比率 | log(pi(y|x) / pi_ref(y|x))：当前模型概率与参考模型概率的比率。 |
| Bradley-Terry | 偏好概率模型：P(A > B) = sigmoid(r_A - r_B)。 |
| 参考模型 | 冻结的 SFT 检查点，用于约束 DPO 更新。 |
| 在线 vs 离线 | 从当前策略收集数据 vs 使用固定数据集。 |
| Beta | 温度参数：越高，模型应越接近参考。 |
