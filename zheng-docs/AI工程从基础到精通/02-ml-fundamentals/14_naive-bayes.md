# 朴素贝叶斯

> "朴素"假设是错误的，但它仍然有效。这就是它的美妙之处。

**类型：** Build
**语言：** Python
**前置知识：** 阶段 2 第 01-07 课（分类、贝叶斯定理）
**时间：** 约 75 分钟

## 学习目标

- 从零实现带拉普拉斯平滑的多项式朴素贝叶斯用于文本分类
- 解释为什么朴素独立性假设在数学上错误，但在实践中产生正确的类别排序
- 比较多项目式、伯努利和高斯朴素贝叶斯变体，为给定特征类型选择合适的变体
- 在高维稀疏数据上评估朴素贝叶斯与逻辑回归，解释其中的偏差-方差权衡

## 问题

你需要对文本分类。邮件分为垃圾或非垃圾。客户评价分为正面或负面。工单分为不同类别。你有数千个特征（每个词一个）和有限的训练数据。

大多数分类器在这里出问题。逻辑回归需要足够的样本可靠地估计成千上万个权重。决策树一次在一个词上分裂，疯狂过拟合。在 10,000 维空间中的 KNN 没有意义，因为每个点到其他每个点的距离都一样。

朴素贝叶斯处理这个。它做了一个数学上错误的假设（给定类别下每个特征独立于其他所有特征），但仍在文本分类上超越"更聪明"的模型，特别是在小训练集上。它单次遍历数据即可训练。它扩展到数百万特征。它产生概率估计（虽然由于独立性假设通常校准不佳）。

理解为什么一个错误的假设会带来好的预测，教会你机器学习中一些根本性的东西：最好的模型不是最正确的，而是对你的数据具有最佳偏差-方差权衡的那个。

## 概念

### 贝叶斯定理（快速复习）

贝叶斯定理翻转条件概率：

```
P(类别 | 特征) = P(特征 | 类别) * P(类别) / P(特征)
```

我们想要 `P(类别 | 特征)` —— 一个文档属于某类别的概率，给定其中的词。我们可以从以下计算：
- `P(特征 | 类别)` —— 在该类别文档中看到这些词的似然
- `P(类别)` —— 类别的先验概率
- `P(特征)` —— 证据，对所有类别相同，比较时可以忽略

`P(类别 | 特征)` 最高的类别胜出。

### 朴素独立性假设

精确计算 `P(特征 | 类别)` 需要估计所有特征的联合概率。有 10,000 个词的词汇表，你需要估计超过 2^10,000 种可能组合的分布。不可能。

朴素假设：给定类别下，每个特征与其他特征条件独立。

```
P(w1, w2, ..., wn | 类别) = P(w1 | 类别) * P(w2 | 类别) * ... * P(wn | 类别)
```

不再是一个不可能的联合分布，你估计 n 个简单的单特征分布。每个只需要一个计数。

这个假设显然是错的。词语"机器"和"学习"在任何文档中都不是独立的。但分类器不需要正确的概率估计。它需要正确的排名——哪个类别的概率最高。独立性假设引入了系统性误差，但这些误差对所有类别的影响类似，所以排名保持正确。

### 为什么它仍然有效

三个原因：

1. **排名优先于校准。** 分类只需要排名最高的类别正确。即使 P(垃圾) = 0.99999 而真实概率是 0.7，分类器仍然正确选择垃圾。我们不需要正确的概率，需要正确的胜者。

2. **高偏差，低方差。** 独立性假设是一个强先验。它严重约束模型，防止过拟合。在有限训练数据下，一个不完全正确但稳定的模型优于理论正确但不稳定的模型。这是偏差-方差权衡在起作用。

3. **特征冗余互相抵消。** 相关特征提供冗余证据。分类器重复计算这个证据，但它也为正确的类别重复计算。如果"机器"和"学习"总是一起出现，两者都为"技术"类别提供证据。NB 计算两次，但为正确的类别计算两次。

第四个实用原因：朴素贝叶斯极快。训练是单次遍历数据计频。预测是矩阵乘法。你可以在几秒钟内训练百万份文档。这种速度意味着你可以更快迭代、尝试更多特征集、比更慢的模型跑更多实验。

### 逐步数学推导

让我们通过一个具体示例追踪。假设有两个类别：垃圾和非垃圾。词汇表有三个词："免费"、"钱"、"会议"。

训练数据：
- 垃圾邮件提到"免费" 80 次、"钱" 60 次、"会议" 10 次（总共 150 词）
- 非垃圾邮件提到"免费" 5 次、"钱" 10 次、"会议" 100 次（总共 115 词）
- 40% 的邮件是垃圾，60% 是非垃圾

使用拉普拉斯平滑（alpha=1）：

```
P(免费 | 垃圾)    = (80 + 1) / (150 + 3) = 81/153 = 0.529
P(钱   | 垃圾)    = (60 + 1) / (150 + 3) = 61/153 = 0.399
P(会议 | 垃圾)    = (10 + 1) / (150 + 3) = 11/153 = 0.072

P(免费 | 非垃圾)  = (5 + 1) / (115 + 3)  = 6/118  = 0.051
P(钱   | 非垃圾)  = (10 + 1) / (115 + 3) = 11/118 = 0.093
P(会议 | 非垃圾)  = (100 + 1) / (115 + 3) = 101/118 = 0.856
```

新邮件包含："免费"（2 次）、"钱"（1 次）、"会议"（0 次）。

```
log P(垃圾 | 邮件)    = log(0.4) + 2*log(0.529) + 1*log(0.399) + 0*log(0.072)
                      = -0.916 + 2*(-0.637) + (-0.919) + 0 = -3.109

log P(非垃圾 | 邮件) = log(0.6) + 2*log(0.051) + 1*log(0.093) + 0*log(0.856)
                      = -0.511 + 2*(-2.976) + (-2.375) + 0 = -8.838
```

垃圾邮件以大幅优势胜出。"免费"出现两次是垃圾邮件的强有力证据。

### 三种变体

朴素贝叶斯有三种形式。每种以不同方式建模 `P(特征 | 类别)`。

#### 多项朴素贝叶斯

将每个特征建模为计数。最适合文本数据，特征是词频或 TF-IDF 值。

```
P(词_i | 类别) = (类别中词_i的计数 + alpha) / (类别总词数 + alpha * 词汇表大小)
```

`alpha` 是拉普拉斯平滑。这个变体是文本分类的主力。

#### 高斯朴素贝叶斯

将每个特征建模为正态分布。最适合连续特征。

```
P(x_i | 类别) = (1 / sqrt(2 * pi * var)) * exp(-(x_i - mean)^2 / (2 * var))
```

每个类别每个特征有自己的均值和方差。当特征在每个类别内确实遵循钟形曲线时效果很好。

#### 伯努利朴素贝叶斯

将每个特征建模为二元（存在或不存在）。最适合短文本或二元特征向量。

```
P(词_i | 类别) = (包含词_i的类别文档数 + alpha) / (总类别文档数 + 2 * alpha)
```

不像多项式，伯努利明确惩罚词的缺失。如果"免费"通常出现在垃圾邮件中但在这封邮件中缺失，伯努利计为反垃圾证据。

### 何时使用每种变体

| 变体 | 特征类型 | 最适合 | 示例 |
|---------|-------------|----------|---------|
| 多项式 | 计数或频率 | 文本分类，词袋 | 邮件垃圾过滤，主题分类 |
| 高斯 | 连续值 | 具有正态分布特征的表格数据 | 鸢尾花分类，传感器数据 |
| 伯努利 | 二元（0/1） | 短文本，二元特征向量 | 短信垃圾过滤，存在/缺失特征 |

### 拉普拉斯平滑

当一个词出现在测试数据中但从未在训练数据的某个类别中出现时怎么办？

无平滑：`P(词 | 类别) = 0/N = 0`。一个零乘过整个乘积使 `P(类别 | 特征) = 0`，无论其他证据如何。一个未见过的词摧毁整个预测。

拉普拉斯平滑为每个特征计数加一个小值 `alpha`（通常为 1）：

```
P(词_i | 类别) = (count(词_i, 类别) + alpha) / (类别总词数 + alpha * 词汇表大小)
```

alpha=1 时，每个词至少获得微小概率。测试邮件中的词"discombobulate"不再杀死垃圾概率。平滑有贝叶斯解释：等同于对词分布放置均匀 Dirichlet 先验。

| Alpha | 效果 | 何时使用 |
|-------|--------|-------------|
| 0.001 | 几乎不平滑，信任数据 | 很大的训练集，不期望有未见特征 |
| 0.1 | 轻微平滑 | 大训练集 |
| 1.0 | 标准拉普拉斯平滑 | 默认起点 |
| 10.0 | 强平滑，摊平分布 | 很小的训练集，期望有很多未见特征 |

### 对数空间计算

乘以数百个概率（每个小于 1）导致浮点下溢。乘积在浮点中变为零，尽管真实值是一个非常小的正数。

解决方案：在对数空间中工作。不是乘概率，而是加它们的对数：

```
log P(类别 | x1, x2, ..., xn) = log P(类别) + sum_i log P(x_i | 类别)
```

这将预测转化为点积：

```
log_scores = X @ log_feature_probs.T + log_class_priors
prediction = argmax(log_scores)
```

矩阵乘法。这就是为什么朴素贝叶斯预测如此之快——它与单层线性模型相同的操作。

### 朴素贝叶斯 vs 逻辑回归

两者都是文本的线性分类器。区别在于它们建模什么。

| 方面 | 朴素贝叶斯 | 逻辑回归 |
|--------|------------|-------------------|
| 类型 | 生成式（建模 P(X|Y)） | 判别式（建模 P(Y|X)） |
| 训练 | 计频 | 优化损失函数 |
| 小数据 | 更好（强先验有帮助） | 更差（不足以估计权重） |
| 大数据 | 更差（错误假设拖累） | 更好（灵活边界） |
| 特征 | 假设独立 | 处理相关性 |
| 速度 | 单次遍历，非常快 | 迭代优化 |
| 校准 | 概率差 | 概率更好 |

经验法则：从朴素贝叶斯开始。如果有足够数据且 NB 达到瓶颈，切换到逻辑回归。

## Build It

`code/naive_bayes.py` 中的代码从零实现 MultinomialNB 和 GaussianNB。

### MultinomialNB

1. **fit(X, y)**：对每个类别，计算每个特征的频次。加拉普拉斯平滑。计算对数概率。存储类别先验。

2. **predict_log_proba(X)**：对每个样本，计算所有类别的 `log P(类别) + sum of log P(特征_i | 类别)`。这是矩阵乘法：`X @ log_probs.T + log_priors`。

3. **predict(X)**：返回对数概率最高的类别。

```python
class MultinomialNB:
    def __init__(self, alpha=1.0):
        self.alpha = alpha

    def fit(self, X, y):
        classes = np.unique(y)
        n_classes = len(classes)
        n_features = X.shape[1]

        self.classes_ = classes
        self.class_log_prior_ = np.zeros(n_classes)
        self.feature_log_prob_ = np.zeros((n_classes, n_features))

        for i, c in enumerate(classes):
            X_c = X[y == c]
            self.class_log_prior_[i] = np.log(X_c.shape[0] / X.shape[0])
            counts = X_c.sum(axis=0) + self.alpha
            self.feature_log_prob_[i] = np.log(counts / counts.sum())

        return self
```

### GaussianNB

对连续特征，我们估计每个类别每个特征的均值和方差：

```python
class GaussianNB:
    def __init__(self):
        pass

    def fit(self, X, y):
        classes = np.unique(y)
        self.classes_ = classes
        self.means_ = np.zeros((len(classes), X.shape[1]))
        self.vars_ = np.zeros((len(classes), X.shape[1]))
        self.priors_ = np.zeros(len(classes))

        for i, c in enumerate(classes):
            X_c = X[y == c]
            self.means_[i] = X_c.mean(axis=0)
            self.vars_[i] = X_c.var(axis=0) + 1e-9
            self.priors_[i] = X_c.shape[0] / X.shape[0]

        return self
```

代码还演示了：
- **平滑比较：** 用不同 alpha 值训练 MultinomialNB 展示平滑强度对准确率的影响。
- **训练大小实验：** NB 准确率如何随训练数据从 20 增长到 1600 样本而改善。NB 即使样本很少也能达到不错准确率——这是它的主要优势。
- **混淆矩阵：** 每个类别的精确率、召回率和 F1 分数。

## Use It

使用 sklearn，两个变体都是一行代码：

```python
from sklearn.naive_bayes import GaussianNB, MultinomialNB

gnb = GaussianNB()
gnb.fit(X_train, y_train)
print(f"GaussianNB 准确率: {gnb.score(X_test, y_test):.3f}")

mnb = MultinomialNB(alpha=1.0)
mnb.fit(X_train_counts, y_train)
print(f"MultinomialNB 准确率: {mnb.score(X_test_counts, y_test):.3f}")
```

### TF-IDF 与朴素贝叶斯

原始词频给每个词的每次出现相等权重。但像"的"和"是"这样的常见词在每个类别中都频繁出现——它们不携带信息。TF-IDF 降低常见词权重，提升稀有的有区分力的词。

```python
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.naive_bayes import MultinomialNB
from sklearn.pipeline import Pipeline

text_clf = Pipeline([
    ("tfidf", TfidfVectorizer()),
    ("classifier", MultinomialNB(alpha=0.1)),
])
```

TF-IDF 值非负，所以可以用 MultinomialNB。TF-IDF + MultinomialNB 是文本分类最强的基线之一。

### 校准 NB 概率

NB 概率校准差。当 NB 说 P(垃圾) = 0.95，真实概率可能是 0.7。如果需要可靠的概率估计，使用 sklearn 的 CalibratedClassifierCV：

```python
from sklearn.calibration import CalibratedClassifierCV

calibrated_nb = CalibratedClassifierCV(MultinomialNB(), cv=5, method="sigmoid")
calibrated_nb.fit(X_train, y_train)
proba = calibrated_nb.predict_proba(X_test)
```

### 常见陷阱

1. **负特征值。** MultinomialNB 要求非负特征。如果有负值，用 GaussianNB 代替。
2. **零方差特征。** GaussianNB 除以方差。如果某特征在类别中方差为零，概率计算崩溃。代码给所有方差加小平滑项（1e-9）防止。
3. **类别不平衡。** 如果 99% 的邮件是非垃圾，先验 P(非垃圾) = 0.99 强到淹没似然证据。可以手动设置类别先验或使用 sklearn 的 class_prior 参数。
4. **特征缩放。** MultinomialNB 不需要缩放（处理的是计数）。GaussianNB 也不需要缩放（它估计每个特征的统计量）。这是优于逻辑回归和 SVM 的优点。

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|------|----------------|----------------------|
| 朴素贝叶斯 | "独立假设分类器" | 使用贝叶斯定理的分类器，假设所有特征在给定类别下独立以简化建模 |
| 拉普拉斯平滑 | "给一切加 +1" | 为所有计数加一个小值以防止零概率，确保测试数据中的未见特征不会清零似然 |
| 生成式分类器 | "建模数据如何生成" | 学习联合分布 P(X,Y) 然后用贝叶斯定理推导 P(Y\|X)，不同于直接建模 P(Y\|X) 的判别式分类器 |
| 多项 NB | "对词计数" | 假设特征是给定类别的离散计数的朴素贝叶斯变体，用于文本分类 |
| 高斯 NB | "对连续值拟合钟形曲线" | 假设特征在给定类别下遵循高斯（正态）分布的朴素贝叶斯变体 |
| 伯努利 NB | "只关心是否出现" | 将特征建模为二元（存在/缺失）的朴素贝叶斯变体，显式惩罚缺失的词 |
| 对数空间 | "加不乘" | 使用概率的对数而不是概率自身以防止浮点下溢，将乘积转为求和 |
| 概率校准 | "修正过度自信" | 调整模型输出的概率使其匹配真实频率，通常通过 Platt 缩放或保序回归进行后处理 |
| 条件独立性 | "特征不交互" | 给定类别标签后特征独立于彼此的朴素假设，通常为假但适用于排序 |
