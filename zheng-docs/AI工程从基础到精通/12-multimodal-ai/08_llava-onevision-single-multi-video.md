# LLaVA-OneVision：单图像、多图像、视频合一的模型

> 在 LLaVA-OneVision（Li 等人，2024 年 8 月）之前，开源 VLM 世界有独立的谱系：LLaVA-1.5 用于单图像，Mantis 和 VILA 等多图像模型，Video-LLaVA 和 Video-LLaMA 等视频模型。每个在其基准上获胜，在其他上失败。LLaVA-OneVision 论证了单一课程（Curriculum）可以训练一个模型在所有三种场景中占主导地位，并且涌现的任务迁移效应（单图像技能导出到视频，多图像推理导出到单图像）优于专家之和。方案出奇地简单：一个在场景之间保持恒定的视觉 token 预算，加上一个从单图像到 OneVision（多图像）再到视频的显式课程。本课阅读预算、课程和涌现行为。

**类型：** 构建
**语言：** Python（标准库，token 预算求解器 + 课程规划器）
**前置条件：** Phase 12 · 05（LLaVA）、Phase 12 · 06（任意分辨率）
**时间：** 约 180 分钟

## 学习目标

- 设计一个在单图像、多图像和视频输入之间保持恒定的视觉 token 预算。
- 排序训练课程，将技能从单图像迁移到视频而没有灾难性遗忘。
- 解释为什么当课程做对时，单个模型在相同参数数量下优于专家模型。
- 指出 LLaVA-OneVision 报告的三种涌现能力：多摄像头推理、标记集合提示（Set-of-Mark Prompting）、iPhone 截图代理。

## 问题所在

图像、多图像和视频各自以不同方式对模型施压。

单图像希望高分辨率 token（AnyRes，约 2880 个视觉 token）以捕获 OCR 和细粒度细节。每样本预算：一张图像，2880 个 token。

多图像希望几张中等分辨率的图像（约每张 576 个 token），使跨图像推理能适应上下文。每样本预算：4-8 张图像，每张 576，2300-4600 个 token。

视频希望低分辨率的很多帧（池化后约每帧 196 个 token）以捕获时序动态。每样本预算：8-32 帧，每帧 196，1600-6200 个 token。

如果你训练独立的模型，你选择一个预算。如果你训练一个模型，你需要预算在场景之间合理缩放而不会撑爆上下文。

OneVision 之前，默认答案是「训练一种场景，忽略其他」。Video-LLaVA 通过额外的训练阶段将视频改造到图像模型上。LLaVA-NeXT 通过平铺增加了多图像支持。没有哪个干净地处理了全部三种。

## 核心概念

### OneVision Token 预算

LLaVA-OneVision 选择一个统一的视觉 token 预算，每样本约 3000-4000 个 token，按场景不同分配：

- 单图像：AnyRes-9（3x3 瓦片 + 缩略图），每个瓦片在 384 下以 729 个块，积极的 2x2 双线性池化 → 每瓦片 182。总计：9 * 182 + 182 = 1820 个 token。或 AnyRes-4 以每瓦片 729 = 2916 + 729。
- 多图像：每张图像中等分辨率（384，不平铺），729 个 token 无池化。预算 6 张图像 → 4374 个 token。
- 视频：32 帧，384 分辨率，积极的 3x3 双线性池化 → 每帧 81 个 token。总计：32 * 81 = 2592 个 token。

分配保持了大致恒定的总 token。LLM 永远不会看到撑爆其上下文的批次。编码器在每个场景下产生不同的几何形状，但 LLM 消费相同的预算。

### 三阶段课程

LLaVA-OneVision 分三个阶段训练：

1. 单图像 SFT（阶段 SI）。所有数据是单图像加文本。在高分辨率 AnyRes 输入上训练。这教会感知、OCR 和细粒度理解。使用 LLaVA-NeXT 数据加上 OneVision 特定的单图像数据。
2. OneVision SFT（阶段 OV）。混合单图像 + 多图像 + 视频（均匀采样帧）。在统一的 token 预算上训练。这教会模型处理异构批次形状。不重置权重——从阶段 SI 继续。
3. 任务迁移（阶段 TT）。继续带有目标任务混合，通常根据产品在多图像或视频上更重。可选的部署微调。

关键：课程顺序很重要。先训练视频或先训练多图像比先训练单图像产生更差的图像性能，即使数据相同。论文明确对此做了消融。

### 为什么课程有效

单图像训练构建感知基础。Patch token 携带细粒度的视觉特征；LLM 学会将它们与文本集成。多图像和视频引入了在没有强大感知基础时难以学习的结构挑战（哪张图像是哪张、什么先发生）。

如果你从零开始一起训练所有场景，模型感知欠拟合（每批次单图像数据有限）而结构过拟合（大量多图像/视频数据）。结果：一个模型能遵循跨图像推理模式但视觉上肤浅。

课程排序从阶段 SI 给你感知强度，然后从阶段 OV 给你组合/时序推理，不丢失任何一方。

### 涌现的跨场景技能

LLaVA-OneVision 论文报告了三种涌现能力：

1. 多摄像头推理。在多图像 + 视频上分开训练；在推理时，被要求推理多摄像头驾驶场景。模型正确整合了视角，尽管从未在训练中看到那种确切格式。
2. 标记集合提示（Set-of-mark prompting）。用户用编号标记标注图像中的物体；模型推理「标记 3 相对于标记 7 在做什么」。既没有在标记上训练，也没有在标注上训练；从空间定位 + 多图像引用的组合中学习。
3. iPhone 截图代理。用户提供 iPhone 屏幕截图，要求规划下一次点击。在 UI 截图、用户工作流视频和多图像前后对比对上训练。泛化到代理用例。

这些不是训练的明确任务；它们从课程的组合结构中涌现出来。

### 视觉 Token 池化

Token 预算需要池化。OneVision 在 2D 块网格上使用双线性插值：24x24 = 576 个块变成 12x12 = 144（2x 因子）或 8x8 = 64（3x 因子）。池化在块网格空间中完成，而非 token 空间，以保持局部性。

每个场景的池化因子选择本身就是一个超参数。更少的池化 = 更多的 token = 更丰富的表示。更多的池化 = 更少的 token = 能容纳更多帧/图像。

### LLaVA-OneVision-1.5

2025 年的后续版本（LLaVA-OneVision-1.5，arXiv 2509.23661）在训练数据、模型权重和代码方面「完全开放」。在某些基准上匹配专有差距，并民主化了方案。相同的课程，更多的数据，更好的基础 LLM。没有架构变化。

### 与 Qwen2.5-VL 的对比

Qwen2.5-VL（第 12.09 课）做出不同的选择。它使用 M-RoPE 和动态 FPS 而不是固定池化。其预算随输入缩放——1 分钟视频比 5 秒视频使用更多的 token。LLaVA-OneVision 固定预算并缩放池化。两者都有效；它们在可配置性和可预测性之间进行权衡。

## 使用指南

`code/main.py` 是 OneVision 风格 VLM 的课程和预算规划器。给定每样本的 token 预算和目标任务场景混合（比如 40% 单图像、30% 多图像、30% 视频），它：

- 分配每个场景的分辨率、池化因子和帧数。
- 检查每个场景是否适应共享预算。
- 报告期望的 token 数量、LLM FLOPs 以及哪些场景 token 不足。
- 打印逐阶段的训练调度。

使用它来规划 OneVision 微调或为 VLM 部署的每请求成本做合理性检查。

## 交付物

本课产出 `outputs/skill-onevision-budget-planner.md`。给定目标的任务分布和每样本预算，它发出 AnyRes 因子、每帧池化、视频帧数和课程阶段权重。每次训练或微调统一场景 VLM 时使用此技能。

## 练习

1. 你的产品支持 80% 单图像、10% 多图像（2-4 张图像）、10% 视频（8-16 帧）。设计 token 预算。你将从不做重度多图像中节省的额外预算放在哪里？

2. 阅读 LLaVA-OneVision 第 4.3 节（涌现能力）。提出课程可能解锁但论文未报告的第四种涌现技能。

3. 调换课程顺序——先训练多图像，然后单图像，然后视频。预测哪些基准会下降及其原因。

4. 论文报告仅用每样本 8 帧训练的视频基准。这能否泛化到推理时的 30 秒视频？什么先出问题——token 预算还是时序推理？

5. 将 24x24 的块双线性池化到 12x12，每个维度减少 4 倍。在标准库 Python 中实现池化，并验证每个 2x2 块的平均值是否匹配双线性输出。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------------|------------------------|
| OneVision 场景 | 「单图像、多图像或视频」 | 统一 VLM 处理的三种输入形状之一；预算在它们之间保持恒定 |
| Token 预算 | 「每样本多少 token」 | LLM 在每个训练/推理样本中看到的总视觉 token，通常 3000-4000 |
| 课程（Curriculum） | 「训练顺序」 | 为涌现迁移选择的阶段排序（单图像 → 多图像 → 视频） |
| 双线性池化（Bilinear pooling） | 「token 缩小」 | 对块网格（2D）应用双线性插值以减少 token 数的同时保持局部性 |
| 涌现技能（Emergent skill） | 「未训练，仍然有效」 | 在推理时出现但没有匹配训练数据的能力，由于课程组合 |
| AnyRes-k | 「k 瓦片设置」 | k 个固定分辨率子瓦片加上一个缩略图，典型 k ∈ {4, 9} |
| 任务迁移 | 「跨场景泛化」 | 在单图像上学到的技能通过共享骨干应用于视频（反之亦然） |
