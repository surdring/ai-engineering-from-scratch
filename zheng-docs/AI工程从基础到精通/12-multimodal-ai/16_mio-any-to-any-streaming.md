# MIO 与任意模态到任意模态的流式多模态模型

> GPT-4o 交付了一个大多数开源模型无法复制的产品：一个能够实时听语音、看视频并回话的智能体。开源生态在2024年底的答案是 MIO（Wang et al.，2024年9月）。MIO 将文本、图像、语音和音乐分词化，在交错序列上训练一个因果 Transformer，并在任意模态之间进行生成。AnyGPT（Zhan et al.，2024年2月）是概念验证；MIO 是规模化版本；Unified-IO 2（Allen AI，2023年12月）是结合了视觉和动作定位的相关工作。本课将解读任意模态到任意模态的模式——四种分词器、一个 Transformer、支持流式解码。

**类型：** 学习
**语言：** Python（标准库，四种模态 Token 分配器 + 流式解码循环）
**前置要求：** 第12阶段 · 11（Chameleon），第6阶段（语音与音频）
**时间：** ~120分钟

## 学习目标

- 设计一个共享词汇表，容纳文本、图像、语音和音乐 Token 而不产生冲突。
- 比较 SEED-Tokenizer（图像）和 SpeechTokenizer 残差 VQ（Residual-VQ）（语音）在压缩与重建权衡上的差异。
- 解释构建任意模态到任意模态生成能力的四阶段课程训练。
- 列举三种开源任意模态到任意模态方案及其主要权衡：MIO、AnyGPT、Unified-IO 2。

## 问题

统一多模态模型说起来容易，大规模构建却很难。截至2024年，大多数"任意模态到任意模态"系统都是流水线式的：视觉模型 → 文本表示 → 语音模型 → 音频。每一步都会丢失信息、增加延迟，并使训练复杂化。GPT-4o 的演示视频展示了一种亚秒级响应的单模型替代方案；开源系统落后了数月。

工程挑战包括：

- 必须为每种模态配备分词器，以足够无损的方式进行压缩以便重建，并以 Transformer 可以消费的速率产生 Token。
- 单一词汇表必须为文本（32k+）、图像（16k+）、语音（4k+）、音乐（8k+）分配空间。至少需要四万多个条目。
- 训练数据必须覆盖每一种输入-输出对（文本→图像、图像→语音、语音→图像等），否则模型必须能够组合。
- 推理必须足够快地流式输出 Token，以满足对话延迟要求（首字节音频时间 <500ms）。

## 核心概念

### 四种模态的四种分词器

MIO 的分词器栈：

- 文本：标准 BPE，词汇量约 32000。
- 图像：SEED-Tokenizer（2023）——带离散码本的量化 VAE，4096 个条目，每张图像 32x32 个 Token。
- 语音：SpeechTokenizer 残差 VQ（2023）——将 16kHz 波形编码为 8 个层级码本；第一层是粗略内容，后续层级添加韵律和说话人身份信息。
- 音乐：类似的残差 VQ（Meta 的 MusicGen / Encodec 系列），4-8 个码本。

每种模态产生整数 Token。Token 在共享词汇表中获得不相交的 ID 范围：

```
text:   0..31999
image:  32000..36095  (4096 个图像 Token)
speech: 36096..40191  (4096 个语音基础 Token，加上残差层)
music:  40192..48383  (8192 个音乐 Token)
sep:    48384..48390  (<image>, <speech>, <music>, </...>, 等)
```

总计：约 48k 词汇量。输入嵌入和输出投影跨越整个词汇表。

### 流式解码

语音生成使用残差 VQ。Transformer 预测基础（第0层）语音 Token；并行解码的残差量化器预测后续层级。每个第0层 Token 在 16kHz 下大约对应 50ms 的音频。

流式模式：

1. 用户对着麦克风说话；实时音频分词器每 50ms 发出语音 Token。
2. MIO 随着 Token 到达消费它们（提示预填充 + 增量前向传播）。
3. 输出 Token 流式生成；并行语音解码器将它们转换为音频样本，延迟约 50-150ms。
4. 首字节音频时间：MIO 论文中约 300-500ms，接近 GPT-4o 的约 250ms。

Mini-Omni（arXiv:2408.16725）、GLM-4-Voice（arXiv:2412.02612）和 Moshi（arXiv:2410.00037）是互补的流式语音-LLM 设计。Moshi 尤其在单 GPU 上实现了 160ms 的往返延迟。

### 四阶段课程训练

MIO 的训练课程：

1. 第一阶段——对齐。大规模模态对语料库：文本-图像、文本-语音、文本-音乐。每对使用各自的 Token 词汇段。训练共享词汇表。
2. 第二阶段——交错。多模态交错文档（包含图片+视频的博客、带文字记录的播客等）。训练跨模态上下文。
3. 第三阶段——语音增强。额外的音频数据，在保持文本能力的同时提升语音质量。
4. 第四阶段——SFT。跨模态指令微调：VQA、描述生成、旁白配音、语音到语音对话。

跳过某个阶段会降低特定能力：跳过第二阶段，模型失去跨模态上下文；跳过第三阶段，语音质量会很差。

### 视觉思维链

MIO 引入了视觉思维链（Chain-of-Visual-Thought）：模型生成中间图像 Token 作为推理步骤。对于"猫在爬树吗？"这个问题，模型：

1. 发出 `<image>` Token 渲染场景（来自输入图像或一个草图）。
2. 发出文本分析该草图。
3. 发出最终答案。

渲染的中间图像充当草稿纸。在空间推理任务上基准测试有所改善。这一思想与文本推理中的思维链（Chain-of-Thought）相似。

### 任意模态到任意模态的竞争者

- AnyGPT（arXiv:2402.12226）：4 种模态（文本、图像、语音、音乐），设计类似。
- Unified-IO 2（arXiv:2312.17172）：添加视觉动作输出、深度、法线。任务多样性更高，规模较小。
- NExT-GPT（arXiv:2309.05519）：LLM + 模态特定扩散解码器。不是单模型方案。
- CoDi（arXiv:2305.11846）：可组合扩散；通过共享潜在空间实现任意模态到任意模态。

MIO 最接近纯 Token 的任意模态到任意模态。AnyGPT 是其概念上的前身。

### 延迟预算

对于对话产品，每个组件的延迟都很重要：

- 麦克风到音频 Token：约 50ms。
- 预填充（音频 Token + 历史记录）：8B 模型上约 100ms。
- 首个输出 Token：约 50ms。
- 并行残差 VQ + 语音解码器：约 100-150ms。

首字节音频总时间：最低约 300ms。GPT-4o 声称约 250ms。Moshi 声称 160ms。MIO/AnyGPT 根据公开基准测试在 400-600ms 范围内。

### 为什么任意模态到任意模态仍然很难

即使在 2026 年，开源任意模态到任意模态模型在两个轴向上仍落后于闭源模型：

- 语音质量。残差 VQ 分词器是有损的；对话语音与 ElevenLabs 级别的语音相比听起来像机器人。
- 跨模态推理。要求模型"唱出你看到的内容"仍然比纯视觉任务更容易失败。

这些是开放的研究问题。Qwen3-Omni（第12.20课）是 2025 年最先进的开源尝试。

## 练习

1. 你的产品接受语音输入并返回语音输出。端到端延迟预算目标是多少？列出花费时间的各个组件。

2. SpeechTokenizer 残差 VQ 使用 8 个码本。提出为什么并行解码残差层级是必要的（而非顺序解码），以及它带来的延迟节省。

3. 你的词汇表有 32k 文本 + 4k 图像 + 4k 语音。添加 8k 音乐和约 10 个分隔符。在隐藏维度 4096 下，嵌入矩阵的参数成本是多少？

4. 视觉思维链会生成一张中间图像。哪类问题会受益？哪类问题会因额外的 Token 而受损？

5. 阅读 Moshi（arXiv:2410.00037）。描述其"内心独白"（inner monologue）技术，并与 MIO 的视觉思维链进行比较。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------|---------|
| 任意模态到任意模态 | "多模态输入/输出" | 单个模型接受并发出文本、图像、语音和音乐，方向任意 |
| 残差 VQ | "语音分词器栈" | 多码本分词，每层添加信息；基础层是内容，后续层是韵律 |
| SEED-Tokenizer | "图像编码" | MIO 使用的 4096 条目码本的离散图像分词器 |
| 视觉思维链 | "视觉草稿纸" | 模型在给出最终答案之前生成一张中间图像作为推理步骤 |
| 首字节音频时间 | "TTFAB" | 从用户语音到首个音频输出的延迟；对话体验要求 <500ms |
| 四阶段课程训练 | "训练配方" | 对齐 -> 交错 -> 语音增强 -> SFT，按此顺序进行 |
