# 线性方程组

> 解 Ax = b 是数学中最古老的问题，现在仍在运行你的神经网络。

**类型：** 构建
**使用语言：** Python
**前置课程：** 阶段 1，第 01-03 课
**预计时间：** ~120 分钟

## 学习目标

- 使用带部分主元的高斯消元和回代解 Ax = b
- 进行 LU、QR 和 Cholesky 分解，解释何时适用每种方法
- 推导最小二乘的正规方程并将其联系到线性和岭回归
- 使用条件数诊断病态系统并应用正则化使其稳定

## 问题

每次训练线性回归，你在解一个线性系统。每次计算最小二乘拟合，你在解一个线性系统。每次神经网络层计算 `y = Wx + b`，它在评估线性系统的一侧。当你添加正则化时，你在修改系统。

方程 Ax = b 无处不在。在线性回归中，A 是数据矩阵，b 是目标向量，x 是权重向量。整个模型归结为：找到 x 使 Ax 尽可能接近 b。

## 概念

### Ax = b 的几何意义

每个方程定义一个超平面。解是所有超平面相交的点。三种情况：唯一解（A 可逆）、无解（不一致）、无穷多解（A 有零空间）。大多数 ML 问题属于「无精确解」类别，因为方程（数据点）多于未知数（参数）——这就是最小二乘发挥作用的地方。

### 列图 vs 行图

行图：A 的每行定义一个方程，作为超平面。列图：A 的每列是一个向量，问题变成 A 的列的哪种线性组合能产生 b。列图更基本——如果 b 在 A 的列空间中，系统有解；如果不在，你找列空间中最近的点，那就是最小二乘解。

### 高斯消元法

将 Ax = b 变换为上三角系统 Ux = c，然后通过回代求解。最直接的方法。部分主元：在每一步选择最大可用主元以最小化误差放大——没有它，消元可能失败或产生垃圾结果。O(n^3) 复杂度。

### LU 分解

将高斯消元中的乘数存储在下三角矩阵 L 中。A = LU。好处：分解一次（O(n^3)），对任何新 b 求解仅 O(n^2)。如果需要对相同 A 解 1000 个不同 b 的系统，LU 可大幅节省工作。

### QR 分解

A = QR，Q 是正交矩阵（Q^T Q = I），R 是上三角矩阵。对于最小二乘问题，QR 在数值上比 LU 更稳定。通过 Gram-Schmidt 构建：取第一列归一化，对后续列减去在之前所有 q 向量上的投影，归一化。求解：Rx = Q^T b。

### Cholesky 分解

对于对称正定矩阵 A（协方差矩阵、正规方程中的 A^T A）：A = L L^T。仅对下三角部分操作，成本为 LU 的一半。如果矩阵保证正定，这是绝佳选择。

### 条件数与病态

条件数衡量输出对输入扰动的敏感度。条件数高 = 病态——输入中的小变化（测量误差、舍入误差）导致输出中的大变化。对病态系统，答案是噪声。通过添加正则化（Ridge/L2）来稳定它。

## 关键术语

| 术语 | 人们常说的 | 实际含义 |
|------|-----------|---------|
| 高斯消元 | "行化简" | 通过行操作将系统简化为上三角形式 |
| LU 分解 | "消元因数" | 将 A 分解为下三角 L 和上三角 U |
| QR 分解 | "正交三角化" | 将 A 分解为正交 Q 和上三角 R |
| Cholesky | "对称 LU" | 对称正定矩阵的 L L^T 分解 |
| 条件数 | "稳定性分数" | 输出变化/输入变化比——高则病态 |
