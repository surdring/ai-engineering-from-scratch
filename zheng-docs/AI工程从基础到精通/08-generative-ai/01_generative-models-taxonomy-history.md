# 生成模型——分类与历史

> 每个图像模型、文本模型、视频模型和 3D 模型都适合五个桶之一。选错了桶你会与数学战斗数周。选对了，该领域过去十二年的进展会清晰地堆叠在你脑海中。

**类型：** 学习
**语言：** Python
**前置要求：** 第 2 阶段（ML 基础），第 3 阶段（深度学习核心），第 7 阶段 · 14（Transformer）
**时间：** 约 45 分钟

## 问题

生成模型做一件事：给定从某个未知分布 `p_data(x)` 抽取的训练样本，输出看起来来自同一分布的新样本。人脸、句子、MIDI 文件、蛋白质结构——如果你眯着眼睛看，都是同一问题。

难点在于 `p_data` 存在于数百万维的空间中（一张 512×512 RGB 图像约 786k 维），样本位于该空间内的薄流形上，而你只有可能 10M 个示例。暴力破解密度是无望的。每个生成模型都是一个妥协，将一个困难问题换成稍微不那么困难的问题。

五个家族在过去十二年中幸存下来。知道每个家族做了什么样的妥协，就能告诉你为什么它在某些任务上获胜而在其他任务上崩溃。

## 概念

![生成模型的五个家族——按建模内容分类](../assets/taxonomy.svg)

**1. 显式密度，可计算。** 将 `log p(x)` 写为你可以实际求值的和。自回归模型（PixelCNN、WaveNet、GPT）因式分解 `p(x) = ∏ p(x_i | x_<i)`。归一化流（RealNVP、Glow）将 `p(x)` 构建为简单基的可逆变换。优点：精确似然，清晰的训练损失。缺点：自回归推理是串行的（长序列慢），流需要可逆架构（架构限制）。

**2. 显式密度，近似。** 从下方限制 `log p(x)`（ELBO）并优化该界限。VAE（Kingma 2013）使用带变分后验的编码器-解码器。扩散模型（DDPM，Ho 2020）训练一个隐式优化加权 ELBO 的去噪器。扩散是 2026 年主导的图像、视频和 3D 骨干。

**3. 隐式密度。** 完全跳过密度；学习一个产生样本的生成器 `G(z)` 和一个区分真假的分辨器 `D(x)`。GAN（Goodfellow 2014）。推理快（一次前向传播）但训练时臭名昭著地不稳定。StyleGAN 1/2/3 即使在 2026 年仍然是固定领域照片级真实感（人脸、卧室）的最先进。

**4. 基于分数 / 连续时间。** 直接学习对数密度的梯度 `∇_x log p(x)`（分数）。Song & Ermon（2019）展示了分数匹配将扩散推广到 SDE。流匹配（Lipman 2023）是 2024-2026 的热点：无模拟训练，更直的路径，比 DDPM 快 4-10 倍的采样。Stable Diffusion 3、Flux、AudioCraft 2 都使用流匹配。

**5. 基于标记的自回归离散编码。** 用 VQ-VAE 或残差量化器将高维数据压缩为短序列离散标记，然后使用 Transformer 建模标记序列。Parti、MuseNet、AudioLM、VALL-E、Sora 的补丁分词器都使用这个。这是桶 1 加上学习的 tokenizer。

## 五问题分流

当新的生成模型论文发布时，在阅读方法部分之前回答这五个问题。

1. **建模的是什么？** 像素、潜在、离散标记、3D 高斯、网格、波形？
2. **密度是显式的还是隐式的？** 他们是否写下 `log p(x)`？
3. **采样：一次还是迭代？** 迭代意味着更慢的推理；一次通常意味着对抗或蒸馏。
4. **条件：无条件、类别、文本、图像、姿态？** 这决定损失和架构脚手架。
5. **评估：FID、CLIP 分数、IS、人类偏好、任务准确率？** 每个都有已知的失败模式（见第 14 课）。

你将在本阶段的每一课中重新回答这五个问题。到最后，它们将成为本能。

## 构建

本课的代码是一个轻量级可视化：用三种玩具方法（核密度、离散直方图和最近样本的"GAN 式"生成器）从样本中拟合 1-D 混合高斯，这样你可以在一屏可打印的问题上看到显式 vs 隐式密度的区别。

运行 `code/main.py`。它从双峰高斯混合中抽取 2000 个样本，然后打印：

```
显式密度（直方图）：p(x in [-0.5, 0.5]) ≈ 0.38
近似密度（KDE）：    p(x in [-0.5, 0.5]) ≈ 0.41
隐式（最近样本生成器）：打印 20 个新样本，无 p(x)
```

注意：前两个让你问"这个点的可能性有多大？"第三个不能。这是*显式 vs 隐式*的区别，对每一课都很重要。

## 使用

2026 年哪个家族用于哪个任务？

| 任务 | 最佳家族 | 原因 |
|------|-------------|-----|
| 照片级人脸，窄域 | StyleGAN 2/3 | 仍然最清晰，最快推理。 |
| 通用文本到图像 | 潜在扩散 + 流匹配 | SD3, Flux.1, DALL-E 3。 |
| 快速文本到图像 | 整流流 + 蒸馏 | SDXL-Turbo, SD3-Turbo, LCM。 |
| 文本到视频 | 扩散 Transformer + 流匹配 | Sora, Veo 2, Kling。 |
| 语音 + 音乐 | 基于标记的 AR（AudioLM, VALL-E, MusicGen）或流匹配（AudioCraft 2） | 离散标记便宜地缩放。 |
| 3D 场景 | 高斯溅射拟合，扩散先验 | 3D-GS 用于重建，扩散用于新视角。 |
| 密度估计（无采样） | 流 | 唯一具有精确 `log p(x)` 的家族。 |
| 模拟 / 物理 | 流匹配，分数 SDE | 直线路径，平滑向量场。 |

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------------|-----------------------|
| 生成模型 | "它创造新东西" | 学习 `p_data(x)` 的采样器，可选暴露 `log p(x)`。 |
| 显式密度 | "你可以求值它" | 模型提供闭式或可计算的 `log p(x)`。 |
| 隐式密度 | "GAN 风格" | 只有采样器——无法求值给定点的 `p(x)`。 |
| ELBO | "证据下界" | `log p(x)` 的可计算下界；VAE 和扩散优化它。 |
| 分数 | "对数密度梯度" | `∇_x log p(x)`；扩散和 SDE 模型学习这个场。 |
| 流形假设 | "数据生活在表面上" | 高维数据集中低维流形；为什么降维有效。 |
| 自回归 | "预测下一个片段" | 将联合分布因式分解为条件概率的乘积。 |
| 潜在 | "压缩编码" | 解码器可以重建输入的低维表示。 |

## 生产说明：五个家族，五种推理形态

每个家族映射到不同的推理服务器成本曲线。生产推理文献将 LLM 推理框架为预填充 + 解码；相同的分解在这里适用：

- **自回归（桶 1 和 5）。** 序列解码主导延迟；KV 缓存、连续批处理和推测解码全部直接适用。
- **VAE / 扩散 / 流匹配（桶 2 和 4）。** 在 LLM 意义上没有解码。成本 = `num_steps × step_cost`，而 `step_cost` 是完整潜在分辨率上的 transformer 或 U-Net 前向传播。生产旋钮是步数（DDIM / DPM-Solver / 蒸馏）、批大小和精度（bf16 / fp8 / int4）。
- **GAN（桶 3）。** 一次前向传播。无调度，无 KV 缓存。TTFT ≈ 总延迟。这就是为什么 StyleGAN 仍在窄域 UX 上获胜。

当你在论文摘要中看到"比扩散更快"时，翻译为"更少的步数 × 相同的步成本"或"相同的步数 × 更便宜的步成本"。其他一切都是营销。
