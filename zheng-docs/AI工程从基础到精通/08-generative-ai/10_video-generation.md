# 视频生成

> 图像是 2-D 张量。视频是 3-D 的。理论相同；计算困难 10-100 倍。OpenAI 的 Sora（2024 年 2 月）证明了这是可能的。到 2026 年 Veo 2、Kling 1.5、Runway Gen-3、Pika 2.0 和 WAN 2.2 以 1080p 从文本交付生产视频——而开放权重技术栈（CogVideoX、HunyuanVideo、Mochi-1、WAN 2.2）落后 12 个月。

**类型：** 构建
**语言：** Python
**前置要求：** 第 8 阶段 · 07（潜在扩散），第 7 阶段 · 09（ViT），第 8 阶段 · 06（DDPM）
**时间：** 约 45 分钟

## 问题

一个 10 秒 1080p 视频以 24fps 是 240 帧 1920×1080×3 像素。每段约 1.5 GB 原始数据。像素空间扩散不可行。你需要：

1. **时空压缩。** 一个 VAE，将视频编码为时空补丁序列，而不是帧。
2. **时间连贯性。** 帧需要共享内容、光照和物体身份跨越数秒。网络必须建模运动。
3. **计算预算。** 视频训练比相同模型大小的图像贵 10-100 倍。
4. **条件。** 文本、图像（首帧）、音频或另一个视频。大多数生产模型接受全部四种。

解决这个问题的架构是应用于时空补丁的**扩散 Transformer（DiT）**，在巨大（提示、字幕、视频）数据集上训练。与第 06 课相同的扩散损失。

## 概念

![视频扩散：分块、DiT、解码](../assets/video-generation.svg)

### 分块

用 3D VAE（学习的时空压缩）编码视频。潜在形状为 `[T_latent, H_latent, W_latent, C_latent]`。分割为大小为 `[t_p, h_p, w_p]` 的补丁。对于 Sora 风格模型，`t_p = 1`（每帧补丁）或 `t_p = 2`（每两帧）。一个 10 秒 1080p 视频压缩为约 20,000-100,000 个补丁。

### 时空 DiT

一个 transformer 处理补丁的扁平序列。每个补丁有 3D 位置嵌入（时间 + y + x）。注意力通常是因子化的：

- **空间注意力** 在每个帧的补丁内。
- **时间注意力** 跨帧在相同空间位置。
- **完整 3D 注意力** 贵 16-100 倍；仅在低分辨率或研究中用于。

### 文本条件

与大型文本编码器的交叉注意力（Sora 的 T5-XXL，CogVideoX-5B 使用 T5-XXL）。长提示很重要——Sora 的训练集有 GPT 生成的密集重字幕，平均每个片段 200 个标记。

### 训练

在时空潜在上的标准扩散损失（ε 或 v 预测）。数据：网络视频 + 约 100M 精选片段 + 合成文本字幕。计算：即使是一个小的研究运行也要 10,000+ GPU 小时；Sora 规模是 100,000+。

## 构建

`code/main.py` 模拟核心时空 DiT 想法：分块小型合成视频，添加每补丁位置嵌入，并将整个序列与补丁上的 transformer 式注意力一起去噪。无 numpy；纯 Python。我们表明即使在 1-D 中，当相邻帧补丁共享去噪器和位置嵌入时，时间连贯性也会出现。

### 步骤 1：分块合成 1-D "视频"

```python
def make_video(T_frames=8, rng=None):
    # "视频"是遵循平滑轨迹的 1-D 值序列
    base = rng.gauss(0, 1)
    return [base + 0.3 * t + rng.gauss(0, 0.1) for t in range(T_frames)]
```

### 步骤 2：每帧位置嵌入

```python
def pos_embed(t, dim):
    return sinusoidal(t, dim)
```

### 步骤 3：去噪器看整个序列

而不是独立去噪每个帧，我们的微型网络拼接所有帧值 + 它们的位置嵌入，并联合预测所有帧的噪声。

### 步骤 4：时间连贯性测试

训练后，采样视频。测量帧到帧的增量。如果模型学会了时间结构，增量保持比独立采样每帧更小。

## 陷阱

- **独立每帧采样 = 闪烁。** 如果你在每帧上分别运行图像扩散，输出会闪烁因为每帧的噪声是独立的。视频扩散通过注意力或共享噪声耦合帧来修复。
- **朴素的 3D 注意力 = OOM。** 在 10 秒 1080p 潜在上的全 3D 注意力是数千亿次操作。因子化为空间 + 时间。
- **数据字幕比大小更重要。** Sora 相对于先前工作的主要升级是在约 10 倍更详细的字幕（GPT-4 重标记片段）上训练。OpenAI 的技术报告明确说明了这一点。
- **首帧条件。** 大多数生产模型也接受图像作为首帧。这是"图像到视频"模式；训练包括此变体。
- **物理漂移。** 长片段（>10 秒）积累微妙的矛盾。滑动窗口生成 + 关键帧锚定有帮助。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------------|-----------------------|
| 视频 VAE | "3-D VAE" | 将 `(T, H, W, C)` 压缩为时空潜在的编码器。 |
| 补丁 | "标记" | 潜在中的固定大小 3-D 块；DiT 的输入。 |
| 因子化注意力 | "空间 + 时间" | 在空间上运行注意力，然后在时间上运行；跳过完整 3-D 注意力。 |
| 图像到视频（I2V） | "让此照片动起来" | 模型接受图像 + 文本，输出从该图像开始的视频。 |
| 关键帧条件 | "锚定帧" | 固定特定帧以控制视频的弧线。 |
| 运动画笔 | "方向提示" | UI 输入，用户将运动向量画到图像上。 |
| 重字幕 | "密集字幕" | 使用 LLM 用详细提示重新标记训练片段。 |
| 闪烁 | "时间伪影" | 帧到帧不一致；通过耦合去噪修复。 |

## 生产说明：视频潜在是内存带宽问题

一个 10 秒 1080p 片段以 24 fps 是 240 帧 × 1920 × 1080 × 3 ≈ 1.5 GB 原始像素。在 4× 视频 VAE 压缩（`2 × 空间 × 2 × 时间`）后，潜在每个请求约 100 MB。通过时空 DiT 以批次 1 运行 30 步，你每步通过 HBM 移动约 3 GB——内存带宽，而不是 FLOPs，是瓶颈。

三个生产旋钮，都直接来自生产推理文献的推理章节：

- **DiT 上的 TP。** 文本到视频模型常规 ≥10B 参数。4 个 H100 上的 TP=4 是标准；TP=2 × PP=2 用于 405B 类模型。每步延迟随 TP 大致线性下降，直到全归约墙。
- **帧批处理 = 连续批处理。** 在生成时，视频在概念上是由注意力链接的一批帧。连续批处理（飞行调度）适用：在帧 `t-1` 被返回的同时开始渲染帧 `t+1`，如果模型架构允许滑动窗口生成。
- **片段级预填充缓存。** 对于图像到视频，首帧条件类似于 LLM 的提示预填充：计算它一次，跨时间解码器传播重用它。这实际上是视频的 KV 缓存。
