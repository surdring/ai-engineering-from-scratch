# 大语言模型的差分隐私

> DP-SGD（差分隐私随机梯度下降）仍是标准方法 —— 注入噪声的梯度更新提供形式的 (ε, δ) 保证。计算、内存和效用的开销显著；参数高效的差分隐私微调（LoRA + DP-SGD）是 2025 年常见配置（ACM 2025）。两组证据存在张力：基于金丝雀的成员推理攻击（Membership Inference Attack, MIA）（Duan et al., 2024）报告对语言模型的攻击成功有限；训练数据提取（Carlini et al., 2021; Nasr et al., 2025）恢复了大量逐字记忆内容。解决方案（arXiv:2503.06808，2025 年 3 月）：差距在于度量内容的不同 —— 插入的金丝雀 vs "最可提取"的数据。新的金丝雀设计无需影子模型即可进行基于损失的 MIA，并在真实数据上给出了首个具有现实差分隐私保证的非平凡 DP 审计。替代方案：PMixED（arXiv:2403.15638）—— 通过专家混合在推理时对下一个 token 分布进行私有预测；DP 合成数据生成（Google Research 2024）。新兴攻击：通过 LLM 反馈的差分隐私反转（Differential Privacy Reversal）—— 置信度分数泄露。

## 问题

LLM 会记忆训练数据。Carlini et al. 2021 展示了生产级语言模型能够按需复现训练数据的逐字文本。差分隐私是形式的防御手段：训练模型使得输出在可证明意义上对任何单个训练样本不敏感。2024-2025 年的证据表明 DP-SGD 是必要的，但部署中使用的 ε 值可能与威胁模型不匹配。

## 概念

### (ε, δ)-差分隐私

若对于任何仅相差一个样本的数据集对和任意事件 S，随机化算法 M 满足：
P(M(D) in S) <= e^ε * P(M(D') in S) + δ，
则 M 是 (ε, δ)-DP 的。

解释：输出分布足够接近（由 ε 参数化），使得任何单一个体的贡献除概率 δ 外无法被可靠推断。

### DP-SGD

Abadi et al. 2016。标准配方：
1. 采样一个小批量（mini-batch）。
2. 计算逐样本梯度。
3. 将每个逐样本梯度裁剪到阈值 C。
4. 对裁剪后的梯度求和，添加标准差为 σ * C 的高斯噪声。
5. 使用带噪声的和更新参数。

隐私代价由会计器（Moments Accountant、Rényi DP 会计器）追踪。LLM 文献中报告的 ε 值因威胁模型、数据敏感度和效用目标而差异很大；没有普适的"安全"默认 ε。已发表的示例在部分 LLM 训练设置中跨度约为 ε ≈ 1–10，但这些仅为示意性值，并非推荐默认值。较低的 ε 通常需要更多噪声，可能增加效用损失。

### LoRA + DP-SGD

对前沿模型进行完整的 DP-SGD 成本过高。LoRA（Hu et al. 2022）将梯度更新限制在小型适配器上，减少了逐样本梯度存储。LoRA + DP-SGD 是 2025 年常见配置。DP 保证适用于适配器；基础模型保持固定。

### 2024-2025 年的张力

两方面的证据：

- **基于金丝雀的 MIA（Duan et al. 2024）。** 在训练数据中插入独特的金丝雀，度量成员推理攻击者能否识别它们。报告对语言模型的攻击成功有限。暗示 MIA 很困难。
- **训练数据提取（Carlini 2021, Nasr et al. 2025）。** 用前缀提示模型；度量其能否恢复训练数据中的逐字文本。报告大量记忆内容。暗示在相关意义上 MIA 很容易。

2025 年 3 月解决方案（arXiv:2503.06808）：两者度量不同内容。MIA 询问"样本 e 是否在 D 中？"，针对插入的金丝雀。而提取询问"我能恢复 D 中哪些内容？""最可提取"的样本才是隐私相关的；金丝雀低估了这一点，因为金丝雀并未被优化为可提取的。

新的金丝雀设计。无需影子模型的基于损失的 MIA。在真实数据上对 LLM 进行具有现实 DP 保证的首个非平凡 DP 审计。

### DP 训练的替代方案

- **PMixED（arXiv:2403.15638）。** 推理时的私有预测。在下一个 token 分布上进行专家混合（Mixture of Experts）；每个专家看到训练数据的一个分片；聚合时添加噪声以实现 DP。完全避免了 DP 训练。
- **DP 合成数据生成（Google Research 2024）。** 使用 DP-SGD 进行 LoRA 微调，采样合成数据，在合成数据上训练下游分类器。

两种方案都规避了完整 DP 训练的效用代价，但代价是不同的威胁模型。

### 通过 LLM 反馈的差分隐私反转

2025 年新兴攻击。使用 DP 训练模型的置信度分数作为预言机（oracle）来重新识别个体。即使输出不泄露，置信度分布也可能泄露。

防御手段：不暴露置信度，或在暴露前截断/量化置信度。这是 (ε, δ)-DP 训练之外的额外要求。

## 关键术语

| 术语 | 人们常说的 | 实际含义 |
|------|-----------|---------|
| DP | "(ε, δ)-差分隐私" | 形式隐私：在相邻数据集变化下输出分布接近 |
| DP-SGD | "注入噪声的 SGD" | 梯度裁剪 + 高斯噪声添加；标准 DP 训练 |
| LoRA + DP-SGD | "高效私有微调" | 在低秩适配器上进行 DP-SGD；2025 年标准配置 |
| MIA | "成员推理" | 判断某个样本是否在训练数据中的攻击 |
| 金丝雀（Canary） | "插入的水印样本" | 用于度量 DP 泄露的独特训练样本 |
| PMixED | "私有推理混合" | 通过专家混合在推理时对下一个 token 分布实现 DP |
| DP 反转（DP Reversal） | "置信度泄露攻击" | 使用模型置信度作为重新识别预言机的攻击 |
