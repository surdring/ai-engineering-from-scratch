# 缩放法则

> 2020 年 Kaplan 论文说：更大的模型，更低的损失。2022 年 Hoffmann 论文说：你训练不足了。计算进入两个桶——参数和标记——而分割并不明显。

**类型：** 学习
**语言：** Python
**前置要求：** 第 7 阶段 · 05（完整 Transformer），第 7 阶段 · 07（GPT）
**时间：** 约 45 分钟

## 问题

当你有 C FLOPs 的训练计算并想要最佳模型时，你面对两个旋钮：

1. **多少参数（N）？** 更大的模型，更高的容量。
2. **多少训练标记（D）？** 更多数据，更好地利用容量。

FLOPs 近似缩放为 `6 × N × D`。你可以推高 N 降低 D，或推高 D 降低 N。哪个更好？

2022 年之前，答案是"大力推 N"。GPT-3（2020）是 175B 参数在约 300B 标记上训练。约 1.7 标记每参数的比率。Kaplan 缩放法则支持这一点。

Hoffmann et al.（2022），训练一个叫 Chinchilla 的小系列模型，发现了不同的东西：最佳比率更接近**20 标记每参数**。GPT-3 训练不足了 10 倍。Chinchilla（70B 参数，1.4T 标记）在每个基准上击败 GPT-3（175B，300B 标记），推理成本低 2.5 倍。

2026 是 Chinchilla 的世界——有一个重要转折。Llama 3 8B 在 15 万亿标记上训练，比率为 1,875 标记每参数。比 Chinchilla 最佳高出 94 倍。对于将大规模使用的模型，推理成本比训练成本更重要，因此对更小的可部署占用进行过度训练（超越 Chinchilla）是 2026 的默认。

## 概念

![Chinchilla 曲线：不同 N/D 比率下损失与计算的关系](../assets/scaling-laws.svg)

### Hoffmann 法则

来自 Chinchilla 论文，损失遵循：

```
L(N, D) = A / N^α + B / D^β + E
```

- `N` = 参数（非嵌入）。
- `D` = 训练标记。
- `α ≈ 0.34`，`β ≈ 0.28`（大致对称）。
- `E ≈ 1.69`，不可约损失下限。
- `A ≈ 406`，`B ≈ 411`。

两项在你扩展时相互权衡。在固定计算（C = 6ND）下对 `N` 求导并求解：

```
N_opt ≈ 0.6 × (C/6)^0.5
D_opt ≈ 0.6 × (C/6)^0.5
D_opt / N_opt ≈ 20
```

计算最优：20 标记每参数。

### 为什么仍然过度训练

Chinchilla 最优在每训练 FLOP 上最小化训练损失。但你只付一次训练成本；推理成本永远付。

对于每月服务一万亿标记的聊天机器人，推理主导总成本。Llama 的方法：训练更小，更长。8B 在 15T 标记上是深度推理优化的：

- 适合消费级 GPU。
- 延迟是 70B Chinchilla 最优的一小部分。
- 质量对大多数任务足够接近。

DeepMind 2024 年的论文（"过度训练是新的最优"）形式化了这一点。对于推理主导的工作负载，正确的比率更接近每参数 100-500 标记，取决于服务量。

### 涌现 vs 平滑

声称：某些能力（算术、多步推理、思维链跟随）在某个规模"涌现"。

Schaeffer et al.（2023）论证这是测量伪影：涌现指标使用不连续评分（精确匹配、阈值准确率），隐藏了底层 logits 的平滑改进。连续指标（交叉熵）显示平滑曲线。

2026 年共识是：通过连续损失的预测是可靠的。基准跳跃通常是评分器伪影。针对连续指标规划预算。

### 2026 图景

缩放法则仍然有效，但是：

| 因素 | 如何改变 |
|--------|-------------|
| 数据质量 | 策展"好"标记（Phi 风格）通过 >2× 有效计算移动曲线 |
| MoE | 总参数与活跃 FLOPs 解耦；按活跃 FLOP 的缩放法则 |
| 后训练 | 某些能力（指令跟随、代码）通过 SFT+RLHF 的变化比预训练更大 |
| 多模态 | 图像 + 文本标记一起缩放；每种模态单独曲线 |
| 合成数据 | 模型生成训练数据；有效计算可以复合 |

Muon 优化器（Kimi Moonlight，2024）展示了在匹配数据下相对于 AdamW 的约 2× 有效计算增益。一些 2026 训练运行默认使用 Muon。改变缩放法则中的绝对常数，而非其形状。

## 构建

见 `code/main.py`。我们实现 Chinchilla 损失方程，并求解每个计算预算下的计算最优 `(N, D)`。

### 步骤 1：Chinchilla 损失

```python
def chinchilla_loss(N, D, A=406.4, B=410.7, alpha=0.34, beta=0.28, E=1.69):
    return A / N ** alpha + B / D ** beta + E
```

在固定 `C = 6ND` 下绘制 `L` 作为 `(N, D)` 的等高线。找到最小值。

### 步骤 2：计算最优前沿

对于从 `1e17` 到 `1e25` FLOPs 的计算预算，找到在 `6ND = C` 约束下最小化损失的 `(N, D)`。验证比率 `D/N ≈ 20`。

### 步骤 3：过度训练成本

计算训练一个 10× 更小的模型（最优 N 的 1/10，最优 D 的 10 倍）所支付的额外损失。报告作为交换的推理 FLOP 节省（与 N 成正比）。

### 步骤 4：与真实模型比较

代入 GPT-3、Chinchilla、Llama 3 8B、DeepSeek-V3（活跃参数）的已知 `(N, D)` 对，比较预测损失与报告损失。

## 使用

你自己不太可能训练前沿模型。但缩放法则告诉你：

1. **你的微调是否有足够数据。** 如果你的任务特定数据低于基础模型每参数 20 标记，预期在某个损失下限处饱和。
2. **是否选择更大的基础模型。** 如果你将所有预算花在推理上，偏好更小、训练更长的模型。
3. **回报在哪里递减。** 超过 Chinchilla 最优的 1000 倍后，对数损失变化变成噪声。

**2026 年的研究轨迹：**

- **数据约束状态。** Web 有有限数量的高质量标记（英语过滤后约 5-10 万亿）。前沿预训练正在接近这个上限。合成数据、多语言、多模态和 RLHF 缩放微调是下一个杠杆。
- **计算乘数技巧。** Muon 优化器、MoE、更好的数据策展——每个都改变绝对常数，而非渐近线。
- **RL 的缩放法则。** 开放问题。早期证据表明 RL 样本中遵循幂律，但指数与预训练非常不同。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------------|-----------------------|
| 参数（N） | "模型大小" | 非嵌入权重计数；决定容量。 |
| 标记（D） | "训练数据" | 看到的训练标记数；决定参数被利用的程度。 |
| 计算（C） | "花费的 FLOPs" | 对于标准 transformer 约为 `6 × N × D`。 |
| Chinchilla 最优 | "D/N ≈ 20" | 最小化预训练每 FLOP 损失的比率。 |
| 过度训练 | "超越 Chinchilla" | 花费额外训练 FLOPs 以节省推理 FLOPs；D/N >> 20。 |
| 不可约损失 | "下限" | 缩放法则中的 `E` 项；数据本身的熵。 |
| 涌现能力 | "规模下的突然跳跃" | 通常是评分器伪影；连续损失是平滑的。 |
| 有效计算 | "训练效率乘数" | 更好的数据/优化器/架构乘以一个 FLOP 走多远。 |
