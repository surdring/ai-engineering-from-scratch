## 优化技术全景

### 1. 显存优化
| 技术 | 原理 | 显存节省 |
|------|------|----------|
| 量化 (INT8/INT4) | 降低参数精度 | 2-4x |
| KV Cache 优化 | PagedAttention, vLLM | 2-3x |
| Flash Attention | IO-aware 精确注意力 | O(n²) → O(n) 显存 |

### 2. 延迟优化
| 技术 | 原理 | 速度提升 |
|------|------|----------|
| 推测解码 | 小模型草稿 + 大模型验证 | 2-3x |
| 量化 | 低精度推理 | 2-3x |
| 批处理 | Continuous Batching | 10x 吞吐 |
| 前缀缓存 | 同一前缀共享 KV Cache | 视场景 |

### 3. 架构级优化
| 技术 | 原理 |
|------|------|
| MoE (Mixture of Experts) | 稀疏激活，每 Token 只用部分参数 |
| GQA (Grouped Query Attention) | 减少 KV Head 数量 |
| SSM/Mamba | 线性复杂度替代注意力 |
| LoRA + 多路服务 | 一个基座 + 多个 LoRA adapter |

## 推理引擎对比

| 引擎 | 核心技术 | 特点 |
|------|----------|------|
| vLLM | PagedAttention | 开源标杆，生态最好 |
| TensorRT-LLM | NVIDIA 专有 | 最高性能，需要编译 |
| llama.cpp | GGUF 量化 | CPU 友好，本地推理 |
| SGLang | RadixAttention | 结构化生成优化 |
| Ollama | llama.cpp 封装 | 易用，一键本地部署 |
