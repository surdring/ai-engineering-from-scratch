## 一、项目概述

### 1.1 项目定位

**AI Engineering from Scratch** 是一个免费、开源、MIT 协议的全栈 AI 工程课程体系。它不是零散的教程合集，而是从数学基础到自主智能体系统的完整"脊柱"——435 节课程、20 个阶段、约 320 小时的系统化学习路径。

### 1.2 核心价值主张

课程引用了关键数据：**84% 的学生已经在使用 AI 工具，但只有 18% 认为自己在专业层面做好了准备**。本课程的目标正是填补这一鸿沟。

核心理念：
- 每一节课产生一个可复用的产物（Prompt、Skill、Agent 或 MCP Server）
- 从原始数学推导入手，先"造轮子"，再使用 PyTorch/框架
- 支持 Python、TypeScript、Rust、Julia 四种编程语言
- 不依赖在线视频，不依赖复制粘贴，强调亲手构建

### 1.3 项目规模

| 指标 | 数值 |
|------|------|
| 阶段（Phase） | 20 |
| 课程（Lesson） | 435 |
| 技能（Skills） | 378 |
| 提示词（Prompts） | 99 |

| 预计总学时 | ~320 小时 |

### 2.3 数据流与自动化

课程维护了一套完整的数据流管道：

```
文件系统 (phases/*)
    │
    ├──[build_catalog.py]──→ catalog.json    （机器可读目录）
    │                              │
    │                    [check_readme_counts.py]
    │                              ↓
    │                        一致性验证
    │
    ├──[audit_lessons.py]──→ 课程质量审计报告
    │                         （检查命名规范、必含文件、
    │                          frontmatter 格式等）
    │
    └──[install_skills.py]──→ 安装产物到目标目录
                              （支持 flat/by-phase/skills 布局）
```

CI 工作流 (`curriculum.yml`) 在每次 push/PR 时自动运行：
- **invariant checks**: 课程质量审计（`audit_lessons.py`）
- **catalog drift check**: 确保 `catalog.json` 与文件系统同步
- **README counts drift check**: 确保 README 中的统计数字准确

### 5.1 六步教学法

每节课遵循统一的教学节奏：

```
MOTTO（核心思想）→ PROBLEM（痛点）→ CONCEPT（概念+图解）
    → BUILD IT（从零实现）→ USE IT（框架实现）→ SHIP IT（产出可复用产物）
```

### 5.2 "Build It / Use It" 双轨制

这是课程最核心的设计理念：

- **Build It**：从原始数学/算法开始，用 stdlib 或最少依赖实现。例如优化器课程中，用纯 Python 实现 SGD、SGDMomentum、Adam、AdamW，包括一个手动实现反向传播的小型神经网络。
- **Use It**：用 PyTorch/sklearn 等生产框架做同样的事，对比验证。这使得学生既理解底层原理，又掌握生产工具。

### 5.3 产物驱动的学习

每节课产出 4 类可复用产物之一：

| 产物类型 | 数量 | 说明 |
|---------|------|------|
| **Prompt** | 99 | 可粘贴到任何 AI 助手的专业提示词 |
| **Skill** | 378 | 可安装到 Claude/Cursor/Codex 等 Agent 的技能模块 |
| **Agent** | 0（开发中） | 可部署的自主工作器 |
| **MCP Server** | 0（开发中） | 可接入 MCP 兼容客户端的服务器 |

产物安装方式：`python3 scripts/install_skills.py <target_dir>`

### 5.4 代码规范

- **无注释原则**：代码应当自解释，不添加注释（LESSON_TEMPLATE 明确要求 "No comments"）
- **渐进式构建**：从最简单的版本开始，逐步增加复杂性
- **可直接运行**：每个代码块应独立可运行
- **多语言并行**：核心概念提供 Python + Rust + TypeScript + Julia 实现

## 九、特色亮点

### 9.1 "脊柱"架构
20 个阶段线性堆叠，从数学做地板到 Agent 做屋顶。可以跳过已知阶段，但不会遇到"跳过底层导致上一层断裂"的情况。这种设计避免了碎片化学习的常见问题。

### 9.2 前沿性
课程覆盖了截至 2026 年的最新技术：
- DeepSeek-V3 架构与 NSA 稀疏注意力
- 多 Token 预测（MTP）
- DualPipe 并行策略
- Letta/MemGPT 的原生推理 Agent 循环
- MCP 协议完整生态（从基础到生产认证）
- A2A 协议
- Claude Agent SDK、OpenAI Agents SDK

### 9.3 批判性思维培养
术语表采用"人们说什么 → 实际上是什么"的格式，AI 误区解析直击常见谬误。课程在 Phase 15 还专门设立了"讨论型"课程探讨递归自我改进的矛盾与边界。

### 9.4 产物驱动
"每节课产出一个可复用工具"是这套课程差异化最显著的特征。435 节课 = 435 个可安装到 AI 助手的技能/提示词/Agent/MCP Server。这不仅是学习的副产品，也是职业作品集的核心资产。

### 9.5 多语言、多范式
同一概念提供 Python（快速原型）、TypeScript（Web 集成）、Rust（高性能）、Julia（科学计算）的多视角实现。

### 10.1 总体评价

**AI Engineering from Scratch** 是目前开源社区中覆盖范围最广、结构最系统、理论深度和实践并重的 AI 工程课程体系。它填补了"知道怎么调用 API"和"真正理解 AI 如何工作"之间的关键鸿沟。
