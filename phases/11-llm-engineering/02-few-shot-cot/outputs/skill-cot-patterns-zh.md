# 推理技术选择指南

当你需要 LLM 推理解决一个问题时，在编写提示之前选择技术。技术决定推理架构。提示填充其中。

## 快速决策树

1. 任务是简单的事实查询或单步分类吗？
   - 是：使用 **零样本（zero-shot）**。CoT 增加成本但无准确率增益。
   - 否：继续。

2. 任务需要多步推理（数学、逻辑、规划）吗？
   - 是：使用 **思维链（Chain-of-Thought）**。继续第 3 步。
   - 否：若格式重要则使用 **Few-Shot**，否则用零样本。

3. 单个推理错误是否可以接受？
   - 是：使用 **Few-Shot CoT**（单样本，温度 0.0）。
   - 否：使用 **自一致性（Self-Consistency）**（N=5，温度 0.7）。继续第 4 步。

4. 问题是具有很多可能路径的搜索/规划问题吗？
   - 是：使用 **思维树（Tree-of-Thought）**。
   - 否：自一致性就足够了。

5. 任务需要外部信息或计算吗？
   - 是：使用 **ReAct**（推理 + 工具调用）。
   - 否：纯推理技术就足够了。

## 技术矩阵

| 技术 | 准确率提升 | 成本倍数 | 延迟 | 最佳适用 |
|-----------|--------------|-----------------|---------|----------|
| 零样本 | 基线 | 1x | ~1s | 简单任务、事实问答 |
| Few-Shot | +5-15% | 1.2x | ~1s | 格式匹配、分类 |
| 零样本 CoT | +10-20% | 1.3x | ~1.5s | 快速推理增强 |
| Few-Shot CoT | +15-25% | 1.5x | ~2s | 数学、逻辑、多步 |
| 自一致性（N=5） | CoT 基础上 +2-5% | 5x | ~5s | 高风险推理 |
| 自一致性（N=10） | N=5 基础上 +1-2% | 10x | ~10s | 仅关键决策 |
| 思维树 | 依赖具体任务 | 10-40x | ~30s+ | 搜索、规划、谜题 |
| ReAct | 依赖具体任务 | 3-10x | ~5-15s | 知识密集型任务 |
| 提示链式 | 单次基础上 +5-10% | 2-5x | ~5-10s | 复杂多部分任务 |

## 模型特定指南

### GPT-4o / GPT-4.1
- 强大的基线推理能力。零样本 CoT 通常足够。
- 3 个示例的 Few-Shot CoT 在 GSM8K 上达到 95%。
- 自一致性仅带来边际收益（95% 到 97%）— 仅对关键任务值得。
- 原生支持结构化输出用于答案提取。

### Claude 3.5 Sonnet / Claude 3.7 Sonnet
- 在遵循结构化提示格式（XML 标签）方面非常出色。
- 使用 XML 分隔示例的 Few-Shot CoT 效果最佳。
- 扩展思考（Claude 3.7）是原生 CoT — 无需提示。
- 自一致性有效，因为 Claude 在温度 0.7 时的推理变化足够大。

### Llama 3.1/3.3 70B
- 从 Few-Shot CoT 中受益最多（相比零样本的准确率差距更大）。
- 推理任务推荐自一致性 N=5。
- 需要比商业模型更明确的格式指令。
- 思维树在本地推理上代价高昂 — 仅考虑用于批处理。

### Gemini 2.5 Pro
- 开箱即用的多步推理能力强。
- 思考模式提供内置 CoT，无需提示工程。
- Few-Shot 示例对格式一致性的帮助优于对准确率的帮助。
- 大上下文窗口（1M）使富含示例的 Few-Shot 变得可行。

## 反模式

**对简单任务使用 CoT**：问「2+2 是多少？让我们逐步思考」浪费 token。模型对简单算术无需推理轨迹就能做对。CoT 在有 3 步以上时才有帮助。

**温度 0.0 时使用自一致性**：所有 N 个样本将完全相同。必须使用温度 > 0（推荐 0.5-0.8）来获得多样化的推理路径。

**对所有任务使用思维树**：思维树需要 O(b^d) 次 LLM 调用，其中 b 为分支因子，d 为深度。一个 b=3, d=3 的树最多需要 39 次调用。仅用于更便宜技术失败的问题。

**使用错误示例的 Few-Shot**：包含推理错误的示例会教会模型犯那些错误。每个示例都必须经过验证。一个错误示例降低的准确率可能超过零示例。

**在没有一致格式的情况下提取答案**：自一致性需要跨样本比较答案。如果答案格式变化（「$18」「18 dollars」「eighteen」），投票失败。始终强制使用：「答案是 [数字]。」

## 成本优化

对于一个每天处理 10,000 次查询的生产系统，按 GPT-4o 定价（$2.50/1M 输入，$10/1M 输出）：

| 技术 | 平均 Token/查询 | 每日成本 | 准确率 |
|-----------|-----------------|------------|----------|
| 零样本 | ~200 | ~$5 | 78% |
| Few-Shot CoT | ~600 | ~$15 | 95% |
| 自一致性（N=5） | ~3,000 | ~$75 | 97% |
| 思维树（b=3, d=2） | ~6,000 | ~$150 | 依赖任务 |

大多数应用的成本最优策略：从 Few-Shot CoT 开始。仅对置信度低的查询添加自一致性（来自构建它部分的升级模式）。

## 与提示链式的集成

推理技术可与提示链式组合：

**链式步骤 1**（抽取）：零样本，温度 0.0
**链式步骤 2**（推理）：Few-Shot CoT，温度 0.0
**链式步骤 3**（验证）：自一致性 N=3，温度 0.7

这个三步链式的成本约为单次 CoT 调用的 3 倍，但能捕捉抽取错误、推理错误，并通过验证步骤提供置信度分数。

## 何时超越提示

如果你花在提示工程上的时间比编写应用代码更多，考虑：

1. **微调（Fine-tuning）**：如果你有 500 个以上标注示例且任务范围狭窄
2. **DSPy 编译**：如果你想要自动化的提示优化
3. **智能体框架（Agent frameworks）**：如果任务需要多轮工具使用（第 14 阶段）
4. **RAG**：如果模型需要访问私有/当前知识（第 06-07 课）

提示技术是基础。它们适用于任何模型、任何提供商，且不需要训练数据。但它们有局限性。知道何时升级到下一层次与掌握技术本身同样重要。
