# 嵌入模式

每个嵌入工作流遵循此约定：

```
text -> embed(text) -> vector (float array)
similarity(vector_a, vector_b) -> score (float)
```

嵌入模型和相似度度量是唯二重要的决策。其余都是管道。

## 何时使用嵌入

- 跨文档语义搜索（查找含义而非关键词）
- 聚类相似项（支持工单、产品评论、Bug 报告）
- 最近邻分类（通过与标注示例的相似度标注新项）
- 推荐系统（查找与用户喜欢的物品相似的物品）
- 去重（使用相似度阈值查找近似重复内容）

## 何时不使用嵌入

- 精确关键词匹配（使用全文搜索）
- 结构化查询（使用 SQL、过滤器）
- 小数据集手工标注更快（<100 项）
- 可解释性比准确率更重要的任务（嵌入不透明）

## 模型选择

根据约束条件选择：

- **需要 API，最佳性价比**：OpenAI text-embedding-3-small（1536d, $0.02/1M tokens）
- **需要最高准确率**：Voyage-3（1024d, $0.06/1M tokens, 最高 MTEB）
- **需要本地/私有**：BGE-M3（1024d, 免费, 多语言, 推荐 GPU）
- **需要快速本地原型**：all-MiniLM-L6-v2（384d, 免费, CPU 可运行）
- **需要多语言**：Cohere embed-v3（1024d）或 BGE-M3（两者多语言表现强）

规则：绝不在索引和查询之间混用嵌入模型。不同模型的向量存在于不兼容的空间中。

## 分块规则

1. 目标每块 256-512 token，50 token 重叠
2. 尽可能不要在句子中间截断
3. 每块附带元数据（源文件、章节标题、位置）
4. 对结构化文档（Markdown、HTML），优先在标题边界处分割
5. 通过搜索已知答案并检查检索结果来测试分块质量

## 相似度度量选择

- **余弦相似度**：默认选择，处理变长文本，已归一化
- **点积**：当向量已单位归一化时使用（OpenAI 模型已归一化），稍快
- **欧几里得距离**：用于聚类任务，当绝对位置重要时使用

当向量归一化后，三者给出相同排名。仅对未归一化向量，选择才有影响。

## 存储优化

三个可叠加的压缩级别：

1. **俄罗斯套娃截断**：减少维度（1536 -> 256 = 6 倍节省, 3-5% 准确率损失）
2. **Float16 量化**：每维度存储减半（2 倍节省, <1% 准确率损失）
3. **二值量化**：每维度 1 bit（32 倍节省, 5-10% 准确率损失, 配合重新评分使用）

生产模式：全语料库二值搜索，对 top-1000 用 float32 向量重新评分。

## 检索后重排

最佳准确率的两阶段流水线：

1. 双编码器检索 top-100 候选（快速，使用预计算嵌入）
2. 交叉编码器重排到 top-10（慢，处理每个查询-文档对）

这比单阶段检索在精确度指标上高出 10-15%。当准确率比延迟更重要时使用。

## 常见错误

- 索引和查询使用不同嵌入模型
- 嵌入整篇文档而非分块（嵌入变成所有内容的平均值）
- 余弦相似度前未归一化向量（大多数模型预归一化，但需验证）
- 忽略分块重叠（边界处分割的句子失去上下文）
- 只存储向量不存储原始文本（检索需要两者）
- 模型变更后不重新嵌入（旧向量不兼容）
- 仅凭准确率选择维度（存储和延迟随维度线性增长）

## 调试嵌入

如果搜索结果不佳：

1. 验证查询嵌入非零（空或仅空白输入产生零向量）
2. 手动检查已知相关文档的相似度分数
3. 尝试改写查询以匹配文档词汇
4. 检查分块边界，确保相关内容未被分割到不同块中
5. 跨度量比较 top-k 结果（余弦、点积、欧几里得）来发现归一化问题
6. 用可轻松匹配的查询（从文档中复制一个句子）测试，确认流水线正常工作

## 生产参数

- 分块大小：256-512 token
- 分块重叠：50 token（分块大小的 10-20%）
- Top-k 检索：直接使用 5-10，重排用 50-100
- 相似度阈值：余弦相似度 0.7+（低于此通常无关）
- 批量嵌入：每 API 调用处理 100-500 个文本以获得吞吐量
- 索引重建：模型变更或文档大幅更新时重新嵌入
