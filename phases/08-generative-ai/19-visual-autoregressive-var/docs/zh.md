# 视觉自回归建模（VAR）：下一尺度预测

> 扩散模型在时间上迭代采样（去噪步骤）。VAR 在尺度上迭代采样——它预测 1×1 标记，然后是 2×2，然后是 4×4，直至最终分辨率，每个尺度以前一个尺度为条件。2024 年的论文表明 VAR 在图像生成上匹配 GPT 风格的缩放定律，并且在相同计算预算下击败 DiT。本课构建核心机制。

**类型：** 构建
**语言：** Python（使用 PyTorch）
**前置要求：** 第 7 阶段第 03 课（多头注意力），第 8 阶段第 06 课（DDPM）
**时间：** 约 90 分钟

## 问题

自回归生成主导了语言建模，因为它可预测地缩放：更多计算、更多参数、更低困惑度、更好输出。2024 年之前图像生成有两个主要的 AR 尝试：PixelRNN/PixelCNN（逐像素）和 DALL-E 1 / Parti / MuseGAN（在 VQ-VAE 编码上逐标记）。

两者都受生成顺序问题的困扰。像素和标记排列在 2D 网格中，但 AR 模型必须以 1D 光栅顺序访问它们。早期角落像素不知道图像最终会变成什么。生成质量比 GPT-on-text 缩放更差，并且在匹配计算下从未达到扩散模型质量。

VAR 通过改变生成的内容来修复生成顺序问题。VAR 不是逐个标记在空间中预测图像标记，而是以递增分辨率预测整个图像。第 1 步：预测 1×1 标记（整体图像"摘要"）。第 2 步：预测 2×2 标记网格（较粗特征）。第 3 步：预测 4×4 网格。第 K 步：预测最终 (H/8)×(W/8) 网格。

每个尺度关注所有先前尺度（在"尺度顺序"上因果），并在自身尺度内并行。顺序问题消失：尺度 k 的整个图像在一次 transformer 传递中产生。

## 概念

### VQ-VAE 多尺度分词器

VAR 需要一个**多尺度离散分词器**。对于图像 x，它产生一系列渐进更高分辨率的标记网格：

```
x -> 编码器 -> 潜在 f
f -> 在 1×1 分词：标记网格 z_1 形状为 (1, 1)
f -> 在 2×2 分词：标记网格 z_2 形状为 (2, 2)
...
f -> 在 (H/p)×(W/p) 分词：标记网格 z_K 形状为 (H/p, W/p)
```

每个 z_k 使用相同的码本（典型大小 4096-16384）。每个尺度的分词不是独立的——它被训练为使每个尺度的残差之和重建 f：

```
f ≈ upsample(embed(z_1), target_size) + ... + upsample(embed(z_K), target_size)
```

这是一个**残差 VQ** 变体。尺度 k 捕获了尺度 1..k-1 遗漏的内容。解码器取所有尺度嵌入的和并产生图像。

多尺度 VQ 分词器训练一次（像 VQGAN）然后冻结。所有生成工作由之上的自回归模型完成。

### 下一尺度预测

生成模型是一个 transformer，它看到来自所有先前尺度的标记，并预测下一尺度的标记。

输入序列结构：
```
[START, z_1 标记, z_2 标记, z_3 标记, ..., z_K 标记]
```

位置嵌入编码尺度索引和尺度内的空间位置。注意力在尺度顺序上是因果的：尺度 k、位置 (i, j) 的标记可以关注尺度 1..k 的所有标记，以及尺度 k 自身中在所使用的任何尺度内顺序中较早出现的标记（VAR 使用固定位置注意力，无尺度内因果性——尺度内所有位置并行预测）。

训练损失：在每个尺度 k，给定所有先前尺度标记预测标记 z_k。离散 VQ 编码上的交叉熵损失。与 GPT 相同的结构，只是"序列"现在是尺度结构的。

### 生成

推理时：
```
生成 z_1 = 从 p(z_1) 采样                    # 1 个标记
生成 z_2 = 从 p(z_2 | z_1) 采样               # 4 个标记并行
生成 z_3 = 从 p(z_3 | z_1, z_2) 采样          # 16 个标记并行
...
解码：f = 尺度和的嵌入和上采样 1..K
图像 = VAE_decoder(f)
```

对于 K = 10 个尺度，生成是 10 次 transformer 前向传播。每次传递并行产生其整个尺度——尺度内没有逐标记自回归。对于 256×256 图像，这大约是 10 次传递 vs DiT 的 28-50 次。

### 为什么下一尺度胜过下一标记

三个结构性优势：
1. **粗到细与自然图像统计对齐。** 人类视觉感知和图像数据集都表现出尺度依赖的规律性：低频结构稳定且可预测；高频细节以低频内容为条件。下一尺度预测利用这一点。
2. **尺度内并行生成。** 与 GPT 风格标记 AR 不同，VAR 一步产生一个尺度的所有标记。有效生成长度是对数尺度而非线性。
3. **无生成顺序偏差。** 尺度 k 的标记看到整个尺度 k-1；没有"左于"或"上于"偏差迫使早期标记在晚期上下文可用之前提交。

### 缩放定律

Tian et al. 证明 VAR 在 ImageNet 上对 FID 遵循幂律缩放曲线——就像 GPT 对困惑度所做的。参数或计算加倍可靠地将误差减半。这是第一个像语言模型一样干净地展示这种缩放行为的图像生成模型。结果是 VAR 尺度预测从计算可预测，而非每个架构的经验猜测。

### 与扩散的关系

VAR 和扩散共享相同的数据压缩故事：都将生成问题分解为一系列更简单的子问题。

- 扩散：逐步添加噪声，学习撤销一步。
- VAR：逐步添加分辨率，学习预测下一个尺度。

它们是穿过问题的不同轴。两者都产生可处理的条件的分布。经验上 VAR 在推理时更快（更少传递，尺度内全部并行），并在类条件 ImageNet 上匹配或击败 DiT。文本条件 VAR（VARclip、HART）是一个活跃的研究方向。

## 构建

在 `code/main.py` 中你将：
1. 在合成"图像"数据（2D 高斯环）上构建一个微型**多尺度 VQ 分词器**。
2. 训练一个**VAR 风格 transformer** 进行下一尺度预测标记。
3. 通过调用 transformer 4 次（4 个尺度）采样并解码。
4. 验证尺度顺序训练使尺度内生成并行。

这是一个玩具实现。重点是看到尺度结构注意力掩码和尺度内并行生成实际工作。

## 练习

1. **尺度数量消融。** 用 4、6、8、10 个尺度训练 VAR。测量重建质量 vs 自回归传递次数。更多尺度 = 更细残差 = 更好质量但更多传递。

2. **码本大小。** 用码本大小 512、4096、16384 训练分词器。更大码本给出更好重建但更难预测。找到拐点。

3. **尺度内并行检查。** 对于训练好的 VAR，显式测量注意力模式。在尺度 k 内，模型是否关注跨尺度位置但不关注尺度内？验证掩码实现。

4. **VAR vs DiT 缩放。** 对于相同的 ImageNet 类条件任务，在匹配参数预算（例如 33M、130M、458M）下训练 VAR 和 DiT。绘制 FID vs 计算。VAR 应在每种规模上领先 DiT——在小规模下复现论文结果。

5. **文本条件。** 扩展 VAR 以通过 adaLN 接受文本嵌入（CLIP 池化）作为额外条件输入。这是 HART 配方。在文本对齐采样上 FID 改善多少？

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|----------------|----------------------|
| VAR | "视觉自回归" | 通过在 VQ 标记网格金字塔上的下一尺度预测进行图像生成 |
| 下一尺度预测 | "先预测粗，再预测细" | 模型以递增分辨率尺度预测标记，以所有先前尺度为条件 |
| 多尺度 VQ 分词器 | "残差 VQ" | VQ-VAE 产生 K 个递增分辨率的标记网格，解码器对所有尺度求和 |
| 尺度 k | "金字塔层级 k" | K 个分辨率层级之一，从 k=1 的 1×1 到 k=K 的 (H/p)×(W/p) |
| 尺度内并行 | "每个尺度一次前向" | 尺度 k 的所有标记在一次 transformer 传递中预测，不是自回归 |
| 跨尺度因果 | "尺度顺序注意力" | 尺度 k 的标记可以关注尺度 1..k 的全部但不能关注尺度 k+1..K |
| 残差 VQ | "加法分词" | 每个尺度的标记编码较低尺度留下的残差；解码器对所有尺度嵌入求和 |
| VAR 缩放定律 | "图像 GPT 缩放" | FID 在计算上遵循可预测的幂律，像语言模型的困惑度 |
| HART | "混合 VAR + 文本" | 结合 MaskGIT 风格迭代解码与 VAR 尺度结构的文本条件 VAR 变体 |
| 尺度位置嵌入 | "(尺度, 行, 列) 三元组" | 位置编码携带尺度索引和尺度内的空间坐标 |
