给定图像目标（分辨率、通道数、彩色 vs 灰度、数据集大小、下游 LM 计算预算、目标 FID），输出：

1. 尺度调度。列出 K 个分辨率级别，从 1x1 到 (H/p) x (W/p)。默认 256x256 用 10 个尺度，512x512 用 14 个。根据 LM 的有效序列长度（尺度面积之和）和每轮尺度内并行预算论证 K 的值。
2. 码本。所有尺度共享单个码本大小 V（典型为 4096 / 8192 / 16384）。根据数据集大小和解码器容量选择 V。确认在校准批次上码本使用率保持在 50% 以上，否则减小 V。
3. 残差共享。确认尺度 1..K 通过求和上采样的嵌入共同重建隐变量（残差 VQ）。明确补丁大小 p 和 VAE 骨干（是否开启 VQGAN 风格的判别器、感知损失权重）。
4. 解码器。VAE 解码器将求和后的隐变量映射回像素。从 VQGAN 解码器、VAR 论文解码器或更轻量的 MAGVIT 风格解码器中选择。对照 FID 目标和解码器 VRAM 进行论证。
5. 位置嵌入。确认使用 (scale_index, row, col) 三元组，每个尺度使用可学习嵌入，尺度内使用 2D 正弦余弦。拒绝展平的 1D 位置；LM 需要尺度标签来应用正确的条件。

拒绝为非残差多尺度分词器做 VAR。没有求和残差，下一尺度条件将变得定义不清，且 LM 将优化与论文所证明的不同目标。拒绝使用每尺度独立码本，除非 V 已根据较小尺度的像素数校准且码本崩塌已缓解。当 K × 平均尺度面积超过 LM 的最大序列长度减去文本条件余量时，拒绝进行下一尺度预测。

示例输入："ImageNet 类别条件 256x256，数据集 1.2M，LM 预算 1.5B 参数，目标 FID < 5.0。"

示例输出：
- 尺度调度：K=10，大小 1, 2, 3, 4, 5, 6, 8, 10, 13, 16。总 token 数 671。
- 码本：共享，V=4096。在 256 分辨率的 ImageNet 上预期使用率 70-80%。
- 残差共享：确认；p=16，VQGAN 骨干 + 感知损失 + 对抗损失，残差和重建 f。
- 解码器：VQGAN 解码器，4 个上采样块，无额外精炼器。
- 位置嵌入：(scale, row, col) 三元组，可学习尺度 token + 尺度内 2D 正弦余弦。
