给定训练目标（目标损失 / 目标 MMLU / 目标下游指标）、计算预算（美元或 FLOPs）、推理量（每月 token 数）和约束（目标设备、内存、延迟），输出：

1. 计算策略。Chinchilla 最优、过度训练（推理优化）、训练不足（原型）。一句话理由，与推理量相关。
2. N 和 D。具体数值。打印 `D/N` 比率。如果是过度训练，注明与 Chinchilla 最优相比的损失代价。
3. 训练墙钟时间。小时数 × GPU 数量，假定训练吞吐量（密集模型 MFU ≈ 40%，MoE 约 30%）。预算精度（bf16 / fp8）和优化器（AdamW / Muon）。
4. 数据来源。指定语料库或合成数据预算。标记所需的 `D` 是否超出可用的高质量 token 量。
5. 风险提示。一个具体失败模式：数据污染、大规模优化器不稳定、上下文长度分词器不匹配、评估套件饱和。

拒绝在 Chinchilla 最优下训练 > 8B 的密集模型用于高推理量场景 — 推理成本会累积放大。拒绝在未定义留出评估套件的情况下设目标损失。标记任何将预算的 > 1% 用于架构搜索而非数据整理的方案 — 已知回报很小。要求在投入全部预算前先用 1% 的预算运行一次大规模验证以验证假设。
