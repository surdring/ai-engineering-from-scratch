# 为什么是 Transformer——RNN 的问题

> RNN 一次处理一个标记。Transformer 一次处理所有标记。这一架构选择在 2017 年后改变了深度学习的每一条缩放曲线。

**类型：** 学习
**语言：** Python
**前置要求：** 第 3 阶段（深度学习核心），第 5 阶段 · 09（序列到序列），第 5 阶段 · 10（注意力机制）
**时间：** 约 45 分钟

## 问题

2017 年之前，地球上每一个最先进的序列模型——语言、翻译、语音——都是循环神经网络。LSTM 和 GRU 在半个十年里赢得了翻译基准的 ImageNet 等效。它们是每个人唯一的工具。

它们有三个致命弱点。序列计算意味着你不能沿时间轴并行化：标记 `t+1` 需要来自标记 `t` 的隐藏状态。一个 1024 标记的序列意味着在可以每周期做 1,000,000 次浮点运算的 GPU 上进行 1024 个串行步骤。训练墙钟时间在专为并行设计的硬件上随序列长度线性增长。

梯度消失意味着 50 个标记之前的信息已经通过 50 个非线性层压缩了。门控循环单元（LSTM、GRU）缓解了挤压但从未消除它。长程依赖——"last summer on a plane to Kyoto I read a book that was..."——常规失败。

固定宽度隐藏状态意味着编码器在解码器看到任何东西之前将整个源序列压缩进单个向量。源序列是 5 个标记还是 500 个不重要；瓶颈是相同的形状。

2017 年论文"Attention Is All You Need"提出了激进的东西：完全放弃循环。让每个位置并行关注每个其他位置。在一次大矩阵乘法中训练，而非 1024 次串行。

结果是到 2026 年统治每个模态。语言（GPT-5、Claude 4、Llama 4）、视觉（ViT、DINOv2、SAM 3）、音频（Whisper）、生物学（AlphaFold 3）、机器人学（RT-2）。相同模块，不同输入。

## 概念

![RNN 串行计算 vs Transformer 并行注意力](../assets/rnn-vs-transformer.svg)

**循环作为瓶颈。** RNN 计算 `h_t = f(h_{t-1}, x_t)`。每一步依赖上一步。你不能在 `h_4` 之前计算 `h_5`。在拥有 10,000+ 并行核心的现代 GPU 上，这在长序列上浪费了 99% 的硅。

**注意力作为广播。** 自注意力同时为每一对 `(i, j)` 计算 `output_i = sum_j(a_ij * v_j)`。整个 N×N 注意力矩阵在一次批量矩阵乘法中填满。没有任何步骤依赖另一个。GPU 热爱它。

**加速不是常数。** 它是 `O(N)` 串行深度和 `O(1)` 串行深度之间的差异。在实践中，在 N=512 的匹配硬件上，transformer 每轮训练快 5-10×，差距随序列长度扩大，直到达到注意力的 `O(N²)` 内存墙（Flash Attention 后来修复了——见第 12 课）。

**Transfomer 的代价。** 注意力内存以 `O(N²)` 缩放。2K 上下文，可以。128K 上下文，你需要滑动窗口、RoPE 外推、Flash Attention 分块或线性注意力变体。循环在时间和内存上都是 `O(N)`；transformer 以内存换时间，然后通过并行化赢回时间。

**归纳偏置转变。** RNN 假设局部性和最近性。Transformer 什么都不假设——每一对都是注意力的候选。这就是为什么 transformer 需要更多数据来训练好，但一旦有数据就能进一步缩放。Chinchilla（2022）形式化了这一点：给定足够的标记，transformer 总是击败等参数量的 RNN。

## 构建

这里没有神经网络——我们数值模拟核心瓶颈，让你在笔记本上感受差距。

### 步骤 1：衡量串行深度

见 `code/main.py`。我们构建两个函数。一个将序列编码为加法链（串行，像 RNN）。一个将其编码为并行规约（广播，像注意力）。相同数学，不同依赖图。

```python
def rnn_style(xs):
    h = 0.0
    for x in xs:
        h = 0.9 * h + x   # 不能并行化：h 依赖之前的 h
    return h

def attention_style(xs):
    return sum(xs) / len(xs)  # 每个 x 独立
```

我们在高达 100,000 元素的序列上计时两者。RNN 版本是 O(N)，单个 CPU 流水线。即使在纯 Python 中，注意力风格的规约在长度 ≥ 1,000 时击败它，因为 Python 的 `sum()` 是 C 实现的，每步没有解释器开销。

### 步骤 2：计算理论操作

两种算法都做 N 次加法。区别在于*依赖深度*：有多少操作必须在下一个开始前串行发生。RNN 深度 = N。注意力深度 = 树规约的 log(N)，或并行扫描的 1。深度，而非操作数量，决定 GPU 时间。

### 步骤 3：长序列上的经验缩放

我们打印一个时间表使 O(N) 差距可见。在 2026 Mac 笔记本上，1000 元素以下的序列太快无法测量。100,000 元素显示清晰的线性扫描。将其扩展到 16,384 标记的 transformer 搭配 12 层 LSTM 等效，你就能看到为什么训练墙钟时间是 2016 年的阻塞点。

## 使用

2026 年何时仍选择 RNN：

| 场景 | 选择 |
|-----------|------|
| 流式推理，一次一个标记，恒定内存 | RNN 或状态空间模型（Mamba、RWKV） |
| 极长序列（>1M 标记），注意力内存爆炸 | 线性注意力、Mamba 2、Hyena |
| 没有矩阵乘法加速器的边缘设备 | 深度可分离 RNN 在 FLOPs/瓦特上仍胜出 |
| 其他（训练、批量推理、高达 128K 上下文） | Transformer |

状态空间模型（SSMs）如 Mamba 本质上是具有结构化参数化的 RNN，使它们兼得两者之长：`O(N)` 扫描内存，通过选择性扫描实现并行训练。它们恢复 90% 的 transformer 质量，且具有更好的长上下文缩放。2026 年大多数前沿实验室训练混合 SSM+transformer 模型（如 Jamba、Samba）——循环没有消亡，它是一个组件。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------|---------|
| 循环 | "RNN 是串行的" | 步骤 `t` 依赖步骤 `t-1` 的计算，强制沿时间轴串行执行。 |
| 串行深度 | "图有多深" | 依赖操作的最长链；即使在无限硬件上也限制墙钟时间。 |
| 注意力 | "让标记互相看" | 加权和 `sum_j a_ij v_j`，其中 `a_ij` 来自位置 i 和 j 之间的相似度分数。 |
| 上下文窗口 | "模型能看多少" | 注意力层可以接受输入的标记数；二次内存成本在此缩放。 |
| 归纳偏置 | "架构中隐含的假设" | 关于数据样子的先验；CNN 假设平移不变性，RNN 假设最近性。 |
| 状态空间模型 | "带代数背后的 RNN" | 通过结构化状态空间矩阵参数化用于并行训练的循环。 |
| 二次瓶颈 | "为什么上下文成本这么高" | 注意力内存 = 序列长度上的 `O(N²)`；Flash Attention 隐藏常数，而非缩放。 |
