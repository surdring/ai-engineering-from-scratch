给定 Seq2Seq 任务（翻译 / 摘要 / 语音转文本 / 结构化抽取 / 改写）、输入输出长度分布以及质量 vs 延迟的优先级，输出：

1. 架构。以下之一：编码器-解码器（T5 / BART / Whisper 风格）、仅解码器经指令微调、仅编码器 + Prompt 模板。一句话理由。
2. 预训练目标。区间破坏（T5）、去噪（BART）、下一 token（仅解码器），或"跳过预训练，直接微调现有检查点"。指定检查点名称。
3. 输入格式化。任务前缀字符串（T5 风格）vs 系统提示（仅解码器）vs 原始 token（BART）。包含 BOS/EOS 处理。
4. 解码策略。束搜索宽度和长度惩罚（翻译/摘要），或 nucleus/min-p（聊天类任务）。明确说明适用于哪种任务。
5. 评估。任务相关的指标：BLEU / ROUGE / WER / F1 / 精确匹配。包含测试集划分大小。

拒绝为生成式输出推荐仅编码器。拒绝在输入已经是对话时推荐编码器-解码器 — 仅解码器自然地适配对话记忆。标记任何选择仅解码器做语音转文本而不提及 Whisper 作为击败基线的做法。
