# 注意力变体选择器

帮助开发者为其新 Transformer 选择并论证注意力拓扑结构，或为已有模型扩展到更长上下文而选择。

## 需收集的输入

1. **训练和目标推理的上下文长度**（训练 16K → 推理扩展，经常不同）。
2. **检索需求**（1-5 分）：1 = 纯聊天，5 = 大海捞针 / RAG / 长仓库上下文代码。
3. **推理内存预算**：每请求 KV 缓存容忍度（正确单位是每层每 token 的字节数）。
4. **训练成本容忍度**：从头训练 SWA 成本低；将差分注意力改造进预训练模型成本高。
5. **硬件目标**：Hopper+ 有完整 FlashAttention-3，Ada 有 FA2，旧 GPU 受遮罩限制。

## 决策规则

- **上下文 ≤ 16K 且检索 ≤ 3**：全注意力 + FlashAttention。不要过早优化。
- **上下文 16–128K 且检索 ≤ 3**：SWA + 全局混合，比例 5:1，窗口 1024（Gemma 3 形状）。在压缩 KV 的同时保持检索可用。
- **上下文 > 128K**：全 SWA，每 4-6 层一个全局层，加位置插值 / YaRN 缩放（第 04 课）。
- **检索 = 5 且训练预算允许**：考虑仅在前 4 层使用差分注意力（KV 翻倍减半，获得大部分 sink 消除收益）。
- **你在发布公开 API**：优先稳定模式（全注意力、SWA、Gemma-3 混合）。除非有 kernel 工程师，跳过原生稀疏/差分注意力。
- **无法更改基础模型**：SWA 可通过遮罩在推理时改造；差分和稀疏注意力无法。

## 始终标记

- 低于 7B 的纯 SWA 模型通常在推理基准上可测量地下降。建议反对。
- 窗口大小 < 512 几乎从不正确。应更大或使用不同拓扑。
- 差分注意力论文中的数据是基于小型模型的（3-7B）。截至 2026 年初，放大证据薄弱。
- 每种变体都与 RoPE / YaRN 缩放交互（第 04 课）。明确声明位置方案。
