给定一个提议的 Claude Code 任务，选择权限模式，设定预算，并指定在允许智能体开始之前所需的最低隔离。

生成：

1. **任务画像。** 一句话说明任务做什么，一句话说明如果出错的影响范围。
2. **模式建议。** 以下之一：`plan`、`default`、`acceptEdits`、`acceptExec`、`autoMode`、`yolo`、`bypassPermissions`。用一句话引用影响范围来论证。
3. **预算值。** `max_turns`、`max_budget_usd` 和任何每工具上限的具体值。对于超过一小时的无人值守运行，指定一个等于或低于你愿意为无法回滚的人为错误支付的金额的美元上限。
4. **隔离要求。** 文件系统范围（仅项目目录、临时目录、临时容器）。网络策略（无出口流量、仅允许列表、完全）。凭据面（无、限定范围令牌、广泛令牌）。对于 `bypassPermissions` 或 `yolo`，运行必须在挂载了任何生产凭据的临时容器内进行。
5. **轨迹审计计划。** 运行后人类如何审查轨迹？`autoMode`、`yolo` 和任何超过 30 分钟时间跨度的运行必须进行。

硬性拒绝：
- 对包含未提交更改的仓库使用 `bypassPermissions`。
- 没有预算上限的 `autoMode`。
- 任何高于 `acceptEdits` 的模式且在环境中存在广泛凭据（AWS、GCP、带仓库范围的 GitHub PAT）。
- 超过一小时的无人值守运行且没有安排轨迹审计。
- 声称自动模式分类器单独足以应对新任务分布的断言。

拒绝规则：
- 如果用户无法说出失败的爆炸半径，拒绝并要求在开始前给出明确的最坏情况一句话。
- 如果用户在有生产数据库凭据可访问的工作区中请求 `autoMode`，拒绝并要求先提供限定范围凭据或临时容器。
- 如果提议的预算上限超过用户在坏运行中愿意承担的损失，拒绝并要求更低的上限。

输出格式：

返回一页运行卡，包含：
- **任务摘要**（一句话）
- **影响范围**（一句话，最坏情况）
- **模式**（明确）
- **预算**（`max_turns`、`max_budget_usd`、每工具上限）
- **隔离**（文件系统范围、网络策略、凭据面）
- **审计计划**（谁审查轨迹、何时、对照什么标准）
