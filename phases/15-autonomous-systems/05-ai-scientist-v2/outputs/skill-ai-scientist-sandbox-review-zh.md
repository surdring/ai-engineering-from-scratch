给定由 AI-Scientist-v2 风格循环产生的自主研究输出（假设、代码、实验、图表、论文草稿），生成双门审查：沙箱审计（是否有任何内容离开？）加科研审计（工作是否可靠？）。

两个门直接映射到以下审计：**沙箱门 = 第 1 项**；**科研门 = 第 2 项（实验审计）+ 第 3 项（润色审计）**。第 4-5 项管理两个门通过后的事情。

生成：

1. **沙箱门。** 在任何产出物离开沙箱之前：
   - 列出循环所做的每个网络调用及其目标。标记任何未经预先批准的调用。
   - 清点循环在工作目录之外写入的每个文件。
   - 确认 Docker / seccomp / gVisor 容器隔离在整个运行期间保持有效。
   - 确认没有子进程逃脱沙箱的监控。
   如果任何检查失败，阻止导出；上报给人类。
2. **实验审计。** 阅读实验代码，而非论文：
   - 验证每个声称的实验确实运行过，且报告的数字是可复现的。
   - 检查失败的实验是否被报告为失败，而非事后被重新包装为负面结果。
   - 检查想法的「新颖性」标签是否经受住人类领域专家的文献检索。
3. **润色审计。** 阅读图表：
   - 确保每个图表的数据来自已记录的实验运行，而非润色阶段的改写。
   - 确认坐标轴、刻度和标注与底层数据匹配。
   - 标记任何图注声称超过数据支持的图表。
4. **披露计划。** 如果产出物用于外部分发：
   - 披露产出物由智能体创作。
   - 披露使用的工具（模型家族、循环版本）。
   - 披露检查它的人工审查者以及他们检查了什么。
5. **负面发布决定。** 如果产出物未通过任何审计步骤，默认是不发布。覆盖此默认需要指定的人类负责人。

硬性拒绝：
- 任何跳过任一门的提交。
- 循环的执行日志缺失或不完整的任何产出物。
- 任何无法追溯到特定实验运行的图表。
- 任何未经领域专家验证的新颖性声明。

拒绝规则：
- 如果运行缺少 Docker 或等效隔离，拒绝并要求在隔离沙箱中重新运行。
- 如果用户无法提供实验阶段的执行日志，拒绝——论文无法审查。
- 如果提议的分发渠道是同行评审场所，而用户提议不披露智能体作者身份，拒绝并要求披露。

输出格式：

返回双门报告：
- **沙箱门裁决**（PASS / BLOCK，附理由）
- **科研门裁决**（涵盖实验审计（2）和润色审计（3））（PASS / BLOCK / REQUIRES_EXPERT，附逐项注释）
- **披露计划**（场所、文本、人工审查者姓名）
- **发布决定**（发布 / 保留 / 拒绝）
- **下一步行动**（谁在什么时间做什么）
