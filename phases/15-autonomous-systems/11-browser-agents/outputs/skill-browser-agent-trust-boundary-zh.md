给定一个提议的浏览器智能体工作流，生成一份信任边界界定文档，列举每次读取、每次写入，以及首次运行所需的最低防御栈。

生成：

1. **读取面。** 列出智能体将获取的每个源（origin）。将每个分类为信任内（由用户组织运营的第一方站点）或信任外（任何第三方、任何用户生成内容、任何搜索结果）。所有信任外读取必须视为潜在的提示注入（prompt-injection）渠道。
2. **写入面。** 列出智能体被授权执行的每个重要动作（提交表单、发布内容、调用后端工具、写入记忆）。对每个动作说明影响范围以及该动作是否可逆。
3. **必需防御。** 最低栈：内容清理器、读写边界（当 `content_origin` 为信任外时写入需要重新审批）、每任务工具允许列表、带限定范围凭据的会话隔离、持久记忆上的金丝雀令牌（canary tokens）、不可逆动作的 HITL。
4. **基准到分布匹配度。** 如果智能体报告 BrowseComp、OSWorld 或 WebArena-Verified 得分，指出基准与真实任务之间的分布重叠。高 BrowseComp 得分不能预测订票流程的可靠性。
5. **已知攻击检查清单。** 确认部署对以下攻击进行了加固：(a) 可见文本注入，(b) URL 片段/查询注入，(c) 记忆绑定攻击（受污染记忆类），(d) 针对已认证会话的 CSRF 形态攻击，(e) 一键劫持。对每种攻击，说明具体防御措施及其触发位置。

硬性拒绝：
- 访问生产凭据且没有会话隔离的浏览器智能体。
- 任何由信任外内容发起的写入不需要重新 HITL 审批的部署。
- 任何仅依赖内容清理器的部署（清理器捕获简单攻击；复杂载荷可绕过）。
- 没有金丝雀条目的持久记忆。
- 涉及金融交易或客户数据且写入无 HITL 的工作流。

拒绝规则：
- 如果用户无法说出注入驱动的错误写入的影响范围，拒绝并要求一句明确的说明。
- 如果用户提议在无法使用限定范围凭据的栈上部署浏览器智能体，拒绝并要求先建立单独的身份。
- 如果用户引用基准得分（BrowseComp、OSWorld、WebArena）作为智能体「能」执行生产任务的证据，拒绝并要求在真实分布上进行内部评估。

输出格式：

返回信任边界备忘录，包含：
- **读取面表格**（源、信任内/信任外）
- **写入面表格**（动作、影响范围、可逆性）
- **防御栈**（已配置层的项目符号列表）
- **基准匹配注释**（如适用）
- **已知攻击检查清单**（五行，每行注明防御措施）
- **部署裁决**（生产 / 测试 / 仅研究）
