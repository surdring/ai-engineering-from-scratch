# 模型评估策略

正确评估任何机器学习模型的清单。按此顺序操作以避免最常见的评估错误。

## 第 1 步：正确划分数据

- 在任何预处理（缩放、插补、编码）之前先划分
- 分类任务使用分层划分（Stratified Split）
- 保留一个测试集，最终只使用一次
- 小数据集使用 5 折或 10 折交叉验证而非单次划分
- 时间序列使用基于时间的划分（绝不打乱）

## 第 2 步：选择正确的指标

### 分类

| 场景 | 使用此指标 | 原因 |
|------|----------|------|
| 类别平衡，简单比较 | 准确率（Accuracy） | 易于解释，当类别相等时有意义 |
| 假阳性代价高（垃圾邮件过滤、欺诈预警） | 精确率（Precision） | 衡量标记项目中有多少真正是正类 |
| 假阴性代价高（癌症筛查、安全检测） | 召回率（Recall） | 衡量实际正类中有多少被捕获 |
| 需要平衡精确率和召回率 | F1 分数 | 调和平均，惩罚极端不平衡 |
| 跨阈值比较模型 | AUC-ROC | 与阈值无关的排序质量 |
| 不平衡数据 | F1、AUC-ROC 或 PR-AUC | 在不平衡类别中准确率具有误导性 |

### 回归

| 场景 | 使用此指标 | 原因 |
|------|----------|------|
| 标准回归，可接受离群值 | RMSE | 与目标同单位，惩罚大误差 |
| 离群值鲁棒评估 | MAE | 对所有误差平等对待，不受离群值主导 |
| 不同尺度上比较模型 | R-squared | 归一化的 0-1 尺度（解释的方差比例） |
| 业务需要金额度量 | MAE 或 RMSE | 直接可解释为误差幅度 |

## 第 3 步：建立基线

在评估模型之前，计算基线性能：
- 分类：多数类预测器（始终预测最常见的类别）
- 回归：始终预测训练目标值的均值
- 任何无法超过这些基线的模型都算不上在学习

## 第 4 步：交叉验证

- 使用 K 折（K=5 或 K=10）以获得稳定估计
- 分类任务使用分层 K 折
- 报告各折的均值和标准差
- 均值 = 0.85、标准差 = 0.02 的模型比均值 = 0.87、标准差 = 0.10 的更可信

## 第 5 步：统计比较模型

- 不要不检查显著性就选择平均分数最高的模型
- 在交叉验证各折上使用成对 t 检验
- 如果 |t| < 2.78（K=5，df=4，p<0.05），差异可能由偶然性导致
- 当性能差异不显著时，考虑选择更简单的模型

## 第 6 步：检查常见错误

- 数据泄漏：是否有测试集信息流入训练？（划分前缩放、目标衍生特征）
- 类别不平衡：准确率是否掩盖了少数类的糟糕表现？
- 过拟合：训练和验证性能之间的差距是否很大？
- 评估次数过多：是否多次查看了测试集？

## 第 7 步：报告最终性能

- 在训练集和验证集合并后的数据上训练
- 对预留的测试集仅评估一次
- 报告所选指标，如可能附上置信区间
- 说明基线比较（比随机/均值好了多少）
