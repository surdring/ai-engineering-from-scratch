# 分类基线指南

在尝试复杂模型之前，先用逻辑回归（Logistic Regression）建立基线。它训练只需几秒，能产生概率输出，且完全可解释。很多现实世界的问题实际上不需要更复杂的方法。

## 决策清单

1. 决策边界是否可能为线性？
   - 是：逻辑回归可能就足够了
   - 否：你仍然需要它作为衡量改进的基线

2. 有多少特征？
   - 50 个以下：标准逻辑回归可以胜任
   - 50 到 10,000：添加 L2 正则化（Ridge）
   - 10,000 以上（如 TF-IDF 文本特征）：使用 L1 正则化（Lasso）或 LinearSVC

3. 数据集是否不平衡？
   - 比例低于 5:1：不需调整可能就可以
   - 5:1 到 50:1：在 sklearn 中使用 `class_weight="balanced"`
   - 50:1 以上：将类别权重与适当的指标（精确率、召回率或 F1）结合

4. 特征是否在不同尺度上？
   - 逻辑回归前始终标准化。它使用基于梯度的优化，未缩放的特征会减慢收敛或扭曲决策边界。

5. 是否有缺失值？
   - 拟合前先插补。逻辑回归不能处理 NaN。
   - 数值列使用中位数插补，类别列使用众数插补。

## 逻辑回归足够好的场景

- 特征关系主要为线性的二分类
- 需要概率输出（不仅是类别标签）
- 可解释性是必需的（标准化后系数指示特征重要性的方向和相对大小）
- 训练数据较小（数百到低数千个样本）
- 需要快速模型用于实时服务（推理时只需单次点积）
- 监管或合规要求要求可解释性

## 何时升级

- 准确率远低于目标且已尝试特征工程
- 特征与目标之间的关系明显非线性（检查残差图）
- 有大量表格数据（10k+ 行）：尝试梯度提升（XGBoost 或 LightGBM）
- 特征具有多项式特征无法捕捉的复杂交互
- 有图像、文本或序列数据：原始输入上的逻辑回归无法工作

## 分类基线的预处理步骤

1. 在任何预处理之前先做**训练/测试集划分**，这可以防止数据泄漏。
2. **处理缺失值**：数值列中位数插补，类别列众数插补。
3. **编码类别特征**：低基数（10 个值以下）用 One-Hot 编码，高基数用目标编码（Target Encoding）。仅在训练折上拟合目标编码（使用袋外编码防止泄漏）。
4. **缩放数值特征**：StandardScaler（零均值、单位方差）。在训练集上拟合，在两者上转换。
5. **拟合逻辑回归**，使用 `C=1.0`（默认正则化）。
6. **评估**：混淆矩阵、精确率、召回率、F1。不仅是准确率。
7. **调优阈值**：默认 0.5 很少是最优的。在 0.1 到 0.9 之间扫描，选择匹配你精确率/召回率优先级的阈值。

## 常见错误

- 在不平衡数据上只评估准确率（预测多数类的模型得分很高但毫无用处）
- 忘记缩放特征（未缩放特征的逻辑回归训练慢且收敛到更差的解）
- 使用测试集来调优决策阈值（应使用验证集或交叉验证）
- 跳过基线直接跳到 XGBoost（你失去了可解释性且没有参考点）
- 不检查多重共线性（高度相关的特征会放大系数方差）

## 快速参考

| 场景 | 模型 | 正则化 | 关键设置 |
|------|------|-------|---------|
| 特征少，可解释 | LogisticRegression | L2（默认） | C=1.0 |
| 特征多，部分无关 | LogisticRegression | L1 | penalty="l1", solver="saga" |
| 高维稀疏（文本） | SGDClassifier | L1 或 ElasticNet | loss="log_loss" |
| 类别不平衡 | LogisticRegression | L2 | class_weight="balanced" |
| 需要概率 | LogisticRegression | L2 | predict_proba() |
| 仅需类别标签 | LinearSVC | L2 | 大数据上比 LR 快 |
