# 回归策略指南

回归预测连续值。正确的方法取决于特征与目标之间的关系、特征数量以及过拟合风险。

## 决策清单

1. 特征与目标之间的关系是否近似线性？
   - 是：从普通线性回归开始
   - 否：尝试多项式特征或非线性模型

2. 相对于样本数量，特征有多少？
   - 特征少，样本多：普通线性回归可以胜任
   - 特征多，样本少：使用正则化（Ridge 或 Lasso）
   - 特征多于样本：Lasso（L1）用于特征选择，或 Ridge（L2）用于收缩所有权重

3. 是否需要可解释性？
   - 是：特征较少的线性回归，或使用 Lasso 进行自动特征选择
   - 否：多项式特征，或转向树模型或神经网络

4. 数据集是否较小（少于 10,000 行）？
   - 使用正规方程（闭式解）以获得速度
   - 交叉验证对于可靠评估至关重要

5. 数据集是否较大（数百万行）？
   - 使用随机梯度下降（SGD）或小批量梯度下降
   - 正规方程由于 O(n^3) 的矩阵求逆过程太慢

## 各方法适用场景

**普通线性回归（OLS）**：任何回归任务的基线。从这里开始。如果 R-squared 可接受且模型简单，到此为止。

**多项式回归（Polynomial Regression）**：散点图显示曲线而非直线。从 2 阶开始。仅当验证性能证明有必要时才增加。阶数 > 5 几乎总是过拟合。

**岭回归（Ridge Regression，L2）**：许多相关特征。所有权重向零收缩但没有一个变为零。适用于认为所有特征都有贡献的情况。

**Lasso 回归（L1）**：特征很多但怀疑只有少数有效。Lasso 将无关特征权重驱至恰好为零，执行自动特征选择。

**弹性网络（Elastic Net）**：结合 L1 和 L2 惩罚。适用于有许多相关特征且希望进行部分特征选择的情况。

## 常见错误

- 梯度下降前跳过特征缩放（收敛变得极慢）
- 使用测试集性能来调超参数（应使用验证集或交叉验证）
- 拟合高阶多项式而不检查验证误差（训练 R^2 总是随阶数增加）
- 忽视残差图（如果残差呈现模式，R^2 可能具有误导性）
- 将 R^2 作为唯一指标（还需检查残差分布、MAE 和特定领域的阈值）

## 快速参考

| 方法 | 适用场景 | 正则化 | 特征选择 |
|------|---------|-------|---------|
| OLS | 基线，特征少 | 无 | 手动 |
| Ridge | 特征多，全部相关 | L2（收缩） | 无 |
| Lasso | 特征多，少数相关 | L1（置零） | 自动 |
| Elastic Net | 许多相关特征 | L1 + L2 | 部分 |
| 多项式 | 非线性关系 | 附加 Ridge/Lasso | 手动选择阶数 |
