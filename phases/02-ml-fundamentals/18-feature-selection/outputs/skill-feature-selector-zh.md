# 特征选择策略

选择和应用合适特征选择方法的快速参考。

## 第 1 步：从清理开始

在应用任何方法之前，先移除明显无用的特征：

- **常量特征**：方差 = 0。移除它们。
- **近似常量特征**：方差 < 0.01（或你定的阈值）。移除它们。
- **重复特征**：完全相同的列。保留一个，丢弃其余的。
- **ID 列**：每行唯一，不含可泛化的信息。移除它们。

这只需几秒钟，在杂乱的实际数据集中可以消除 10-30% 的特征。

## 第 2 步：根据情况选择方法

### 快速决策树

1. **< 50 个特征？** 从互信息排序开始。保留 Top K。
2. **50 - 500 个特征？** 先用方差阈值过滤，然后线性模型用 L1（Lasso），树模型用树特征重要性。
3. **> 500 个特征？** 链式方法：方差阈值 -> 互信息过滤（Top 50%）-> 在幸存特征上使用 RFE。
4. **需要可解释性？** L1 正则化给出精确的零/非零。树特征重要性给出排序分数。
5. **需要捕捉非线性关系？** 互信息或基于树的特征重要性。避免 L1（仅限线性）。
6. **需要特征交互？** RFE 或基于树的特征重要性。过滤方法无法捕捉交互。

### 方法参考

| 方法 | 适用场景 | 避免场景 |
|------|---------|--------|
| 方差阈值 | 始终，作为第一步 | 绝不要跳过这一步 |
| 互信息 | 快速排序，非线性关系 | 需要特征交互检测时 |
| RFE | 彻底选择，中等特征数量 | 模型代价极高，> 1000 特征 |
| L1 / Lasso | 线性模型，快速嵌入式选择 | 非线性问题，高度相关特征 |
| 树特征重要性 | 非线性关系，特征交互 | 受高基数特征偏差影响 |
| 排列重要性 | 模型无关的验证，最终检查 | 对初步筛选来说太慢 |

## 第 3 步：验证你的选择

- 比较使用所选特征与所有特征的模型性能
- 使用交叉验证，而非单次训练/测试划分
- 如果性能下降超过 1-2%，你可能移除了一些有用的特征
- 如果性能提升，你成功移除了噪声

## 第 4 步：处理常见陷阱

### 相关特征
- L1 从一组相关特征中任意选择一个并将其他置零
- 先计算相关矩阵，决定保留哪些相关特征
- 树特征重要性会在相关特征之间分散重要性

### 数据泄漏
- 仅在训练数据上拟合特征选择
- 将相同的选择应用到测试数据
- 交叉验证中，特征选择必须在每一折内进行

### 对特征选择过拟合
- RFE 迭代次数太多会对训练集过拟合
- 在留出数据上验证，而非用于选择的数据
- 使用稳定性选择（在子样本上重复）以获得更鲁棒的结果

## 第 5 步：生产环境检查清单

- [ ] 方差阈值作为第一个过滤器已应用
- [ ] 特征选择仅在训练数据上拟合
- [ ] 所选特征已记录（名称、使用方法、分数）
- [ ] 性能比较：所选特征 vs 所有特征
- [ ] 已交叉验证，而非单次划分评估
- [ ] 特征选择已集成到训练流水线中（非手动完成）
- [ ] 特征漂移监控已到位（所选特征可能变得过时）
