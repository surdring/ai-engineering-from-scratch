# 不平衡数据策略

处理不平衡分类的决策清单。按此顺序操作，为你的问题选择正确的方法。

## 第 1 步：衡量不平衡程度

- 统计每个类别的样本数
- 计算不平衡比率（多数类 / 少数类）
- 轻度：比率 < 3:1（如 70/30）
- 中度：比率 3:1 到 20:1（如 95/5）
- 严重：比率 > 20:1（如 99/1）

## 第 2 步：选择正确的指标

对不平衡数据集优先使用精确率/召回率/F1 而非准确率。根据问题选择：

| 情况 | 主要指标 | 辅助指标 |
|------|---------|---------|
| 遗漏正类代价极高（欺诈、疾病） | 召回率 | F2 分数 |
| 误报代价高（垃圾邮件过滤、推荐） | 精确率 | F0.5 分数 |
| 两者大致同等重要 | F1 分数 | MCC |
| 需要单一排序指标 | AUPRC | AUC-ROC |
| 需要跨数据集比较 | MCC | AUPRC |

## 第 3 步：选择再平衡策略

### 按不平衡严重程度

| 不平衡程度 | 首先尝试 | 其次尝试 | 避免 |
|-----------|---------|---------|------|
| 轻度（< 3:1） | 类别权重 | 阈值调优 | 过采样（不必要） |
| 中度（3:1 到 20:1） | SMOTE + 类别权重 | 在此基础上阈值调优 | 欠采样（数据损失太多） |
| 严重（> 20:1） | SMOTE + 类别权重 + 阈值 | 带平衡装袋的集成 | 仅欠采样 |

### 按数据集规模

| 数据集规模 | 首选策略 | 原因 |
|-----------|---------|------|
| < 1,000 样本 | 过采样或 SMOTE | 无法承受丢失多数类数据 |
| 1,000 - 10,000 | SMOTE + 阈值调优 | 有足够少数类样本用于 k-NN |
| > 10,000 | 类别权重或欠采样 | 快，少数类数据充足 |

## 第 4 步：应用技术

### 类别权重（始终首先尝试）
- sklearn 中：`class_weight='balanced'`
- 无需修改数据
- 适用于任何基于损失的模型
- 在期望上等价于过采样

### SMOTE
- 仅应用于训练数据（绝不在测试/验证集上）
- 使用 k=5 近邻（默认）
- 与类别权重结合使用以获得最佳效果
- 注意边界附近有噪声的合成点

### 阈值调优
- 训练模型，在验证集上获取预测概率
- 在 0.05 到 0.95 之间扫描阈值
- 选择最大化所选指标的阈值
- 始终在验证数据上调优，绝不在测试数据上

## 第 5 步：正确验证

- 使用分层交叉验证（保持各折的类别比例）
- 在原始（非重采样）的测试集上报告指标
- 绝不在划分前应用 SMOTE——仅在训练折上
- 与「始终预测多数类」的基线比较

## 第 6 步：需要避免的常见错误

- 在训练/测试划分前对整个数据集应用 SMOTE（数据泄漏）
- 使用准确率作为评估指标
- 不先尝试类别权重（最简单的方法，通常足够）
- 过采样后进行交叉验证（合成点会泄漏到各折）
- 忽略阈值调优（免费的性能提升，无需重新训练）
- 在小数据集上使用随机欠采样（丢弃了太多数据）

## 快速决策树

1. 不平衡比率 < 3:1？ -> 仅尝试类别权重
2. 数据集 > 10,000 样本？ -> 类别权重 + 阈值调优
3. 数据集 < 1,000 样本？ -> SMOTE + 类别权重
4. 否则 -> SMOTE + 类别权重 + 阈值调优
5. 仍然不够好？ -> 平衡装袋集成
