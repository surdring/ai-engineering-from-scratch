# 多模态智能体与计算机使用（综合项目）

> 2026 年的前沿产品是一个多模态智能体，能读取屏幕截图、点击按钮、导航网页 UI、填写表单，并端到端完成工作流。SeeClick 和 CogAgent（2024）证明了 GUI 定位（GUI Grounding）原语。Ferret-UI 增加了移动端支持。ChartAgent 为图表引入了视觉工具使用。VisualWebArena 和 AgentVista（2026）是前沿模型追逐的基准测试——然而即使 Gemini 3 Pro 和 Claude Opus 4.7 在 AgentVista 的困难任务上也仅得分约 30%。本综合项目将串联第12阶段的所有线索：感知（高分辨率 VLM）、推理（带工具使用的 LLM）、定位（坐标输出）、长时程记忆和评估。

**类型：** 综合项目
**语言：** Python（标准库，动作 Schema + 智能体循环骨架）
**前置要求：** 第12阶段 · 05（LLaVA），第12阶段 · 09（Qwen-VL JSON），第14阶段（Agent 工程）
**时间：** ~240分钟

## 学习目标

- 设计一个多模态智能体循环：感知 → 推理 → 行动 → 观察 → 重复。
- 构建一个 VLM 可以作为 JSON 输出的 GUI 定位输出 Schema（点击坐标、输入文本、滚动、拖拽）。
- 比较纯截图智能体 vs 无障碍树智能体 vs 混合智能体。
- 在一个小型 VisualWebArena 切片上建立多模态智能体基准测试评估。

## 问题

一个预订网站的工作流："帮我找一张4月15日飞东京的航班，靠过道座位，800美元以下，预订它。"

多模态智能体需要：

1. 截取浏览器的屏幕截图。
2. 将截图 + URL + 目标解析为计划。
3. 发出结构化动作：点击（在 x,y）、输入"Tokyo"（在元素 E）、向下滚动、选择（单选按钮）。
4. 将动作应用到浏览器。
5. 观察新状态（下一张屏幕截图）。
6. 重复直到任务完成。

每一步都是一次多模态 VLM 调用。VLM 输出必须是可解析的 JSON。错误会跨步骤累积，因此恢复能力至关重要。

## 核心概念

### GUI 定位——原语

GUI 定位是：给定一张屏幕截图和一条自然语言指令，输出要点击的 (x, y) 坐标（或其他动作）。

SeeClick（arXiv:2401.10935）是首个大规模开源成果：在合成 + 真实 GUI 数据上微调 VLM，以纯文本 Token 输出坐标。有效。

CogAgent（arXiv:2312.08914）为密集 UI 添加了 1120x1120 高分辨率编码。得分：网页导航约 84%。

Ferret-UI（arXiv:2404.05719）专注于移动端 UI，集成 iOS 无障碍数据。

输出格式通常是 JSON：

```json
{"action": "click", "x": 384, "y": 220, "element_desc": "搜索按钮"}
```

`element_desc` 有助于恢复：如果屏幕截图之间坐标偏移，语义提示让系统可以重新定位。

### 动作 Schema

典型的动作 Schema 有 6-10 种动作类型：

- `click`：(x, y)
- `type`：(text, x?, y?)
- `scroll`：(direction, amount)
- `drag`：(x0, y0, x1, y1)
- `select`：(option_index)
- `hover`：(x, y)
- `navigate`：(url)
- `wait`：(ms)
- `done`：(success, explanation)

智能体每步发出一个动作。浏览器包装器执行并返回新状态。

### 纯截图 vs 无障碍树

两种输入模式：

- 纯截图：完整图像，无结构信息。最通用；适用于任何应用。
- 无障碍树：结构化的 DOM / iOS 无障碍信息。定位更可靠；在有树的场景下效果最好。
- 混合：两者兼有，无障碍树作为原子动作的可靠定位器，截图用于语义上下文。

生产级智能体在可能时使用混合模式。浏览器自动化（Selenium + 无障碍）总是有树；桌面应用有时有。

### 长时程记忆

一个 20 步的工作流生成 20 张屏幕截图。VLM 的上下文很快填满。三种压缩策略：

- 摘要链：每 5 步后，总结已发生的事情，丢弃旧的屏幕截图。
- 跳帧：保留第一张、最后一张和每隔两张的屏幕截图。
- 工具记录日志：执行动作，保留所做操作的文本日志；不再回顾旧屏幕截图。

Claude 的计算机使用 API 使用日志模式。更简单，更可靠。

### 视觉工具使用

ChartAgent（arXiv:2510.04514）为图表理解引入了视觉工具使用：裁剪、缩放、OCR、调用外部检测。智能体可以输出"裁剪到区域 (100, 200, 300, 400) 然后调用 OCR"作为工具调用。工具返回文本；VLM 继续推理。

这一模式可以推广：标记集提示（Set-of-Mark Prompting）、区域标注和外部检测工具都符合相同的"输出工具调用，接收结构化响应" Schema。

### 2026 年的基准测试

- ScreenSpot-Pro。约 1k 网页截图的 GUI 定位。开源 SOTA Qwen2.5-VL-72B 约 85%。前沿约 90%。
- VisualWebArena。端到端网页任务（购物、论坛、分类广告）。开源 SOTA 约 20%。Gemini 3 Pro 约 27%。
- AgentVista（arXiv:2602.23166）。2026 年最难的基准测试。12 个领域的真实工作流。前沿模型得分 27-40%；开源模型 10-20%。
- WebArena / WebShop。较旧的基准测试；已被前沿模型饱和。

### 为什么仍然很难

智能体性能瓶颈：

1. 精细尺度的视觉定位。"点击那个小 X"在移动端分辨率下经常失败。
2. 长时程规划。10 个动作后，智能体偏离目标。
3. 错误恢复。当一次点击失败（错误按钮），检测 + 恢复很少是训练数据的内容。
4. 跨页面上下文。在标签页之间跳转或长表单会丢失状态。

研究方向：记忆架构、显式重新规划、多模态验证（动作成功的屏幕截图匹配）。

### 综合项目构建任务

综合项目任务：构建一个计算机使用智能体，能够：

1. 读取预订网站模拟页面的 HTML + 屏幕截图。
2. 规划多步序列：搜索 → 选择 → 填表单 → 提交。
3. 发出符合动作 Schema 的 JSON 动作。
4. 在固定的 10 任务切片上评估。

本课提供可轻松扩展为真实浏览器的脚手架代码。

## 实践

`code/main.py` 是综合项目脚手架：

- 动作 Schema JSON 定义（10 种动作）。
- 模拟浏览器状态（字典）。
- 智能体循环骨架：接收状态，发出动作，应用，循环。
- 10 任务迷你基准测试（合成页面），测量端到端成功率。
- 动作失败时的错误恢复钩子。

## 练习

1. 扩展动作 Schema，添加 `screenshot_region` 工具（裁剪 + 缩放）。哪些任务受益？

2. 阅读 AgentVista（arXiv:2602.23166）。描述最难的任务类别以及为什么前沿模型仍然失败。

3. 长时程记忆压缩：设计一个摘要链，最多保留 4 张实时屏幕截图，任意数量的日志记录。

4. 构建错误恢复钩子：动作失败时（按钮未找到），智能体接下来做什么？

5. 在 10 个网页任务上比较纯截图 Claude 4.7 和混合截图 + 无障碍树 Qwen2.5-VL。哪种在哪些任务上胜出？

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------|---------|
| GUI 定位 | "点击坐标" | 模型输出指令目标在屏幕截图上的 (x,y) 坐标 |
| 动作 Schema | "工具定义" | JSON 描述的有效动作（点击、输入、滚动、拖拽） |
| 无障碍树 | "结构化 DOM" | 来自浏览器/iOS API 的可机读 UI 层级 |
| 混合智能体 | "截图 + 树" | 同时使用图像和结构化信息；比单独任一种更可靠 |
| 视觉工具使用 | "缩放/裁剪/检测" | 智能体在规划中途调用外部视觉工具（OCR、检测） |
| 摘要链 | "记忆压缩" | 定期文本摘要替代长屏幕截图历史 |
| VisualWebArena | "端到端网页基准测试" | 2024 年端到端网页任务基准测试 |
| AgentVista | "2026 年高难度基准测试" | 12 个领域的真实工作流；Gemini 3 Pro 也仅得分约 30% |
