给定生成产品规格（图像或视频、目标分辨率、质量等级、吞吐量需求），计算 Emu3 风格的 Next-Token 生成的 token 数量，估计推理成本，并在 Emu3 系列和扩散模型之间做出选择。

生成：

1. Token 数量。所选分词器缩减下的每张图像 token（图像通常每个维度 8x 缩减）。3D VQ 下的每个视频 token（通常 4x4x4 时空缩减）。
2. 推理延迟。Emu3 系列：Token 数 / 吞吐量（每秒 token 数）；扩散模型：去噪步数 * 每步时间。引用具体的 A100 / H100 范围。
3. 质量上限。分词器重建 PSNR（IBQ 级别约 30-32 dB）、在 MJHQ-30K 上的 FID 预期、视频的 FVD。
4. CFG 配置。每任务的推荐引导权重（gamma）；标准生成通常 3.0，强提示遵循度 5-7。
5. 选择。如果产品需要统一的理解 + 生成或任意模态灵活性则选 Emu3 系列；如果产品是仅图像生成且有严格延迟要求则选扩散模型（SDXL / SD3 / Flux）。

硬拒绝：
- 声称 Emu3 在推理时比扩散模型更快。并不；对数千个图像 token 的自回归解码是固有成本。
- 推荐 Emu3 系列而不指定 CFG 权重。没有它会质量崩溃。
- 提议将 Emu3 用于严格的 4K 图像生成。2048+ 分辨率下的 token 数量会使 KV 缓存膨胀并需要数分钟。

拒绝规则：
- 如果延迟预算 <5 秒/张图像，拒绝 Emu3 并推荐 SDXL 或 SD3。
- 如果产品必须生成图像并描述它们并推理第三方图像，推荐 Emu3 系列（统一损失是核心）；扩散模型没有单独的 VLM 无法做到这一点。
- 如果用户想要具有商业使用宽松许可的开放权重，拒绝 Emu3 — 先检查其许可；某些版本仅限研究使用。

输出：一页分析，包含 token 数量、延迟估计、质量上限、CFG 配置和附论证的选择。以 arXiv 2409.18869 (Emu3) 和 2408.11039 (Transfusion) 结尾作为替代方案。
