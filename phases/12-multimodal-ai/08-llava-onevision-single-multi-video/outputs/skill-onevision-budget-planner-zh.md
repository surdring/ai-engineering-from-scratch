给定产品的预期任务分布 — 单图像、多图像和视频请求的百分比 — 以及每个样本的视觉 token 预算，为每种场景输出分配计划和训练课程大纲。

生成：

1. 每种场景配置。单图像：AnyRes 切块数 + 缩略图 + 池化因子；多图像：每样本图像数 + 每张图像池化；视频：帧数 + 每帧池化。
2. Token 预算平衡。每种场景的总 token 应落在目标预算的 ±30% 以内；标记任何低于目标 70%（token 不足）或超过 130%（上下文风险）的场景。
3. 课程大纲。三个阶段（SI → OV → TT）及数据权重。TT 阶段使用用户的产品组合。
4. 预期的涌现技能。给定用户的产品组合，预测可能出现的 LLaVA-OneVision 风格的涌现能力（多摄像头、标记集、截图智能体或产品特定的变体）。
5. 训练数据估算。每个阶段所需的近似 token / 图像 / 帧数量，给定 7B 基础 LLM，引用 OneVision-1.5 数据规模。

硬拒绝：
- 提议将视频或多图像放在单图像之前的阶段顺序。OneVision 显示这会损失 2-4 MMMU。
- 当产品 80% 是单图像时将全部预算分配给视频。浪费而非平衡。
- 假设 AnyRes-16（4x4 网格）能在 4k token 预算下运行而不进行激进池化。不可能。

拒绝规则：
- 如果每样本 token 预算低于 1024，拒绝多图像或视频用例 — 低于该底线场景会崩溃。
- 如果用户想要 5+ 帧视频以完整 729 token 分辨率运行，拒绝；推荐 3x 池化或更少帧数。
- 如果产品分布完全省略单图像，拒绝并推荐 Qwen2.5-VL 风格的 M-RoPE 代替 — OneVision 的课程大纲假设单图像是感知基础。

输出：一页计划，包含每场景 token 配置、课程阶段权重、涌现技能预测和数据规模估计。以指向 arXiv 2408.03326 (OneVision) 和 arXiv 2509.23661 (OneVision-1.5 完全开源) 的指针结尾。
