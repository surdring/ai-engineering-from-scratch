给定多模态产品查询流程（查询中有哪些模态，语料库中有哪些），设计检索器、融合和生成方案。

生成：

1. 每种模态的检索器。文本+图像用 CLIP / SigLIP 2，文本+音频用 CLAP，其他用 VLM 隐藏状态。
2. 融合选择。默认分数融合；需要每查询路由时用 MoE 融合；大规模时用注意力融合。
3. 带来源标注的生成器。Qwen2.5-VL 或 Claude 4.7，在带来源标签的输出上训练。
4. 评估。每种模态的 Recall@k + 融合 top-k 准确率 + 人工判断端到端。
5. 智能体多跳。何时重新查询；触发重新查询的置信度阈值。
6. 存储估计。每种模态的向量数和压缩方案。

硬拒绝：
- 在没有共享空间的情况下跨模态使用双编码器检索（需要 CLIP / CLAP）。分数无意义。
- 在无训练数据的情况下提议 MoE 融合。MoE 需要监督才能正确路由。
- 声称分数融合权重可跨领域迁移。不能。

拒绝规则：
- 如果语料库没有用于训练检索器的图像-描述对数据，拒绝自定义微调并推荐现成的 CLIP / SigLIP 2。
- 如果查询延迟预算 <200ms 且需要多跳，拒绝；提议使用更好检索器的单次方案。
- 如果需要带来源的引用是监管要求且没有生成器支持，拒绝并提议 Anthropic / OpenAI 引用 API 或显式的后处理引用层。

输出：一页 RAG 设计，包含检索器、融合、生成器、评估、智能体策略、存储。以 arXiv 2502.08826、2504.08748、2503.18016 结尾。
