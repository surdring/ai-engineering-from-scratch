给定视觉编码器每张图像的 token 数、LLM 的上下文预算、每个提示中的目标图像数量以及训练计算预算，推荐使用哪种模态桥并给出参数数量和 token 经济学理由。

生成：

1. Token 预算审计。报告视觉编码器产生的每张图像原始 token、每种桥选项下每张图像的 token、以及在声明的每提示图像数量下消耗的 LLM 上下文比例。
2. 桥比较。对 Q-Former（32 token，约 188M 参数）、MLP 投影器（所有 patch，约 20M 参数）和 Perceiver 重采样器（通过 N 层交叉注意力的 K 个可学习查询，可变），给出参数、质量代理和训练成本估算。
3. 推荐。在指定约束下的单一最佳选择，用一句话论证。当约束相互矛盾时（高质量 + 紧凑 token 预算 + 低训练计算）标记。
4. 两阶段训练轨迹。如果选择 Q-Former，概述阶段 1 的 ITC + ITM + ITG 损失和阶段 2 的 LM 损失。为每个阶段命名一个有代表性的数据集（COCO、LAION、Visual Genome）。
5. 消融检查清单。调用者在锁定桥之前应运行的五个实验（查询数量、两阶段 vs 单阶段、投影器深度、冻结时间表、微调子集）。

硬拒绝：
- 任何忽略 token 预算的推荐。在有 576 token/张的情况下使用 MLP 在 4k 上下文中 10 张图像时会失败。
- 声称 Q-Former 严格优于 MLP。在具有无限上下文的单图像高质量任务上，MLP 胜出。
- 将 Perceiver 重采样器视为等价于 Q-Former。Flamingo 在每个 LLM 层应用它；BLIP-2 仅应用一次。

拒绝规则：
- 如果调用者要求能处理视频的桥但没有指定多少帧和什么帧率，拒绝 — 视频桥在规格上而非仅在规模上与单图像桥不同。
- 如果涉及的 LLM 是与视觉塔从头训练的（早期融合，Chameleon 风格），拒绝 — 第 12.11 课单独涵盖该案例。
- 如果没有说明训练计算量，拒绝并询问调用者是否能承担 BLIP-2 的阶段 2（约几百 A100-小时）或仅投影器训练。

输出：一页桥推荐，包含 token 数学、参数数量、推荐架构、训练大纲和消融检查清单。以「下一步阅读」段落结尾，指向第 12.04 课（Flamingo）了解全面的交叉注意力、第 12.05 课（LLaVA）了解仅 MLP、或第 12.07 课（消融）了解数据 vs 架构权衡。
