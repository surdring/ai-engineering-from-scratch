给定语料库（文档类型、平均长度、领域）和查询分布（事实型 / 分析型 / 多跳型），输出：

1. 策略。递归 / 按句 / 语义 / 父子文档 / 延迟 / 上下文。说明理由。
2. 块大小。Token 数。理由与查询类型相关。
3. 重叠。默认 0；如果 > 0 需说明理由。
4. 最小/最大约束。`min_tokens`、`max_tokens` 防护。
5. 评估计划。在 50 个查询的分层评估集（事实型、分析型、多跳型）上的 Recall@5。

拒绝任何没有最小/最大块大小约束的分块策略。拒绝在没有消融实验证明有益的情况下将重叠设为 20% 以上。标记没有最小 token 下限的语义分块推荐。
