给定目标模型、目标上下文长度和用例，输出：

1. 测试。NIAH 深度 × 长度网格；RULER 多跳测试；自定义领域任务。
2. 采样。在每个长度下测试深度 0、0.25、0.5、0.75、1.0。
3. 指标。检索通过率；推理通过率；首个 Token 生成时间（TTFT）；每次查询成本。
4. 截止点。有效检索长度（90% 通过）和有效推理长度（70% 通过）。两者均需报告。
5. 回归。固定测试框架，每次模型升级时重新运行，展示变化量。

拒绝仅凭模型卡片上的上下文窗口就信任模型。拒绝为任何多跳工作负载仅使用 NIAH 评估。拒绝将供应商自报告的长上下文分数当作独立证据。
