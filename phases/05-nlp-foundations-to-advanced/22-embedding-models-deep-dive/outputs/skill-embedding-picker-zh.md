给定语料库（规模、语言、领域、平均长度）、部署目标（云端 / 边缘 / 本地部署）、延迟预算和存储预算，输出：

1. 模型。指定的检查点或 API。一句话说明理由。
2. 维度。全维度 / Matryoshka 截断 / int8 量化。理由与存储预算相关。
3. 模式。密集 / 稀疏 / 多向量 / 混合。说明理由。
4. 如果模型卡片要求，提供查询前缀/模板。
5. 评估计划。与领域相关的 MTEB 任务 + 带 nDCG@10 的留出领域评估。

拒绝在没有领域验证的情况下推荐将 Matryoshka 截断到 < 64 维。拒绝为少于 1 万段落的语料库推荐 ColBERTv2（开销不合理）。标记将长文档语料库（> 8k token）路由到只有 512 token 窗口的模型。
