给定用例（领域、语言、词汇开放性、合规需求），输出：

1. 模式。领域列表、每个领域的槽位、每个槽位的开放 vs 封闭词汇。
2. 抽取器。基于规则 / Seq2Seq / LLM-配合-Pydantic。说明理由。
3. 更新策略。重新生成完整状态 / 增量更新；纠正处理；否定处理。
4. 评估。留出对话集上的联合目标准确率（Joint Goal Accuracy），槽位级精确率/召回率，最难槽位上的混淆矩阵。
5. 确认流程。何时显式请求用户确认（破坏性操作、低置信度抽取）。

拒绝为合规敏感的槽位使用纯 LLM DST，除非具备基于规则的二次检查。拒绝任何无法在用户纠正时回滚槽位的 DST。标记没有版本标签的模式设计。
