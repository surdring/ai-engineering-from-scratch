# 综合项目 17 — 个性化 AI 导师（自适应的、多模态的、带记忆的）

> Khanmigo（Khan Academy）、Duolingo Max、Google LearnLM / Gemini for Education、Quizlet Q-Chat 和 Synthesis Tutor 在 2026 年都大规模交付了自适应多模态辅导。共同形态是苏格拉底式策略（绝不直接给出答案）、每次交互后更新的学习者模型（贝叶斯知识追踪风格）、语音 + 文本 + 拍照数学输入、课程图谱检索、间隔重复调度，以及针对适龄内容的严格安全过滤器。本综合项目旨在交付一个学科特定的导师（K-12 代数或 Python 入门），对 10 名学习者进行为期两周的效能研究，并通过内容安全审计。

**类型：** 综合项目
**语言：** Python（后端，学习者模型）、TypeScript（Web 应用）、SQL（课程图谱通过 Postgres + Neo4j）
**前置知识：** 阶段 5（NLP）、阶段 6（语音）、阶段 11（LLM 工程）、阶段 12（多模态）、阶段 14（Agent）、阶段 17（基础设施）、阶段 18（安全）
**涵盖阶段：** P5 · P6 · P11 · P12 · P14 · P17 · P18
**时间：** 30 小时

## 问题

自适应辅导曾经是教育科技研究的小众领域。到 2026 年，它已成为消费级产品。Khanmigo 已在美国大多数学区部署。Duolingo Max 达到了数千万月活跃用户。Google 的 LearnLM / Gemini for Education 为 Google Classroom 中的辅导提供支持。Quizlet Q-Chat 与闪卡并排存在。Synthesis Tutor 以面向好奇儿童的导师身份病毒式传播。共同元素：多模态输入（打字、语音、拍摄方程）、苏格拉底式教学法（先问后解释）、每次交互后更新的学习者模型，以及严格的适龄安全。

你将为一个特定群体构建这些元素中的一个。衡量标准是实际的效能研究：10 名学习者两周内的前测和后测分数。语音循环必须感觉自然（综合项目 03 子技术栈）。记忆必须是尊重隐私的。安全过滤器必须通过面向 K-12 的 COPPA 意识红队审查。

## 概念

四个组件。**导师策略**是一个苏格拉底式循环：当学习者要求答案时，策略提出引导性问题；当他们答对时，进入下一个概念；当他们卡住时，提供支架式提示。**学习者模型**是贝叶斯知识追踪（或一个简单变体），在每次交互后更新每个课程节点的掌握概率。**课程图谱**是一个包含概念及其前置边的 Neo4j 图；策略遍历图谱以选择下一个概念。**记忆**是一个情景 + 语义存储（agentmemory 风格），保存过去的交互、错误和偏好。

用户体验是多模态的。用于键入答案的文本输入。通过 LiveKit + Whisper 的语音输入（复用综合项目 03）。通过 dots.ocr 或 PaliGemma 2 的拍照数学问题输入。通过 Cartesia Sonic-2 的语音输出。安全使用 Llama Guard 4 加上适龄过滤器（阻止成人内容、暴力、自残）和 COPPA 意识的记忆保留策略。

效能研究是交付物。10 名学习者，前测和后测，两周。报告学习增益差异和置信区间。与非自适应基线（相同内容线性交付，不使用导师策略）进行比较。

## 技术栈

- 学科选择：K-12 代数或 Python 入门（任选其一深入）
- 导师策略：LangGraph 运行在 Claude Sonnet 4.7（配合提示缓存）
- 学习者模型：贝叶斯知识追踪（经典）或用于间隔的 FSRS
- 课程图谱：概念 Neo4j + 前置边 + 开放教育资源内容
- 记忆：agentmemory 风格持久化向量 + 情景 + 语义存储
- 语音：LiveKit Agents 1.0 + Cartesia Sonic-2（复用综合项目 03 子技术栈）
- 拍照数学：dots.ocr 或 PaliGemma 2 用于方程识别
- 安全：Llama Guard 4 + 自定义适龄过滤器
- 评估：Bloom 层次问题生成、前/后测工具、效能研究工具

## 构建步骤

1. **课程图谱。** 构建一个包含 50-150 个概念节点的 Neo4j（例如，K-12 代数从 "数轴" 到 "二次公式"），带有前置边。为每个节点附加开放教育资源内容（Open Textbook、OpenStax）。

2. **学习者模型。** 使用先验概率初始化贝叶斯知识追踪：猜测、遗忘、学习率。每次交互后更新每个概念的掌握程度。按学习者持久化。

3. **导师策略。** LangGraph 节点：`read_signal`（学习者的答案是正确的 / 部分正确 / 卡住的？）、`select_concept`（遍历课程图谱选择最高优先级概念）、`scaffold`（苏格拉底式提示）、`update_mastery`。

4. **记忆。** 每次交互写入情景存储。错误和偏好提升到语义记忆。COPPA 意识保留策略：1 年后自动删除，家长可访问。

5. **语音路径。** LiveKit Agents 工作进程连接到导师策略。ASR 通过 Whisper-v3-turbo。TTS 通过 Cartesia Sonic-2。支持打断（复用综合项目 03 机制）。

6. **拍照数学路径。** 上传或拍摄图像；运行 dots.ocr 或 PaliGemma 2 识别方程；作为结构化输入馈送给导师。

7. **安全。** 每个模型输出通过 Llama Guard 4 + 适龄过滤器（阻止自残、成人内容、暴力）。记忆访问按学习者 ID 限定范围；用于删除的家长访问表面。

8. **效能研究。** 10 名学习者，前测（标准化 30 题基线），两周导师交互（每周 3 次），后测。与相同内容的非自适应基线群体（10 名学习者）进行比较。

9. **每周进度报告。** 每个学习者自动生成 PDF 摘要，包含探索的主题、掌握轨迹和推荐的下一步。

## 使用方式

```
学习者："我不明白为什么 3x + 6 = 12 意味着 x = 2"
[signal]   卡住
[concept]  '分离变量'（前置：加减等式）
[scaffold] "你会从两边减去什么数字来开始？"
学习者："6"
[signal]   正确
[mastery]  加减等式：0.62 -> 0.77
[concept]  继续 '分离变量'
[scaffold] "很好。那么 3x / 3 等于什么？"
```

## 练习

1. 运行带和不带自适应学习者模型的效能研究（随机概念顺序）。报告差异。预期自适应会胜出，但差距大小是有趣的数字。

2. 添加多模态探针：同一概念问题以文本、语音和图片形式交付。衡量学习者是否以他们偏好的模态更快收敛。

3. 构建家长仪表板：练习的主题、掌握轨迹、即将学习的概念、安全事件（任何防护栏触发）。COPPA 对齐。

4. 添加语言切换模式：导师接受西班牙语输入并用西班牙语教学。衡量 X-Guard 覆盖。

5. 压力测试记忆隐私：验证学习者 A 无法看到学习者 B 的数据，即使是语音片段重新摄入攻击。记录尝试访问并告警。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------------|------------------------|
| 苏格拉底式策略（Socratic policy） | "提问，不灌输" | 导师提出引导性问题而非给出答案 |
| 贝叶斯知识追踪（Bayesian knowledge tracing） | "BKT" | 每个概念掌握概率的经典学习者模型方程 |
| FSRS | "自由间隔重复调度器（Free Spaced Repetition Scheduler）" | 2024 年间隔重复调度器，优于 SM-2 |
| 课程图谱（Curriculum graph） | "概念 DAG" | 带有前置边的概念 Neo4j 图 |
| 情景记忆（Episodic memory） | "每次交互日志" | 每次交互存储以便后续检索 |
| 语义记忆（Semantic memory） | "学到的模式存储" | 从情景记忆提升的压缩错误和偏好 |
| COPPA | "儿童隐私法" | 美国法律，限制收集 13 岁以下儿童的数据 |
