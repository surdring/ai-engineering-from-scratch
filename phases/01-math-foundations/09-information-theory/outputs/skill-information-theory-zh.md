# 机器学习中的信息论

在机器学习系统中何时使用熵（Entropy）、交叉熵（Cross-Entropy）、KL 散度（KL Divergence）和互信息（Mutual Information）。

## 决策清单

1. 衡量单个分布的不确定性？使用**熵**。
2. 衡量模型对真实标签的逼近程度？使用**交叉熵**（这是你的分类损失函数）。
3. 衡量两个分布之间的距离？使用 **KL 散度**。
4. 检查两个变量是否相关？使用**互信息**。
5. 报告语言模型质量？使用**困惑度（Perplexity）**（交叉熵的指数）。
6. 将一个模型蒸馏到另一个模型？最小化教师到学生的 **KL 散度**。

## 各度量的适用场景

| 度量 | 公式 | 用例 | ML 应用 |
|---|---|---|---|
| 熵 H(P) | -sum(p log p) | 这个分布有多不确定？ | 数据复杂度、最大熵模型 |
| 交叉熵 H(P,Q) | -sum(p log q) | 模型 Q 对真实 P 的预测有多好？ | 分类损失、语言模型损失 |
| KL 散度 D(P\|\|Q) | sum(p log(p/q)) | P 和 Q 有多不同？ | VAE 损失（ELBO）、知识蒸馏、RLHF |
| 互信息 I(X;Y) | H(X) - H(X\|Y) | Y 能告诉我们关于 X 的多少信息？ | 特征选择、表征学习 |
| 困惑度 | exp(H(P,Q)) 或 2^H | 模型有多困惑？ | 语言模型评估 |
| 条件熵 H(X\|Y) | -sum(p(x,y) log p(x\|y)) | 知道 Y 后 X 的剩余不确定性 | 特征信息量 |

## 关键关系

```
交叉熵 = 熵 + KL 散度
H(P, Q) = H(P) + D_KL(P || Q)

训练时 H(P) 为常数，因此：
  最小化交叉熵 = 最小化 KL 散度

互信息 = 熵 - 条件熵
I(X; Y) = H(X) - H(X|Y) = H(Y) - H(Y|X)

困惑度 = exp(交叉熵（以 nat 为单位）)
       = 2^(交叉熵（以 bit 为单位）)
```

## 快速参考：公式和单位

| 公式 | 比特（log 底数 2） | 纳特（log 底数 e） |
|---|---|---|
| 信息量：-log(p) | -log2(p) | -ln(p) |
| 熵：-sum(p log p) | 比特 | 纳特 |
| 1 nat = | 1.4427 比特 | 1 纳特 |
| PyTorch 默认 | -- | 纳特 |
| 信息论论文 | 比特 | -- |

## 解读数值

| 熵值 | 含义 |
|---|---|
| 0 | 确定性。某个结果概率为 1。 |
| log(n) | 最大不确定性。n 个结果上的均匀分布。 |
| 低 | 分布集中。模型自信。 |
| 高 | 分布平坦。模型不确定。 |

| 困惑度值 | 语言模型质量 |
|---|---|
| 1 | 完美预测（实践中不可能） |
| 10 | 平均在约 10 个等可能 token 中选择 |
| 50 | 标准基准上的 GPT-2 水平 |
| < 10 | 对代表性好的领域的顶尖水平 |

## 常见错误

- 计算 KL 散度并将其视为对称的。D_KL(P||Q) != D_KL(Q||P)。如需对称度量，使用 Jensen-Shannon 散度：JS = 0.5 * KL(P||M) + 0.5 * KL(Q||M)，其中 M = 0.5*(P+Q)。
- 忘记 one-hot 标签下的交叉熵简化为 -log(p_true_class)。当真分布是 one-hot 时，不需要对所有类别求和。
- 代码中使用 log 底数 2 但报告纳特（或反之）。PyTorch 默认使用自然对数。乘以 log2(e) = 1.4427 将纳特转换为比特。
- 计算空事件或零概率事件的熵。约定：0 * log(0) = 0，因为 lim(p->0) p*log(p) = 0。
- 在不同词汇表之间比较困惑度。词汇量 50k 且困惑度 30 的模型与词汇量 10k 且困惑度 30 的模型没有直接可比性。

## 各概念在生产 ML 中的出现位置

| 概念 | 出现位置 |
|---|---|
| 交叉熵损失 | 每个分类模型（nn.CrossEntropyLoss） |
| KL 散度 | VAE ELBO、PPO 裁剪、知识蒸馏 |
| 熵正则化 | RL 中的探索奖励（更高熵 = 更多探索） |
| 互信息 | 特征选择、InfoNCE 损失（对比学习） |
| 困惑度 | 语言模型基准（越低越好） |
| 标签平滑 | 用软目标替代 one-hot，减少交叉熵的过置信 |
| 温度缩放 | 在 softmax 前将 logits 除以 T，控制输出熵 |
