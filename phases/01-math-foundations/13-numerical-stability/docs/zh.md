## 问题

你的模型训练了三小时，然后损失变成 NaN。你加了一行打印语句。logit 在第 9000 步还好，在第 9001 步变成 inf。到第 9002 步每个梯度都是 nan，训练死掉了。

或者你的模型训练完成了，但准确率比论文声称的低 2%。你检查了一切。架构匹配。超参数匹配。数据匹配。问题是论文用的是 float32 而你用的是 float16 却没有正确缩放。

数值稳定性不是理论问题。它决定一次训练运行能否成功。你调试的每个严重的 ML bug 最终都会归结为浮点问题。

## 概念

### IEEE 754：计算机如何存储实数

float32：1 位符号 + 8 位指数 + 23 位尾数。约 7 位十进制精度。最大值约 3.4e38。

float16：5 位指数 + 10 位尾数。约 3 位精度。最大值仅 65,504，对 ML 来说小得令人不安。

bfloat16：Google 的答案——与 float32 相同的 8 位指数（相同范围，高达 3.4e38）但仅 7 位尾数（精度更低）。对训练神经网络，范围比精度更重要。

### 为什么 0.1 + 0.2 != 0.3

0.1 在二进制中无法精确表示——它是一个无限循环小数。永远不要用 `==` 比较浮点数。使用 `abs(a-b) < epsilon` 或 `math.isclose()`。

### 灾难性消去

当你减去两个几乎相等的浮点数时，有效数字相互抵消，剩下的是舍入噪声被提升为前导数字。一次减法就能产生 19% 的相对误差。修复方法：重新排列公式以避免减去大而几乎相等的数字。方差计算使用 Welford 算法。对数概率在整个过程中使用 log 空间。

### 上溢和下溢

上溢：结果太大无法表示。下溢：结果太小（比最小可表示正数更接近零）。

`exp()` 函数是 ML 中上溢的主要来源：exp(88.7) 刚好适合 float32，exp(89.0) = inf（上溢）。`log()` 函数走另一个方向：log(0.0) = -inf。

### Log-Sum-Exp 技巧

直接计算 `log(sum(exp(x_i)))` 数值上很危险。技巧：在指数化之前减去最大值。

```
log(sum(exp(x_i))) = max(x) + log(sum(exp(x_i - max(x))))
```

减去 max(x) 后，最大的指数是 exp(0) = 1。不可能上溢。和中至少有一项是 1，所以 log >= 0。不可能下溢到 -inf。

### 为什么 softmax 需要最大值减法技巧

无技巧：exp(100) = inf（float32）。有技巧：减去 max=102，exp(-2)=0.135, exp(-1)=0.368, exp(0)=1.0。完全是数值稳定的。

### Log-Softmax 和交叉熵

PyTorch 的 `CrossEntropyLoss` 在内部使用 log-softmax——它永远不先计算完整的 softmax 概率。log_softmax(x_i) = x_i - max(x) - log(sum(exp(x_j - max(x))))。这对梯度同样重要：当概率接近零时，softmax 梯度趋近于零，而 log-softmax 梯度保持稳定。

### 混合精度训练

在前向和反向传播中使用 float16 以获得速度和内存（2x），但在更新中使用 float32 以保持精度。梯度累积在 float16 中，所以梯度值 < 2^-24 会下溢到零——损失缩放通过将损失乘以一个大常数来解决这个问题。

### 梯度检查

验证解析梯度与数值梯度的标准方法：中心有限差分 `f(x+h) - f(x-h) / (2h)`。h 太小会导致灾难性消去，h 太大会产生截断误差。经验法则：h = 1e-5 用于 float32，h = 1e-3 用于 float16。

## 关键术语

| 术语 | 人们常说的 | 实际含义 |
|------|-----------|---------|
| 上溢 | "变成无穷大" | 数值超过最大可表示值，变成 inf |
| 下溢 | "变成零" | 数值变得太小无法分辨，变成 0 |
| 灾难性消去 | "精度丢失" | 减去几乎相等的值，丢失所有有效数字 |
| log-sum-exp | "稳定化技巧" | 在指数化前减去最大值，防止上溢/下溢 |
| 混合精度 | "float16 + float32" | 在 float16 中训练大部分，主权重保持在 float32 |
