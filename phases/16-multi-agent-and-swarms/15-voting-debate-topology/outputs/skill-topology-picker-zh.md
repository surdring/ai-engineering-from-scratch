给定一个任务描述，推荐一个多智能体拓扑和规模。

生成：

1. **任务指纹。** 研究型（长时间跨度、开放式）、快速事实型（封闭形式答案）、逐步细化（分阶段流水线）或意见型（无真实标签）。选择一个；如果跨越两种，选择主导形态。
2. **拓扑。** 星形（star）、链形（chain）、树形（tree）或图形（graph）。根据指纹论证：
   - 研究型 → 图形（任意对任意批评）
   - 快速事实型 → 星形（中心聚合）
   - 逐步细化 → 链形（或分治时用树形）
   - 意见型 → 以上皆非；推荐单智能体 + 人类决策
3. **智能体数量 N。** 3 是最便宜的有效集成；5 是常见的甜点；7+ 是特殊用途。图形拓扑上超过 5 时，警告协调税（coordination tax）。
4. **异质性画像。** 如果单一文化很重要（研究、推理），至少一个智能体必须来自不同的基础模型家族。在 N=5 时优先使用 3 种不同的基础模型。
5. **轮次上限。** 1 轮 = 投票。2 轮 = 一次细化。3 轮 = 从众主导前的最大值。从不无界。
6. **聚合。** 票数（廉价）、置信度加权（第 14 课的 CP-WBFT）、几何中位（DecentLLMs）或评判打分。默认置信度加权，除非成本约束要求票数。
7. **升级。** 低于阈值共识 → 升级到哪里？人类、另一个使用不同基础模型的集成，还是弃权？

硬性拒绝：

- 任何在图形拓扑上推荐 10+ 智能体的方案。协调税主导；先测量。
- 对开放研究问题使用星形拓扑。星形失去了任意对任意批评的收益。
- 任何将同一基础模型运行 N 次并称之为多智能体的建议。这是伪装的 self-consistency；正确标注。
- 无界轮次。奖励从众；辩论运行时间越长，智能体越因压力而非逻辑达成一致。

拒绝规则：

- 如果任务没有真实标签（意见、综合、创意），声明投票是咨询性的。推荐单智能体 + 人类决策。
- 如果用户无法访问多个基础模型，标记单一文化上限并推荐带温度变化的 self-consistency 作为后备。
- 如果任务简单（单次事实查找，< 100 个推理令牌），推荐带 self-consistency N=5 的单智能体。

输出：一页简要。以一句话建议开头（「图形拓扑，N=5 个来自 3 种不同基础模型的智能体，2 轮，置信度加权聚合，低于阈值升级给人类。」），然后是上述七节。以预算估算结尾：每查询预期令牌数和预期延迟（秒）。
