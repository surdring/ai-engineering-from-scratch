# 投票、自一致性与辩论拓扑

> 最便宜的聚合：采样 N 个独立代理，多数投票。Wang 等人 2022 年的自一致性（Self-Consistency）用一个模型采样 N 次做到了。多 Agent 通过**异构（Heterogeneous）**代理将其扩展以摆脱单文化——不同模型、不同提示、不同温度、不同上下文。超越多数投票，辩论拓扑很重要：MultiAgentBench（arXiv:2503.01935，ACL 2025）评估了星形 / 链形 / 树形 / 图形协调，发现**图形对研究最佳**，约 4 个代理后出现「协调税（Coordination Tax）」。AgentVerse（ICLR 2024）记录了两种涌现模式——志愿者行为（Volunteer Behavior）和顺从行为（Conformity Behavior）——顺从既是功能（找到共识）也是风险（群体思维，第 24 课）。本课绘制拓扑空间，构建每种变体，并测量协调税。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置知识：** 第 16 阶段 · 07（心智社会与辩论）、第 16 阶段 · 14（共识与 BFT）
**时间：** 约 75 分钟

## 问题

辩论可以提高准确率（Du 等人，arXiv:2305.14325）。它也可以降低准确率。辩论是否有帮助取决于四个结构性选择：

1. 谁和谁说话（拓扑）。
2. 多少轮（Du 2023：轮数和代理数都有独立贡献）。
3. 代理是否异构（不同基础模型打破单文化）。
4. 是否存在对抗性声音（钢人论证 vs 稻草人论证）。

那些直接接上「运行 5 个代理并投票」的团队经常会比单个代理退步。失败不是随机的。它们追踪拓扑和异构性。本课是拓扑地图。

## 概念

### 自一致性，单模型基线

Wang 等人 2022 年（「Self-Consistency Improves Chain of Thought Reasoning」）在 temperature > 0 下对同一模型采样 N 次，并对推理路径答案进行多数投票。GSM8K 上的结果：N=40 次采样相比单次贪婪解码有大幅增益。自一致性是多 Agent 投票的单 Agent 前身。

局限：自一致性使用一个基础模型。误差按构造相关。如果模型有系统性偏见，所有 N 个样本共享它。

### 多 Agent 投票，异构扩展

将 N 个样本替换为 N 个*不同*的代理。不同的基础模型（Claude、GPT、Llama）、不同的提示、不同的工具访问。好处：不相关误差。成本：不同代理成本不同；协调它们增加了开销。

2026 年异构辩论的规范名称是 **A-HMAD**——对抗性异构多 Agent 辩论（Adversarial Heterogeneous Multi-Agent Debate）。虽非普遍采用，但论文使用该术语表示「不同模型辩论，减少来自单文化崩溃的相关误差」。

### 四种拓扑

```
star           chain         tree           graph

    ┌─A─┐      A─B─C─D    ┌──A──┐         A───B
    │   │                   │     │         │ × │
    B   C                   B     C         D───C
    │   │                  / \   / \
    D   E                 D   E F   G      （全连接）
```

星形（Star）：一个枢纽，所有其他只与枢纽对话。等同于没有反向通道的监督者-工作者。
链形（Chain）：线性，每个代理看到前一个的输出。类似管道。
树形（Tree）：层次化，被层次化代理系统使用（第 06 课）。
图形（Graph）：任意到任意。包含全连接团和任意 DAG。

### 协调税（MultiAgentBench）

MultiAgentBench（MARBLE，ACL 2025，arXiv:2503.01935）在包括研究、编码和规划的任务套件上对星形、链形、树形、图形进行了基准测试。关键测量结果：

- **图形**拓扑在研究任务上获胜。信息任意到任意流动；代理可以相互批判。
- **星形**在快速答案事实性任务上获胜。枢纽过滤和整合。
- **链形**在逐步管道（分阶段细化）上获胜。
- **协调税**在图形拓扑中约 4 个代理后出现。墙钟和 token 成本增长快于质量。

4 个代理的上限是经验的，而非根本的。它反映了 2026 年 LLM 上下文容量：每个代理的上下文被同伴输出填满，一旦每个人都能看到每个人，添加代理 N+1 的边际价值就下降了。

### 多 Agent 辩论策略（「Should we be going MAD?」）

arXiv:2311.17371 是 2023 年 MAD 策略综述。被他人复现的关键发现：与自一致性*结构相似*的 MAD 变体（独立采样 + 聚合）在使用相同预算时通常表现不如自一致性。当代理真正异构且辩论具有对抗性结构（一个代理反对）时，MAD 帮助最大。

### AgentVerse 涌现模式

AgentVerse（ICLR 2024，https://proceedings.iclr.cc/paper_files/paper/2024/file/578e65cdee35d00c708d4c64bce32971-Paper-Conference.pdf）记录了从多 Agent 辩论中即使没有显式设计也涌现出的两种行为：

- **志愿者（Volunteer）。** 代理在未被提示的情况下主动提供帮助（「我可以采取下一步」）。有用于：它将工作分配给对子任务最有能力的代理。
- **顺从（Conformity）。** 代理调整其立场以匹配批评者，即使批评者是错的。这是谄媚的辩论等价物（第 14 课）。

顺从是为什么「辩论直到一致」奖励欺凌者。有界轮次和单独裁判可缓解。

### 异构性：实际移动准确率的关键旋钮

2024-2026 年实践文献中的一个模式：将你的 N 个代理中的一个替换为不同的基础模型，比将 N 增加 1 带来更大的准确率提升。直觉是单文化——每个新的独立误差源比一个额外的相关样本更有价值。

在极限情况下，异构性胜过数量性。在有清晰基准真相的大多数任务上，三个不同模型胜过五个相同模型的副本。

### 陪审团方法

Sibyl 框架（在 Minsky-LLM 文献中被引用）形式化了一个「陪审团」——一组小型专业化代理，在每个阶段通过投票细化答案。与普通多数投票不同，陪审团有角色：一个代理交叉审问，一个提供上下文，一个评分可信度。陪审团方法是普通投票（廉价、易单文化）和完整 MAD（昂贵、易顺从）之间的中间地带。

### 什么时候投票加辩论占优

- 问题有基准真相（事实、数学、代码行为）。投票收敛有意义。
- 代理可以访问不同来源或工具（异构性可用）。
- 轮次有界（通常 2-3），且有单独裁判或验证者。
- 预算允许 3-5 个代理。在图形拓扑上超过 5-7 个，协调税占主导。

### 什么时候投票加辩论有害

- 问题以意见为形状。代理收敛到看起来最自信的答案，而非最正确的。
- 所有代理共享一个基础模型。单文化使共识毫无意义。
- 轮次无界。顺从每次都赢。
- 任务简单。一个带 N=5 自一致性的代理更便宜且同样准确。

## 构建

`code/main.py` 实现：

- `run_star(agents, hub, question)`——枢纽轮询每个工作者、聚合。
- `run_chain(agents, question)`——顺序细化。
- `run_tree(root, children, question)`——带深度-2 聚合的层次化。
- `run_graph(agents, question, rounds)`——全对全辩论、有界轮次。
- 一个脚本化的异构性调节旋钮：每个代理有一个 `error_bias` 表示其系统性错误。
- 一个测量工具，在 N=3、5、7 时运行每种拓扑并报告（准确率、总 token、模拟墙钟）。

运行：

```
python3 code/main.py
```

预期输出：拓扑 × N →（准确率、token、延迟）表格。图形在研究风格任务上 N=3-5 时获胜；星形在快速事实性任务上获胜；N=7 时图形显示协调税（延迟比准确率膨胀得更快）。

## 实践

`outputs/skill-topology-picker.md` 是一个技能，读取任务描述并推荐拓扑（星形 / 链形 / 树形 / 图形）、N（代理数量）、异构性配置（使用哪些基础模型）以及轮次上限。

## 交付

对于任何集成：

- 从一个强基础模型的 **N=5 自一致性**开始。它是廉价的基线。
- 如果准确率重要，升级到 **N=3 异构投票**。测量增量。
- 只有当任务有结构（研究、多步骤）且有界轮次可行时，才升级到**辩论拓扑**。
- 始终记录少数簇。当少数持续正确时，你有了多样性信号。
- 将墙钟和 token 与准确率一起基准测试。「以 10 倍成本获得更好准确率」是商业决策。

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|------|---------|---------|
| Self-consistency | 「采样 N 次，投票」 | Wang 2022。单模型，N 次 temperature>0 采样，对推理路径多数投票。 |
| Heterogeneity | 「不同模型」 | 不同基础模型或提示族的集成。打破单文化。 |
| MAD | 「多 Agent 辩论」 | 代理在多轮中交换批判的通用术语。见 Du 2023。 |
| A-HMAD | 「对抗性异构 MAD」 | 强调不同模型 + 对抗性结构的 MAD 变体。 |
| Topology | 「谁和谁说话」 | 星形、链形、树形、图形。决定信息流。 |
| Coordination tax | 「递减回报」 | 图形拓扑上约 4 个代理后，成本增长快于质量。 |
| Volunteer behavior | 「未被提示的帮助」 | AgentVerse 涌现模式：代理主动提出采取步骤。 |
| Conformity behavior | 「压力下的同意」 | AgentVerse 涌现模式：代理与批评者对齐。 |
| Jury | 「小型专业小组」 | Sibyl 风格集成，带角色（审问者、提供上下文者、评分者）。 |
