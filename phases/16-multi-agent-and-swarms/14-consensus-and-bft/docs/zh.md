# 共识与面向 Agent 的拜占庭容错

> 经典分布式系统 BFT（Byzantine Fault Tolerance）遇上随机 LLM。2025-2026 年出现了三个研究方向：**CP-WBFT**（arXiv:2511.10400）通过置信度探测（Confidence Probe）对每个投票加权；**DecentLLMs**（arXiv:2507.14928）采用无领导者（Leaderless）并行工作者提议和几何中位数（Geometric Median）聚合；**WBFT**（arXiv:2505.05103）将加权投票与层次结构聚类（Hierarchical Structure Clustering）结合，以拆分核心（Core）和边缘（Edge）节点。来自「Can AI Agents Agree?」（arXiv:2603.01213）的诚实实证结果是，即使是标量（Scalar）协议在今天也是脆弱的——一个欺骗性代理就可以破坏混合代理（Mixture-of-Agents）。BFT 是必要的但不充分。本课构建一个最小 BFT 协议，注入三种代理特定的攻击（拜占庭谎言、谄媚顺从、相关误差单文化），并测量每种共识变体如何应对。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置知识：** 第 16 阶段 · 07（心智社会与辩论）、第 16 阶段 · 13（共享记忆）
**时间：** 约 75 分钟

## 问题

你有 N 个 LLM 代理，每个产生一个答案。它们意见分歧。多数投票选出了错误的一方，因为两个代理是相关的（相同基础模型、相同训练数据、相同失败模式）。第三个代理恰好以新颖的方式错了——所以多数是虚假的多数。

现在添加一个欺骗性代理：它故意撒谎。或者一个谄媚代理：它同意最后说话的人。在经典 BFT 中，假设拜占庭节点是少数 `f < n/3` 并任意行为。2026 年的现实是，LLM 节点即使是诚实的也是随机的，跨模型相关，并被彼此的输出影响。你不能将它们视为独立的伯努利（Bernoulli）投票者。

经典 BFT（PBFT，1999）没有错——它是不完整的。它处理任意位翻转。它不处理「三个诚实的代理因为共享训练数据而共享一个幻觉」。本课从 PBFT 的基础构建，并层叠三种 2025-2026 年的适应。

## 概念

### 经典 BFT 给你什么

实用拜占庭容错（Practical Byzantine Fault Tolerance，Castro & Liskov，OSDI 1999）容忍 `f < n/3` 个拜占庭节点。协议有三个阶段（pre-prepare、prepare、commit）和两个原语（签名消息、法定人数证书）。在 `n >= 3f + 1` 个诚实或恶意节点中达成单一值的一致。

保证很强，但假设：

1. **独立故障。** 拜占庭方不协调。
2. **诚实节点真正诚实。** 诚实输出的正确性不是问题；协议只解决分歧。
3. **问题有基准真相答案。** 对一个错误事实达成共识仍然是共识。

LLM 代理违反所有三条。运行相同基础模型的两个代理共享故障。「诚实」的 LLM 仍然会幻觉。对模棱两可的问题，「真相」是代理决定的——不存在外部预言机。

### 三种 LLM 特定的攻击

**拜占庭谎言。** 一个代理输出故意错误答案。经典 BFT 在 `f < n/3` 时处理。

**谄媚顺从。** 一个代理在投票前读取其他代理的答案，并与最后发言的人对齐。非恶意，但与最响亮的声音相关。经典 BFT 不防止此，因为代理通过了所有签名检查。

**相关误差单文化（Monoculture）。** 三个代理共享一个基础模型。它们对同一个错误答案产生幻觉。多数是错误的。经典 BFT 不帮助，因为三个代理都「诚实」地同意了。

### 2025-2026 年回应

**CP-WBFT**（arXiv:2511.10400）——置信度探测加权 BFT（Confidence-Probed Weighted BFT）。每个投票者在其答案上附加一个置信度探测（自我报告的概率，或单独校准模型的预测）。投票权重随置信度缩放。报告在完全图上 BFT 改进 +85.71%。缓解：谄媚顺从（顺从代理对它们主动采取的立场往往有低置信度）。

**DecentLLMs**（arXiv:2507.14928）——无领导者。工作者代理并行提议，评估者代理评分提案，最终答案是评分位置的几何中位数。在 `f < n/2` 时鲁棒。缓解：拜占庭谎言和相关误差（几何中位数对异常值鲁棒，并拉向密集簇，而非模型偏见的平均）。

**WBFT**（arXiv:2505.05103）——带层次结构聚类的加权 BFT（Weighted BFT with Hierarchical Structure Clustering）。投票权重由响应质量加上从历史学习的信任分数分配。将代理聚类为核心和边缘；核心代理必须首先达成共识，边缘代理跟随。缓解：可扩展性（核心共识小而快）和部分单文化（核心可以针对多样性选择）。

### 实证：「Can AI Agents Agree?」（arXiv:2603.01213）

该论文在多个前沿模型上测量了标量协议（LLM 代理对单个数值达成一致）。发现令人不安：

- 即使没有对手，LLM 代理在许多基准上对标量问题的分歧率超过 30%。
- 一个采用欺骗性角色（Deceptive Persona）的代理可以将混合代理共识从诚实基线拉开 40+ 个百分点。
- 分歧率与模型多样性相关——异构集成比同质的更分歧（好：不相关误差），但收敛也更慢（坏：更长的一致时间）。

要点：BFT 给你对齐输出的机制，但它不告诉你对齐的输出是否正确。结合验证（第 16 阶段 · 08 角色专业化）、多样性（第 16 阶段 · 15 辩论变体）和评估者代理（第 16 阶段 · 24 基准）。

### 核心协议，简化版

LLM 代理的最小 BFT 轮次：

```
1. 任务到达；每个代理 i 产生答案 a_i
2. 每个代理附加置信度探测 c_i，范围 [0, 1]
3. 聚合器从所有 n 个代理收集 (a_i, c_i)
4. 聚合器按语义簇分组（等价答案）
5. 聚合器为每个簇 C 计算权重：
     w(C) = sum_{i in C} c_i
6. 胜者 = 权重最大的簇，如果 max > threshold * sum(c_i)
   否则：重试或升级
7. 少数簇带来源追溯记录，供事后审计
```

语义聚类步骤是 LLM 特定的特点。两个答案「研究报告 4.2%」和「4.2% 改进」是同一个簇。朴素的字符串相等检查会漏掉它。在生产中，使用廉价的嵌入模型或显式规范化。

### 阈值调优

`threshold` 参数决定何时接受、何时重试。太低：你接受弱多数。太高：你从不接受任何东西。经验范围：`n=5-7` 个代理时 0.5-0.67，更小 `n` 时更高。低于阈值，升级给人类或不同的代理集成。

### 共识不帮助的地方

- **模棱两可的问题。** 如果问题没有基准真相，共识是意见。如实说。
- **复合问题。** 「写代码并解释它」——两个答案。对每个独立投票。
- **多轮对抗。** 如果代理可以观察前几轮并模仿（Du 2023 辩论），它们开始彼此同意而不顾真相。限制轮次（通常 2-3）。

## 构建

`code/main.py` 实现：

- `AgentVoter`——带（答案、置信度）的脚本化策略。
- `MajorityVote`——经典多数票。
- `CPWBFT`——带语义聚类的置信度加权投票。
- `DecentLLMs`——对评分提案的几何中位数聚合。
- `Scenario`——在三种攻击模式下运行每种聚合器。

已实现的攻击模式：

1. `byzantine`：一个代理以高置信度撒谎。
2. `sycophancy`：一个代理复制它看到的第一个答案，带有匹配的置信度。
3. `monoculture`：三个代理共享一个错误答案（相关误差），带中等置信度。

运行：

```
python3 code/main.py
```

预期输出：（攻击、聚合器）→ 最终答案的表格，正确答案高亮。多数票在单文化案例中失败。CPWBFT 的置信度加权缓解了谄媚。当单文化少于一半人口时，DecentLLMs 的几何中位数拉向诚实簇。

## 交付

在部署任何共识机制之前：

- **至少用上述三种模式进行攻击测试。** 你的协议应可预测地失败，而非悄无声息。
- **带来源追溯记录每个少数簇。** 少数簇是你对相关误差的预警系统。
- **强制有界轮次。** 不要「继续辩论直到一致」——那奖励谄媚。
- **将一致性与正确性分开。** 共识输出交给验证者；验证者独立于集成。
- **监控一致率。** 急剧上升意味着顺从偏见；急剧下降意味着模型漂移。

## 练习

1. 运行 `code/main.py`。确认多数票在单文化攻击下失败，但 CPWBFT 在单文化置信度低于 0.7 时部分缓解。
2. 添加第四种攻击模式：**沉默弃权**——一个代理拒绝回答（「我不知道」）。每种聚合器应如何处理弃权？实现你的选择。
3. 将语义聚类从字符串规范化切换为嵌入相似度（使用任何开源嵌入模型）。谄媚攻击会怎样？
4. 阅读 CP-WBFT（arXiv:2511.10400）。实现置信度探测校准步骤（一个单独的校准模型检查每个代理自我报告的置信度）。测量单文化场景上的准确率增益。
5. 阅读「Can AI Agents Agree?」（arXiv:2603.01213）。复现一个简化的标量协议实验：三个代理，一个标量问题，欺骗性角色提示。CPWBFT 或 DecentLLMs 能捕获它吗？

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|------|---------|---------|
| BFT | 「拜占庭容错」 | Castro-Liskov 1999 协议，用于 `f < n/3` 任意故障的共识。 |
| Byzantine | 「任何坏行为」 | 一个可以撒谎、丢弃消息、静默失败的节点——除了安全崩溃之外的任何行为。 |
| Confidence probe | 「你有多确定？」 | 附加到投票上的自我报告或校准器预测的概率。 |
| Semantic clustering | 「相同答案，不同措辞」 | 在计票前将等价答案分组。 |
| Geometric median | 「鲁棒中心」 | 最小化到采样点距离和的点。对异常值鲁棒，与均值不同。 |
| Monoculture | 「相同模型，相同失败」 | 代理共享训练数据或基础模型时的相关误差。 |
| Sycophantic conformity | 「同意响亮的声音」 | 代理的投票偏向首先/最响亮发言的人。 |
| Core/Edge | 「层次化 BFT」 | WBFT 拆分：小核心先达成共识，边缘节点跟随。限制延迟。 |
