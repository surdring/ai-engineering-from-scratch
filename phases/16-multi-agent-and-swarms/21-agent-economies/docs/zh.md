# 代理经济、Token 激励与声誉

> 长周期自主代理（Long-horizon autonomous agents）（METR 的 1 小时到 8 小时工作曲线）需要经济代理能力。正在兴起的 **5 层架构栈** 是：**DePIN**（去中心化物理基础设施，物理计算）→ **身份（Identity）**（W3C DID + 声誉资本（Reputation Capital））→ **认知（Cognition）**（RAG + MCP）→ **结算（Settlement）**（账户抽象（Account Abstraction））→ **治理（Governance）**（代理 DAO（Agentic DAOs））。生产级代理激励网络包括 **Bittensor**（TAO 子网奖励特定任务模型）、**Fetch.ai / ASI 联盟**（ASI-1 Mini LLM + FET token）和 **Gonka**（基于 Transformer 的工作量证明，将计算资源重新分配给生产性 AI 任务）。学术方面：AAMAS 2025 的去中心化 LaMAS 使用 **Shapley 值（Shapley value）信用归因**来公平奖励贡献代理；Google Research 的"大语言模型机制设计"提出了单调聚合下的**Token 拍卖**与二阶价格支付。本课构建一个最小化代理市场，将 Shapley 值信用归因应用于多代理流水线，并运行一个二阶价格 Token 拍卖，使博弈论机制具体落地。

**类型：** 学习
**语言：** Python（标准库）
**前置条件：** Phase 16 · 16（谈判与讨价还价）、Phase 16 · 09（并行集群网络）
**时间：** ~75 分钟

## 问题

当代理联合产生价值但需要分别获得奖励时，多代理系统变得复杂。经典机制——均等分配、最后贡献者通吃——不公平或可被博弈。通过 Shapley 值的联盟奖励在构造上是公平的，但计算成本高昂。2025-2026 年的文献推动了实用的近似方法：Shapley 采样、单调聚合拍卖以及通过已确认贡献累积的链上声誉。

除了信用归因，该领域已转向实际的经济代理：Bittensor TAO 奖励挖掘计算以微调子网专用模型，Fetch.ai/ASI 用 FET Token 奖励 ASI-1 Mini LLM 使用，Gonka 将 Transformer 工作量证明重新分配给生产性 AI 任务。自主交易的代理今天已存在；问题在于如何对齐激励。

本课将代理经济视为一个特定的问题族——信用归因、机制设计（Mechanism Design）和声誉——并用最少的数学构建每个部分，使概念深入人心。

## 概念

### 5 层代理经济栈

1. **DePIN（物理计算）。** 出租 GPU、存储、带宽的去中心化基础设施。Bittensor 子网、Render Network、Akash。不特定于代理；代理使用它们。
2. **身份。** W3C 去中心化标识符（Decentralized Identifiers，DID）为每个代理提供独立于任何平台的持久 ID。声誉累积到 DID 上。代理网络协议（ANP）使用 DID 作为发现层。
3. **认知。** 代理的推理循环：LLM + RAG + MCP。这是其他阶段构建的内容。
4. **结算。** 账户抽象（ERC-4337）允许代理从自己的余额中支付 gas 费，而无需持有 ETH。代理可以为服务、彼此或计算付费。
5. **治理。** 代理 DAO：人类*和*代理对协议变更进行投票的治理结构，投票权与声誉挂钩。

并非每个生产系统都使用全部五层。Bittensor 使用 1、2、部分 3、部分 4，不使用 5。OpenAI 代理除 3 之外不使用任何层。这个栈是一个参考地图，而非要求。

### Bittensor、Fetch.ai、Gonka——实际运行情况

**Bittensor（TAO）。** 子网是专门化任务（语言建模、图像生成、预测）。矿工提交模型输出。验证者对其进行排名；基于质押权重的评分分配 TAO 奖励。每个子网有自己的评估方式。经济启示：按特定任务输出质量付费，而非按使用的计算量付费。

**Fetch.ai / ASI 联盟。** ASI-1 Mini LLM 在 Fetch.ai 网络上运行；用户用 FET Token 支付推理费用。这里代理作为对等体的叙事更强：Fetch 上的一个代理可以调用另一个代理执行任务并用 FET 支付。

**Gonka。** Transformer 工作量证明：其"工作"是 Transformer 的前向传播。矿工通过运行具有已知正确输出（来自训练数据）的推理任务来获得收益。资源生产性的 PoW 而非基于哈希的 PoW。

截至 2026 年 4 月，这三者都是生产级的。收益分配方式不同。Bittensor 按相对于子网验证者的质量奖励；Fetch 按付费用户衡量的效用奖励；Gonka 按可验证的推理工作奖励。

### Shapley 值信用归因

三个代理在一个任务上协作。输出得分为 0.8。谁的贡献是多少？

Shapley 值：满足四个公理（效率性（Efficiency）、对称性（Symmetry）、线性性（Linearity）、空性（Null））的唯一信用分配。对于代理 `i`：

```
shapley(i) = (1/N!) * sum over all orderings O of (v(S_i_O ∪ {i}) - v(S_i_O))
```

其中 `S_i_O` 是在排序 `O` 中 `i` 之前的代理集合。实践中：枚举所有排列，记录每个代理在每个排列中的边际贡献，取平均。

对于 N=3 个代理，有 6 种排列。对于 N=10，有 360 万种——因此实践中采样排序而非枚举。

### 聚合的二阶价格拍卖

Google Research（"大语言模型机制设计"）提出了用于聚合 LLM 输出的二阶价格 Token 拍卖。设置：N 个代理各自提出一个补全；每个代理对被选中有一个私有价值。拍卖者选择价值最高的提案，并支付*第二高*价值。在单调聚合（价值取决于哪个提案被选中，而非有多少提案被出价）下，这是诚实的——代理出价其真实价值。

这对 LLM 系统为何重要：你可以将补全任务外包给具有不同定价的多个代理；拍卖选出最佳并公平支付，代理没有动机虚报。

### 声誉资本

绑定到 DID 的声誉分数从已确认的贡献中累积。一个简单的更新规则：

```
rep(i, t+1) = alpha * rep(i, t) + (1 - alpha) * contribution_quality(i, t)
```

衰减因子 `alpha` 接近 1。声誉：

- 读取成本低，用于路由决策（"将困难任务发送给高声誉代理"）。
- 伪造成本高（随时间累积，绑定到 DID）。
- 可以被削减（Slash）：未通过验证的贡献会扣分。

### AAMAS 2025 去中心化 LaMAS

LaMAS 提案（AAMAS 2025）结合了：DID 身份、Shapley 值信用归因和一个简单的拍卖机制。核心主张：去中心化信用归因步骤使系统可审计并免疫单点操纵。

### 经济学在何处失效

- **价格预言机（Price Oracle）操纵。** 如果信用函数可被博弈，代理就会博弈它。每个机制都需要对抗性测试。
- **女巫攻击（Sybil Attacks）。** 一个操作者生成 N 个虚假代理来夸大自己的贡献。DID 减缓但不能阻止这一点；声誉的伪造成本是缓解措施。
- **验证成本。** 信用归因的公平性取决于验证者。如果验证成本低（小型 LLM），则可被博弈；如果成本高（人类评审组），系统无法扩展。
- **监管悬置。** 代理经济与金融监管交叉。截至 2026 年，Bittensor、Fetch 和 Gonka 在某些司法管辖区都处于法律灰色地带。

### 代理经济何时有意义

- **具有异构操作者的开放网络。** 没有单一团队控制所有代理。
- **可验证的输出。** 没有验证，信用归因只是猜测。
- **长周期工作流。** 一次性任务不会从声誉累积中受益。
- **Token 化支付在您所在司法管辖区法律上可行。**

在封闭的企业系统中，经济学让位于更简单的分配（管理者分配工作，指标是内部的）。经济学文献主要适用于开放网络。

## 交付它

2026 年运行代理经济：

- **从声誉开始，而非 Token。** 声誉实现成本低，且本身就有价值；Token 增加了法律和经济复杂性。
- **奖励之前先验证。** 没有独立验证步骤，绝不分配信用。自报质量会累积女巫博弈。
- **Shapley 采样，而非 Shapley 精确。** 采样 100-1000 种排序；精确枚举无法扩展。
- **限制衰减因子和声誉下限。** 无界衰减会抹去合法贡献者；衰减过慢会奖励过时的高声誉代理。
- **对抗性审计机制。** 在开放网络之前运行红队（Red-team）场景。每个机制都有博弈论；你要找到漏洞，而非让攻击者找到。

## 练习

1. 运行 `code/main.py`。确认 Shapley 值之和等于总价值（效率性公理）。更改价值函数；Shapley 分配是否以预期方向变化？
2. 实现 Shapley *采样*（对 K 种排序的蒙特卡洛）。K 如何影响近似精度？与 N=4 的精确计算对比。
3. 在拍卖前实现一个联盟形成步骤：代理可以合并为团队并以一个单元出价。哪些联盟会形成？结果是否比单独出价更优（Pareto-better）？
4. 阅读 Google Research 的机制设计文章。找出一个假设，如果违反，会破坏诚实性。该失败模式在 LLM 环境中是什么样子？
5. 阅读 AAMAS 2025 去中心化 LaMAS 论文。在合成任务上对 10 个代理实现其 Shapley 步骤。精确计算需要多长时间？100 次采样的近似程度如何？
