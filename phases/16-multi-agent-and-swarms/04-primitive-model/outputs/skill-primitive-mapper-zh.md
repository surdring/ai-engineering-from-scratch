给定一个多智能体框架（或使用该框架的代码库），生成四原语映射，使读者能在一段话中理解该框架。

生成：

1. **智能体定义。** 如何构造一个智能体？什么参数？它携带什么状态？指出确切的类或工厂。
2. **交接机制。** 使用三种交接模式中的哪一种——函数返回、图边或发言人选择？如果是混合的，哪个是主要的？展示触发一次交接的最小代码。
3. **共享状态模型。** 完整消息池还是投影视图？内存还是持久（检查点）？对并发写入者是否线程安全？谁协调冲突？
4. **编排器类型。** 静态、LLM 选择、交接驱动还是队列驱动？如果 LLM 选择，默认哪个模型？如果静态，图是循环还是 DAG？
5. **跨轴权衡。** 每项一句话：确定性、可扩展性上限、可调试性、典型失败模式。

硬性拒绝：

- 任何声称某抽象是「新」的但未证明其不能归约为四个原语之一的映射。如果你不能归约，精确指出差距，而非发明第五个原语。
- 仅引用营销文档的框架比较。始终引用框架仓库或官方 cookbook 中的具体代码示例。
- 像「框架 X 对智能体更好」这样的陈述，未说明框架优化的是哪个原语。

拒绝规则：

- 如果框架是闭源的且公共文档未暴露智能体-交接-状态-编排器面，声明没有内部信息无法映射。
- 如果用户提供代码库但没有框架（手工构建的智能体），改为映射自定义实现，并标记哪个原语设计不足。
- 如果框架早于 2024 年（原始 AutoGen v0.2、pre-Swarm）且不再维护，包含一行注释说明其后继者是否保留了该映射。

输出：一页框架简要文档。以一句话总结开头（「框架 X 将交接固定为图边，并通过 reducer 暴露共享状态。」），然后是上述五节，最后以结尾段指出此框架的原语最适合哪种生产项目。
