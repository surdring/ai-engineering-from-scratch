给定一个面向用户的任务（通常是分诊或基于技能的路由），生成一个准备好映射到 OpenAI Swarm 或 OpenAI Agents SDK 的交接（handoff）拓扑。

生成：

1. **智能体名单。** 每个智能体：名称、一句话目的、工具以及可以交接给哪些其他智能体。
2. **交接函数。** 每个智能体的工具签名。每个交接函数返回一个目标 Agent。
3. **上下文传递策略。** 在每个交接边上：完整历史、最近 N 条消息或摘要快照。论证。
4. **护栏。** 每个智能体的输入验证（允许什么提示触发向敏感专家的交接），需要时在交接处进行认证。
5. **循环检测。** 检测乒乓消息的规则（例如「A 交接给 B；B 交接回 A」连续发生超过一次）。
6. **后备行为。** 如果交接目标缺失（移除的智能体、认证失败），哪个智能体处理会话。
7. **会话/记忆计划。** 是否使用 Agents SDK sessions、调用方管理的记忆或完全没有记忆。

硬性拒绝：

- 任何没有循环检测的交接设计。
- 向具有不同工具权限的专业智能体传递完整历史的交接函数（安全风险）。
- 假设 Swarm 的无状态行为但需要多轮记忆的设计——改用 Agents SDK sessions。

拒绝规则：

- 如果任务需要并行执行，拒绝 Swarm 并推荐监督者（第 05 课）。
- 如果任务需要确定性审计/重放，拒绝并推荐 LangGraph 静态图。
- 如果任务是简单的 DAG 阶段（研究 → 编码 → 审查），推荐 CrewAI Sequential。

输出：一页交接简要。以安全说明结尾：提示注入如何触发不想要的交接，以及什么护栏可以阻止它。
