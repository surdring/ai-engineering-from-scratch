给定一个需要智能体群体涌现行为的场景（社会模拟、游戏 NPC、策略预演、市场动态），设计模拟。

生成：

1. **群体规模和异质性。** N 个智能体；哪些共享基础模型，哪些不同；提示家族；角色分布。Smallville 使用 25 个同质智能体和个性化人设；更大的群体受益于异质性。
2. **记忆模式。** 每条目的字段：`(ts, kind, content, importance, embedding_ref, source_ids)`。新近度衰减常数；重要性评分流程；相关性指标（与嵌入模型 X 的余弦相似度）。压缩的保留策略。
3. **反思频率。** 触发条件：未处理重要性总和 > 阈值，或每 N 次观察，或定期滴答。每次触发产生的反思数量。反思提示模板。
4. **计划跨度。** 天 / 小时 / 动作级别。哪些是强制性的；哪些可选。修订触发条件：一条新观察的重要性 > 阈值且与活动计划矛盾。
5. **世界模型。** 空间网格、社会图谱、资源约束。什么构成一次观察（视线、对话、通知）。架构不学习什么范数约束，必须显式编码（容量限制、关闭时间、私密空间）。
6. **种子目标。** 哪些智能体被播种了哪些优先级。可能竞争的重叠目标；应共存的不竞争目标。
7. **预算。** 每智能体每滴答的 LLM 调用次数（观察 + 检索 + 反思 + 计划 + 行动）。每智能体每滴答的预期令牌数。T 个滴答的总模拟成本。
8. **评估指标。** 可信度（人类评分者）、目标达成率、协调事件计数、作为失败信号的空间范式违规。

硬性拒绝：

- 没有显式空间/社会范数编码的设计。架构会违反它们（Park 2023 的关店、单人卫生间失败）。
- 具有可变记忆的设计。记忆必须是仅追加的；修正是新条目。
- 每滴答都运行反思的设计。这是预算低效的；反思昂贵，触发应是基于阈值的。
- 大规模 N（> 50）没有记忆压缩策略。检索成本随流长度增长。

拒绝规则：

- 如果场景需要涌现的*任务执行*而非涌现的*社会行为*，推荐监督者/角色/原语模式（阶段 16 · 05-08）。Smallville 用于社会模拟。
- 如果预算允许每滴答总共 < 100 次 LLM 调用，推荐 N = 3-5 加密集交互，而非更大群体。
- 如果场景不从涌现中受益（严格脚本化任务），推荐单智能体 + 工具。

输出：一页设计简要。以一句话摘要开头（「Smallville 风格模拟：15 个异质智能体，重要性总和 > 120 时反思，3 级计划跨度，带容量约束的空间网格，通过可信度 + 协调事件衡量。」），然后是上述八节。以预期的涌现行为和需要关注的前三种失败模式结尾。
