给定一个任务和智能体名单，生成 GroupChat 配置：选择器选择、选择器输入、终止规则和护栏。

生成：

1. **选择器变体。** 轮询（便宜、公平、上下文无关）、LLM 选择（上下文感知、昂贵）或自定义（LLM + 基于规则的后备）。
2. **选择器输入。** 如果 LLM 选择：最近 N 条消息、智能体专长、轮次数。如果自定义：显式规则。
3. **终止规则。** 最大轮次、TERMINATE 令牌、目标达成验证器或组合。
4. **热发言者缓解。** 每智能体轮次上限、选择器输入中的发言者均衡得分、K 轮连续发言后强制轮换。
5. **上下文膨胀缓解。** 投影计划（每角色范围视图）、摘要检查点、每智能体上下文上限。
6. **可观测性。** 记录选择器输入、选择器选择、每轮智能体延迟。

硬性拒绝：

- 任何不记录选择器输入/输出的 LLM 选择配置。调试变得不可能。
- 没有 max_rounds 上限的配置。
- 推理任务上的对称聊天（无专门化）——改用辩论（第 07 课）。

拒绝规则：

- 如果任务具有已知的 DAG 结构，拒绝 GroupChat 并推荐 LangGraph 静态图以确保确定性。
- 如果任务要求严格审计追踪，拒绝 GroupChat；推荐带检查点的 LangGraph。
- 如果智能体数量超过 5-6，拒绝扁平 GroupChat 并推荐嵌套组或层级模式。

输出：一页 GroupChat 配置简要。以成本估算结尾（LLM 选择每轮产生一次选择器调用）。
