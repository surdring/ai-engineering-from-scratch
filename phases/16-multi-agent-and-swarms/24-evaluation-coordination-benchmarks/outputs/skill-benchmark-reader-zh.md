给定一个已发布或内部的多智能体基准性能声明，对声明进行评分并揭示注意事项。

生成：

1. **基准 + 分割识别。** 哪个基准（MARBLE、COMMA、MedAgentBoard、AgentArch、SWE-bench Pro、SWE-bench Verified、自定义）？哪个分割（完整、保留、污染清理）？未知分割是不合格的。
2. **污染状态。** 基准是否在被测模型的训练截止日期之后？如果基准早于训练截止日期，标记污染风险并贬低声明的价值。
3. **基线质量。** vs 单 LLM、vs 随机、vs 先前多智能体工作。vs 未调优的同一系统不算；这是消融实验，不是基线。
4. **统计显著性。** N 次试验、置信区间或标准误、p 值或等价物。在 N < 50 次试验上无统计的声明是支持不足的。
5. **任务多样性。** 一个任务、一个领域还是多个？单任务声明不意味着泛化能力。
6. **成本披露。** 每任务令牌数、每任务墙钟时间、每任务美元成本。20 倍成本下的 90% 解决方案是业务决策；没有成本，声明是不完整的。
7. **字母等级 + 一句话裁决。**

   - **A:** 全部六项检查通过；声明可能稳健。
   - **B:** 一个弱点；声明在注明注意事项的情况下是可信的。
   - **C:** 两个弱点；声明有提示性但需要复现。
   - **D:** 三个或更多弱点；声明不是证据。
   - **F:** 不合格问题（未披露分割的污染、无统计、无基线）。

硬性拒绝：

- 引用「SWE-bench」而不指定 Verified vs Pro 的声明。40+ 分的差距使这种模糊报告不可接受。
- 没有基线比较的声明。「我们的系统做到 X%」是一个数字，不是结果。
- 基于不到 20 次试验的多智能体系统声明。方差太高。
- 多智能体系统未报告成本。协调税是实质性的。

拒绝规则：

- 如果基准不公开可用且用户没有内部审计追踪，无法评分。建议发布评估工件。
- 如果声明来自当前正在同行评审的论文（arXiv 预印本、未提交），作为预防措施降一级，直到被复现。
- 如果用户自己就是请求审计的声明者，直接运行审计；当声明尚未准备好发表时标记。

输出：一页评分卡。以一句话摘要开头（「等级：C——基准选择好、基线充分，但没有污染检查，没有成本披露。」），然后是上述七节。以优先级列表「为提升等级需要修复什么」结尾。
