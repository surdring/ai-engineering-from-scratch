# 神经网络调试清单

训练出问题时的系统调试协议。按顺序逐项检查——大多数 bug 在前三步就能捕获。

## 训练前（预防 bug）

1. 打印模型架构和参数量。规模对你的数据来说合理吗？
2. 用随机输入运行一次前向传播。输出形状与目标形状匹配吗？
3. 检查标签是否为正确的 dtype（CrossEntropyLoss 需要 Long，BCELoss 需要 Float）
4. 验证数据归一化：输入均值应接近 0，标准差接近 1
5. 打印 5 个随机（输入，标签）对。标签与你预期的一致吗？
6. 确认训练/测试划分没有重复样本

## 单批次过拟合测试（60 秒，捕获 80% 的 bug）

1. 从训练集中取出 8-32 个样本
2. 用合理的学习率训练 200 步
3. 损失应接近 0。训练准确率应达到 100%
4. 如果失败：bug 在你的模型、损失函数或训练循环中——而非数据或超参数
5. 如果通过：继续进行完整训练

## 损失不下降

1. 检查学习率。尝试 3 个值：当前/10、当前、当前*10
2. 打印每层的梯度范数。全为零表示网络死亡或图被分离
3. 检查参数上的 `requires_grad=True`。检查是否调用了 `loss.backward()`
4. 检查 `loss.backward()` 之前是否调用了 `optimizer.zero_grad()`
5. 检查 `loss.backward()` 之后是否调用了 `optimizer.step()`
6. 验证模型参数是否传给了优化器：`optimizer = Adam(model.parameters())`

## 损失为 NaN 或 Inf

1. 将学习率降低 10 倍
2. 在所有 log() 调用中添加 epsilon：`torch.log(x + 1e-7)`
3. 在所有除法中添加 epsilon：`x / (y + 1e-8)`
4. 在 BCE 损失之前钳位预测值：`torch.clamp(pred, 1e-7, 1 - 1e-7)`
5. 使用 `torch.autograd.detect_anomaly()` 找到具体操作
6. 检查输入数据中是否有 NaN：`assert not torch.isnan(x).any()`

## 损失振荡

1. 将学习率降低 3-10 倍
2. 增加批量大小（减少梯度噪声）
3. 添加梯度裁剪：`torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)`
4. 从 SGD 切换到 Adam（逐参数自适应学习率）
5. 在前 5-10% 的训练中添加学习率预热

## 过拟合（训练准确率高，测试准确率低）

1. 添加 Dropout（从 p=0.1 开始，增加到 0.5）
2. 在优化器中添加权重衰减：`Adam(params, weight_decay=1e-4)`
3. 降低模型规模（更少的层或更窄的层）
4. 添加数据增强
5. 使用早停法：验证损失连续增加 5+ 个 epoch 时停止
6. 检查训练集和测试集之间的数据泄漏

## 欠拟合（训练和测试准确率都低）

1. 增加模型容量（更多层、更宽的层）
2. 训练更多 epoch
3. 增加学习率（小心操作）
4. 临时移除正则化以验证模型能够学习
5. 检查模型是否对任务有足够的表达能力

## 死亡 ReLU 神经元

1. 检查每层零激活的比例。>50% 是有问题的
2. 切换到 LeakyReLU(0.01) 或 GELU
3. 对权重使用 Kaiming 初始化
4. 降低学习率（大的更新会将神经元推入死亡区域）
5. 在激活函数之前添加批归一化

## 快速参考：学习率起点

| 优化器 | 任务 | 起始学习率 |
|--------|------|-----------|
| Adam | 从头训练 | 1e-3 |
| Adam | 微调预训练模型 | 1e-5 |
| SGD + 动量 | 从头训练 | 1e-1 |
| SGD + 动量 | 微调预训练模型 | 1e-3 |
| AdamW | Transformer 训练 | 3e-4 |

## 快速参考：批量大小影响

| 批量大小 | 梯度噪声 | 内存 | 泛化能力 |
|---------|---------|------|---------|
| 8-16 | 高（噪声大） | 低 | 通常更好 |
| 32-64 | 中等 | 中等 | 好的默认值 |
| 128-256 | 低（平滑） | 高 | 可能需要预热 |
| 512+ | 极低 | 极高 | 需要学习率缩放 |

## 当一切无效时

1. 将模型简化为 1 层隐藏层。它能学习吗？
2. 将数据简化到 100 个样本。它能过拟合吗？
3. 将损失替换为 MSE。它能收敛吗？
4. 将优化器替换为 SGD(lr=0.01)。它能取得进展吗？
5. 将数据替换为合成数据（如 y = x[0] > 0）。它能学习吗？
6. 如果以上都不行：bug 在你没有查看的代码中（数据加载、预处理、张量形状）
