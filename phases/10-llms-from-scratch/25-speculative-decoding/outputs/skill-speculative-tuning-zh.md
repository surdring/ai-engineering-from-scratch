给定目标模型（大小、系列、分词器）、工作负载遥测（任务混合、提示 vs 解码 token 比率、p50/p99 解码延迟、加速器和 HBM 余量、平均批次大小、采样温度分布）以及可用草稿检查点，输出：

1. 草稿选择。从同系列小型模型（Llama-3.2-1B 配 Llama-70B）、蒸馏草稿（Qwen3-0.6B-spec）、挂在目标上的 Medusa 头、或"不使用推测式解码"（如果没有草稿的 FLOP 成本比小于 30%）中选择。逐字节确认与目标的分词器匹配；拒绝不匹配的分词器。
2. 草稿长度 K。argmax(E[tokens] / (1 + K × c))，其中 c 是草稿对目标的成本比。展示 K 在 2、3、4、5、6 的演算，使用 5000 token 分布内数据的校准运行中测得的 alpha。默认：对话 K=4，代码 K=6，高温创意写作 K=2。
3. 温度门控。设置一个温度阈值，超过此值禁用推测式解码。默认 0.8；如果校准显示 alpha 更早崩塌，下调至 0.6。拒绝任何依赖单请求检查且增加超过 50 微秒延迟的温度门控。
4. 树预算。如果服务栈支持树草稿，batch 8 以下选择小型固定树（深度 2，分支 3-2）；batch 32 以上使用平面链。声明验证器 KV 临时缓冲区大小（字节为单位）并确认适配 HBM 余量。
5. 回退策略。命名指标（最近 1000 次验证的滑动窗口 alpha）和阈值（alpha 低于 0.4），服务器在此条件下退回到该请求流的普通自回归解码。包含回退决策的单请求生命周期。

在批次大小超过验证器为计算密集型临界点的条件下拒绝推测式解码。超过此点，推测器旨在吸收的闲置 FLOPs 不再存在，吞吐量下降。对任何测得 alpha 低于 0.4 的任务类别拒绝推测式解码：草稿开销占主导，墙钟延迟反而更差。拒绝未在留出 1000 token 样本上针对目标验证过的草稿：未验证的草稿是静默的 KL 漂移。

示例输入：「Llama-3.3-70B on 8xH100，对话工作负载，batch 16，p50 解码 28 ms，p99 60 ms，温度分布均值 0.4 / 最大 1.2，校准显示对话 alpha 0.78，代码 alpha 0.61。」

示例输出：
- 草稿：Llama-3.2-1B-Instruct-spec。相同分词器，相同系列，比率 c 约 0.03。
- K：4。E[tokens/verify] = 对话 3.4，代码 2.5。K=5 对话多 0.1 token，多 0.03 额外 c；拒绝。
- 温度门控：0.8。校准集上超过 0.8 时 alpha 降至 0.45 以下。
- 树预算：深度 2 分支 (3, 2)。batch 16 下 KV 临时缓冲区 480 MB，适配。
- 回退：最近 1000 次验证滑动窗口 alpha < 0.40 时，对该流禁用推测式解码 30 秒，然后重新探测。
