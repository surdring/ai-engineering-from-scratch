给定长上下文预训练运行规格（目标上下文、基础架构、可用训练 token、GPU 拓扑、部署目标），生成 NSA 集成计划。

输出：

1. 压缩块大小 `l`。选择 32、64 或 128。对照目标上下文论证：16k-32k 选 `l = 32`，64k-128k 选 `l = 64`，256k 以上选 `l = 128`。`l` 越大意味着压缩键越少但路由信号更粗。
2. Top-k 选择数量。选择 8 到 32。论文默认值为 16。对照目标任务混合论证：推理密集型任务（数学、代码）受益于更高的 `k`，因为选择精度更重要。检索密集型任务用较低 `k` 即可。
3. 滑动窗口 `W`。选择 256、512 或 1024。默认 512。结构化内容（代码）用较短窗口，本地上下文足够；短文用较长窗口。
4. 门控 MLP。指定宽度和初始化。默认：从 `hidden` 到 3 的线性层，使用 `sigmoid` 或 `softplus` 激活。警告：如果门控权重崩塌为偏向某个分支，表明 `l`、`k` 或 `W` 调参不当。
5. 内核选择。确认目标加速器的 Triton 或 CUDA 内核可用。拒绝推理时回退到密集注意力（NSA 的全部意义在于节省解码计算）。如果只有前向内核而没有反向内核，拒绝预训练并推荐在现有密集检查点上进行继续训练。

直接拒绝项：
- 对密集注意力预训练的模型使用 NSA 且不进行持续预训练。不能在推理时简单嫁接。
- 目标上下文低于 16k。三路分支开销占主导。
- 在没有 NSA 内核支持的栈上进行仅推理部署。改为推荐 MLA 或滑动窗口注意力。

拒绝规则：
- 如果没有长上下文评估数据（RULER、LongBench、大海捞针），拒绝并请求先获取校准数据。
- 如果训练数据上下文分布以短序列为主，拒绝并推荐在集成 NSA 前对数据重新加权。
- 如果加速器比 A100 更老，拒绝 — NSA 的内核优势假设 H100/H200/MI300 内存层次。

输出：一页集成计划，列出 `l`、`k`、`W`、门控配置、内核路径以及目标上下文下的预期计算节省。以"成功标准"段落结尾：在 RULER 或 LongBench 上保持 NSA 而非回退的具体数值（对比匹配的密集注意力基线的百分比点）。包含回退触发器 — 低于该值时架构应回退到 MLA 或密集 GQA 的指标阈值。
