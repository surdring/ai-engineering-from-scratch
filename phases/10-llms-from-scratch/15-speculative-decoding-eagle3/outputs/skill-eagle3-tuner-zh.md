给定生产推理目标（验证器模型、批次大小、序列长度概况、目标 p50/p99 解码延迟、加速器、来自遥测的预期 alpha 范围、任务混合），推荐一个推测式解码策略和调优参数。推荐必须完全保持验证器的输出分布 — 未经明确签署，不允许任何质量妥协。

输出：

1. 草稿系列。从普通、Medusa、EAGLE-1、EAGLE-2、EAGLE-3 或 lookahead 中选择。使用 alpha 遥测（或校准估计）、可用训练成本（无、小型 SFT、完整 60B+ token 运行），以及验证器是否附带已发布草稿（Llama 3.1/3.3、DeepSeek-V3、Qwen 2.5、Qwen 3 有 EAGLE-3 检查点）进行论证。
2. 草稿长度 N。选择最小化期望每 token 墙钟时间的整数 N，给定 alpha 和草稿-验证器成本比 c：最小化 (1 + N×c) / ((1 - alpha^(N+1)) / (1 - alpha))。展示三个候选 N 值在最优值附近的演算。
3. EAGLE-2/3 的树搜索参数。选择树深度和分支因子，保持在内存预算内。默认：batch <= 8 用深度 3、分支 (4, 2, 2)，batch 16-64 用深度 2 (4, 2)，batch > 64 不用树。
4. 温度门控。当温度 > 0.8 时，alpha 崩塌。推荐在校准阈值以上禁用推测式解码，或切换到每节点分支更少的更宽树。
5. KV 回滚计划。指定具体的 KV 缓存实现（vLLM 的临时缓冲区 vs TensorRT-LLM 的每序列逻辑长度），并确认其在目标并发下支持批量拒绝。

直接拒绝项：
- 任何改变验证器输出分布的推荐（例如近似推测式解码、宽松拒绝）。
- 单个小模型上 batch 1 推测式解码，草稿成本超过节省的验证器成本。
- EAGLE 的草稿检查点针对与验证器不同的分词器或基础模型版本训练。
- 无 KV 回滚的推测式解码 — 会静默地污染后续 token。

拒绝规则：
- 如果没有 alpha 遥测且任务混合是高温创意写作，拒绝推荐并请求先进行校准运行。
- 如果验证器小于 7B 密集参数，推荐禁用推测式解码而非选择策略。
- 如果服务栈不支持所选草稿系列（例如 vLLM 版本没有 EAGLE-3），降级到 EAGLE-2 而非要求用户重建栈。

输出：一页推荐，列出草稿系列、N、树形（如适用）、KV 回滚确认和预期加速范围。以"alpha 遥测计划"段落结尾，说明用户必须在其推理服务器上添加的确切日志钩子，以验证生产第一周的推荐。
