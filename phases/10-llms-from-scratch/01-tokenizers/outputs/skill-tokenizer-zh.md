# 分词器选择与实现

在启动 LLM 项目时，使用以下决策框架进行分词器选择。

## 何时使用每种分词器

**字节级 BPE（tiktoken）：** 你在构建或微调 GPT 系列模型。你需要保证能处理任何输入字节序列。你希望没有未知 token。

**WordPiece（Hugging Face）：** 你正在使用 BERT 系列模型进行分类、NER 或嵌入任务。你需要"##"延续前缀用于依赖词边界信号的下游任务。

**SentencePiece（BPE 或 Unigram）：** 你正在从头训练。你需要语言无关的分词。你的数据包含 CJK 语言、泰语或其他不含空格词边界的文字。LLaMA、T5 和大多数多语言模型使用它。

## 词汇量大小指南

- 32K token：单语言模型的良好默认值，保持嵌入层较小
- 50K-64K token：多语言或代码密集型模型更佳
- 100K+ token：仅在有大规模训练数据且需要短序列时使用

更大词汇量意味着更短序列（更便宜的推理），但嵌入矩阵参数更多。对于 100K 词汇表、4096 维嵌入，仅嵌入层就有 400M 参数。

## 关键预分词规则

1. 在 BPE 之前按空白分词，防止跨词合并
2. 如希望模型学习算术，则分别拆分每个数字
3. 分词前进行 Unicode 标准化（NFC），保证行为一致性
4. 为你的用例添加特殊 token：`<pad>`、`<eos>`、`<bos>`、`<unk>` 以及任何任务特定标记

## 分词器行为的危险信号

- 目标语言繁殖率（Fertility）高于 2.0：模型浪费上下文窗口
- 常见领域词汇拆分为 3 个以上 token：使用领域数据重新训练
- 数字分词不一致：检查数字拆分规则
- 大词汇表中有许多仅使用一次的 token：减少词汇量

## 构建自定义分词器 — 检查清单

1. 收集代表性训练数据（目标领域至少 1GB 文本）
2. 选择算法：通用选 BPE，多语言选 Unigram
3. 依据上述指南设置词汇量大小
4. 配置预分词：空白分割、数字处理、标点符号
5. 添加特殊 token
6. 使用 Hugging Face tokenizers 库训练（Rust 后端，速度快）
7. 验证：在所有目标语言的留出文本上检查繁殖率
8. 测试边界情况：空字符串、超长输入、二进制数据、表情符号、从右到左文本
9. 保存并版本化分词器，与模型检查点放在一起
