给定模型架构（隐藏维度、头数、KV 头数、层数、每头维度）、目标上下文长度、幻觉或长上下文概况（现有评估中的失败模式）以及训练预算（可用 token、GPU 小时），生成 DIFF V2 的集成计划。

输出：

1. 集成模式。从头预训练、训练中途架构切换，或 Q 投影的 LoRA 微调。对照训练预算和可用现有权重进行论证。
2. 架构差异。具体逐字段变更列表：哪些投影扩大、哪些不变、增加多少参数、减法在注意力块中的位置。包含按层深度的 `lambda_init` 调度（论文默认 `0.8 - 0.6 × exp(-0.3 × (depth - 1))`；如果分层遥测显示不稳定，则调整每层）。
3. 内核选择。确认 FlashAttention 2 或 3 支持 V2 的头数翻倍。拒绝 V1 的自定义内核路径，除非用户明确需要它来保证可复现性。
4. 内存预算。KV 缓存保持在基线水平（KV 头不变）。计算每 token 激活内存增量（额外 Q 头、额外计算）。报告目标上下文下的绝对数字。
5. 训练稳定性计划。描述需要监控的内容：每层 `lambda` 漂移、每头注意力熵、Q 投影的梯度方差。如果遥测指示发散，命名应触发回退到基线注意力的具体指标。

直接拒绝项：
- 在没有持续预训练的情况下向预训练模型添加 DIFF 注意力。输出分布会漂移 — 不是即插即用的修复。
- 2026 年 4 月后的任何新运行使用 DIFF V1。V2 在所有测量维度上严格更优。
- 集成 DIFF 的同时不启用长上下文训练数据。收益仅在 32k 以上显示。
- 在无对照实验的情况下将 `lambda_init` 改为负值。负初始值减去的超过噪声底线，会使训练崩塌。

拒绝规则：
- 如果目标上下文低于 16k，拒绝集成并推荐标准注意力。增加的参数成本无法用噪声底线论证来证明。
- 如果用户无法提供长上下文评估数据（RULER、大海捞针、MultiNeedle），拒绝并请求先获取校准数据。
- 如果用户使用的是 FlashAttention 2 之前的栈，拒绝并推荐先升级栈再尝试集成。

输出：一页集成计划，列出模式、参数增量、KV 缓存影响、FlashAttention 确认、`lambda` 调度以及 3 指标监控面板。以"成功标准"段落结尾，说明在 RULER 64k 或等价测试上保持 DIFF V2 架构而非回退的特定长上下文评估数值（百分比点增量）。
