# 差分注意力（V2）

> Softmax 注意力将一小部分概率扩散到每个不匹配的 token 上。在 10 万个 token 上，这些噪声会累积并淹没信号。差分 Transformer（Ye 等人，ICLR 2025）通过计算两个 softmax 的差值来修复此问题，减去共享的噪声基底。DIFF V2（微软，2026 年 1 月）是生产栈重写版本：解码延迟与基准 Transformer 匹配，无需自定义内核，兼容 FlashAttention。本课覆盖 V1 到 V2 的完整过程，包含一个可在标准库 Python 中运行的差分操作的可运行玩具实现。

**类型：** 构建
**语言：** Python（标准库）
**前置知识：** 第七阶段 · 02（自注意力），第七阶段 · 15（注意力变体），第十阶段 · 14（架构遍历）
**时间：** 约 60 分钟

## 学习目标

- 精确说明为什么 softmax 注意力存在噪声基底，以及它为什么随上下文长度增长。
- 推导差分注意力公式，解释为什么减法消除共享噪声分量同时保留信号。
- 遍历 V1 到 V2 的差异：什么变快了，什么变简单了，什么变稳定了，以及每个改变为什么对生产预训练是必需的。
- 从头在纯 Python 中实现差分注意力，并在合成的信号加噪声查询上经验性地验证噪声消除属性。

## 问题

标准 softmax 注意力有一个在大规模下变成操作难题的数学属性。对于查询 `q`，注意力权重是 `softmax(qK^T / sqrt(d))`。Softmax 永远不可能产生精确的零——每个不匹配的 token 都会获得一些正质量。这些残差质量就是噪声，并且随上下文长度缩放。在 128k token 下，即使每个不匹配 token 只获得 0.001% 的概率，127,999 个这样的 token 合计贡献约 12% 的总量。模型必须学会绕过随上下文增长的噪声基底。

这在经验上表现为注意力头干扰：长上下文 RAG 中幻觉化的引用、100k token 检索任务中的中间丢失失败，以及在超过 32k 的干草堆中找针基准测试上的细微准确率下降。差分 Transformer 论文（arXiv:2410.05258，ICLR 2025）测量了差距：DIFF Transformer 相比同规模基准达到了更低的困惑度、更高的长上下文准确率和更少的幻觉。

DIFF V1 有三个问题使其无法进入前沿预训练流水线。其值缓存必须在每个解码步骤加载两次，需要自定义 CUDA 内核来打破 FlashAttention 兼容性，而且其每个头的 RMSNorm 在 70B 以上规模的长时间训练中不稳定。DIFF V2（微软 unilm 博客，2026 年 1 月 20 日）修复了全部三个问题。本课遍历两个版本，构建差分操作符，并在玩具查询上基准测试噪声消除。

## 核心概念

### Softmax 的噪声基底

对于查询 `q` 和键 `K = [k_1, ..., k_N]`，注意力权重为：

```
w_i = exp(q . k_i / sqrt(d)) / sum_j exp(q . k_j / sqrt(d))
```

没有任何 `w_i` 是零。如果 `k_i` 与 `q` 完全无关，得分 `q . k_i` 并非 0——它在零附近波动，方差为 `||q||^2 / d`。经过 softmax 归一化后，每个无关 token 仍然贡献 `O(1/N)` 的加权和。无关 token 的总贡献是 `O((N-1)/N) = O(1)`——这不是一个小量。

模型想要的是类似硬 top-k 的东西：在匹配 token 上高权重，在其他所有位置接近零权重。Softmax 过于平滑，无法直接做到。

### 差分思想

将每个头的 Q 和 K 投影分成两部分：Q = (Q_1, Q_2) 和 K = (K_1, K_2)。计算两个注意力图：

```
A_1 = softmax(Q_1 K_1^T / sqrt(d))
A_2 = softmax(Q_2 K_2^T / sqrt(d))
```

输出：

```
DiffAttn = (A_1 - lambda * A_2) V
```

减法消除了两个图共享的任何噪声分布。如果两个图都在 127k 个无关 token 上有大致均匀的权重（它们在随机初始化时会如此），这些就会相互抵消。信号——在少数真正相关 token 上的峰值权重——只有在两个图中以相同幅度出现时才会抵消，而一旦模型开始训练，这种情况不会发生。

`lambda` 是每个头的可学习标量，参数化为 `lambda = exp(lambda_q1 · lambda_k1) - exp(lambda_q2 · lambda_k2) + lambda_init`。它可以是负数。`lambda_init` 默认为一个小正数，如 0.8。

### 为什么这类似于面向噪声消除的麦克风降噪

想象两个有噪声的麦克风录制同一个声音。两个麦克风都拾取说话者以及相关的背景噪声。将其中一个减去另一个，共享的噪声就会消失。声音幸存下来，因为两个信号在相位或幅度上有足够差异，防止了完全抵消。每个头的 `lambda` 正是学习这种平衡。

### V1 与 V2：差异

V1 保持参数数量与基准 Transformer 相等。为了获得每个头两个查询，它将头维度减半。这牺牲了头表达能力，并且——更痛苦的是——将每个头的值缓存减半。解码必须每个步骤加载值缓存两次（每个 softmax 分支一次）。结果：尽管参数数量匹配，解码比基准慢。

V2 将查询头数量加倍，保持 KV 头不变（从上投影中借用参数）。头维度与基准相同。减法之后，额外维度被投影回基准 Transformer 的 O_W 投影大小。三件事同时发生：

1. 解码速度与基准匹配（KV 缓存只加载一次）。
2. FlashAttention 无需更改即可运行（无需自定义内核）。
3. 解码时的算术强度提高（从 HBM 加载的每字节执行更多计算）。

V2 还移除了 V1 用来稳定减法的每个头 RMSNorm。在 70B 级别的预训练规模下，该 RMSNorm 在训练后期不稳定。V2 用更简单的初始化方案取代它，在没有额外模块的情况下保持训练稳定。

### 何时使用

| 工作负载 | 收益 |
|----------|---------|
| 长上下文 RAG（64k+） | 更干净的注意力图，更少的幻觉化引用 |
| 干草堆中找针基准测试 | 超过 32k 时准确率显著提升 |
| 多文档 QA | 更少的跨文档干扰 |
| 8k 代码补全 | 边际收益，不值得架构改变 |
| 短对话（< 4k） | 基本上与基准无法区分 |

价值随上下文长度增长。在 4k token 下，噪声基底足够小，标准注意力完全没问题。在 128k 下它在损害你。

### 与其他 2026 年配置的配合

## 构建实现

`code/main.py` 在纯 Python 中实现差分注意力。一个具有已知信号加噪声结构的玩具查询让你可以直接测量噪声消除比。

### 步骤 1：标准 softmax 注意力

标准库矩阵操作：列表的列表，手动矩阵乘法，带模型稳定性最大值减法的 softmax。

```python
def softmax(row):
    m = max(row)
    exps = [math.exp(x - m) for x in row]
    s = sum(exps)
    return [e / s for e in exps]
```

### 步骤 2：将 Q、K 分成两半

V1 风格：将头维度减半。V2 风格：保持头维度并将头数量加倍。玩具实现使用 V1 以便于教学清晰——数学完全相同，只是簿记方式不同。

### 步骤 3：两个 softmax 分支 + 减法

```python
A1 = [softmax([dot(q1, k) / scale for k in K1]) for q1 in Q1]
A2 = [softmax([dot(q2, k) / scale for k in K2]) for q2 in Q2]
diff_weights = [[a1 - lam * a2 for a1, a2 in zip(r1, r2)] for r1, r2 in zip(A1, A2)]
out = [[sum(w * v[j] for w, v in zip(row, V)) for j in range(d_v)] for row in diff_weights]
```

注意：输出权重可以是负数。这没问题——值缓存仍然处理带符号的贡献。后续的 V 投影吸收符号。

### 步骤 4：噪声消除测量

构建一个长度为 1024 的合成序列。将信号 token 放在已知位置，其余用噪声填充。计算 (a) 信号位置上的标准 softmax 注意力权重和 (b) 差分注意力权重。测量每个的信噪比。DIFF 注意力可靠地产生更高的信噪比，根据两个分支被训练到多大差异，倍数在 3x-10x 之间。

### 步骤 5：V1 与 V2 参数核算

给定一个配置（hidden=4096, heads=32, d_head=128），打印：

- 基准 Transformer：Q、K、V 各为 `hidden * hidden`，MLP 为 4 * hidden。
- DIFF V1：Q、K 各为 `hidden * hidden`，V 为 `hidden * hidden`（不变），头维度在内部减半。添加每个头的 `lambda` 参数（O(heads * d_head)）。
- DIFF V2：Q 为 `2 * hidden * hidden`，K 为 `hidden * hidden`，V 为 `hidden * hidden`。额外维度在 O_W 之前投影回去。添加相同的 `lambda` 参数。

玩具测量 V2 的额外参数成本（每个注意力块大约 `hidden * hidden` 额外参数）并打印。

## 使用方式

截至 2026 年 4 月，DIFF V2 尚未在每一个生产推理服务器中发布，但集成正在 vLLM 和 SGLang 中进行。同时，该模式出现在：

- 微软内部的长上下文生产模型。
- 多个以 256k 以上上下文为目标的开放模型训练运行的科研复现。
- 在交替层上组合 DIFF 注意力与滑动窗口注意力的混合架构。

2026 年何时使用：

- 从头训练一个以 64k 以上有效上下文为目标的新模型。从一开始就添加差分注意力；以后重新训练成本很高。
- 微调一个中间丢失失败主导你的评估的长上下文模型。在 Q 投影上的 LoRA 可以近似 DIFF 结构。

何时不使用：

- 你正在服务于一个具有稳定长上下文性能的预训练稠密模型。重新训练成本在已有权重上很少回本。
- 你的上下文始终在 16k 以下。噪声基底可以忽略。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|----------------|------------------------|
| 差分注意力 | "两个 softmax 互减" | 将 Q、K 分成两半，计算两个 softmax 图，将第二个（按 lambda 缩放）从第一个中减去，然后乘以 V |
| 噪声基底 | "softmax 的非零尾部" | softmax 在每个无关 token 上放置的 O(1/N) 权重，在长上下文中总和为 O(1) |
| lambda | "减法尺度" | 每个头可学习标量，参数化为 `exp(lq1·lk1) - exp(lq2·lk2) + lambda_init`；可以为负数 |
| DIFF V1 | "ICLR 2025 版本" | 原始差分 Transformer；减半头维度以保持参数数量，需要自定义内核，解码较慢 |
| DIFF V2 | "2026 年 1 月修复" | 加倍 Q 头，保持 KV 头；解码速度匹配基准，兼容 FlashAttention |
| 每个头 RMSNorm | "V1 稳定器" | V1 在差值之后应用的额外归一化；V2 将其移除以防止训练后期不稳定 |
| 信噪比 | "多少注意力被浪费" | 真实信号位置上的权重与无关位置上平均权重的比率 |
| 中间丢失 | "长上下文失败模式" | 经验现象，即检索准确率在长上下文中间位置的文档上下降——差分注意力减少此问题 |
| 算术强度 | "每加载字节的 FLOPs" | V2 在解码时通过加倍每次 KV 加载的查询数而提高的比率；对内存受限的解码很重要 |
