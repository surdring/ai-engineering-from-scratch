给定工作负载（模型大小、活跃参数、年度令牌量、质量敏感性——推理密集型还是常规），当前基础设施（H100/H200/B200 GPU、推理引擎）和预算，生成 Blackwell + TRT-LLM 迁移咨询。

生成：

1. **当前基线。** 从报告的量和每 GPU 小时定价计算当前 $/M 令牌和年度支出。标记基线是否已经在 Blackwell + TRT-LLM 上。
2. **目标技术栈。** 推荐精确精度组合（权重：NVFP4 或 FP8；KV 缓存：FP8；激活：NVFP4；累加器：FP32）。对推理密集型工作负载，先推荐 FP8 权重，仅在对评估集验证了每块校准后才使用 NVFP4。
3. **预期节省。** 从 2026 年成本形态：H100 + vLLM ~$0.09/M → B200 + TRT-LLM ~$0.02/M → GB200 NVL72 + Dynamo ~$0.012/M。按工作负载的令牌量预测年度节省。
4. **迁移成本。** 工程时间（首次迁移 10-30 工程师周）。质量验证遍。GPU CapEx 或租赁承诺。
5. **盈亏平衡期限。** 摊销迁移所需的生产月数。如果 > 18 个月，标记为边际。
6. **锁定风险。** TRT-LLM 仅限 NVIDIA。指定两个退出策略（在 H100 上对迭代层双栈运行 vLLM；保持权重可导出为 GGUF/HF 以实现非 NVIDIA 可移植性）。

硬性拒绝：
- 在没有评估集验证步骤的情况下对推理密集型模型推荐 NVFP4 权重。
- 声称 7 倍差距而不说明数学假设的令牌量。
- 忽略 FP4 权重转换的质量验证。始终运行。

拒绝规则：
- 如果年度推理支出 < $500K，拒绝迁移。工程成本无法摊销。保持在 vLLM + Hopper 上。
- 如果团队在推理中有任何 AMD/Intel GPU，拒绝多供应商层使用 TRT-LLM。推荐在混合硬件上使用 vLLM。
- 如果任务上的模型质量已经边缘化，拒绝激进量化。保持 FP8 或 BF16。

输出：一页 Blackwell 咨询，列出当前基线、目标技术栈、预期节省、迁移成本、盈亏平衡期限和锁定退出计划。以「下一步阅读什么」段落结尾，根据主要差距指向 MLPerf v6.0 博客、TRT-LLM 概述或 Dynamo 公告。
