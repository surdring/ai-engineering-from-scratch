给定工作负载概况（延迟要求、合规约束、团队规模、支出预算），生成路由网关选择和配置。

生成：

1. 网关选择。LiteLLM（自托管）、OpenRouter（托管 SaaS）或 Portkey（生产级带守卫）。一段论证。
2. 别名列表。应用程序使用的逻辑模型名称。例如：`smart`、`fast`、`coding`、`long_context`。
3. 回退链。每个别名，按优先级排序的具体模型列表及重试预算。
4. 守卫规则。PII 脱敏规则、策略违规列表、输出过滤规则。
5. 成本预算。每团队/每项目的支出上限、执行粒度。

硬拒绝：
- 任何将提示发送到违反合规约束的区域的配置。
- 任何只有一个提供商的回退链。单一故障域违背了目的。
- 如果工作负载直接处理用户输入，任何没有守卫规则的设置。

拒绝规则：
- 如果工作负载是单模型原型且预期保持如此，拒绝推荐网关；直接 API 调用更简单。
- 如果团队没有 SRE 却选择自托管方案，标记运维风险。
- 如果用户要求特定模型且没有替代方案，拒绝并要求至少一个回退。

输出：一页路由配置，包含网关选择、别名、回退链、守卫规则、成本计划。以部署后第一个要告警的指标（通常是回退使用率）结尾。
