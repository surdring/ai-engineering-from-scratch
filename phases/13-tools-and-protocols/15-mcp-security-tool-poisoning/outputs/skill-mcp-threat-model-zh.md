给定 MCP 部署（服务器列表、工具列表、权限列表），生成威胁模型。

生成：

1. 攻击适用性。对七种攻击类别（工具投毒、卷款跑路、影子攻击、MPMA、寄生工具链、Sampling 攻击、供应链伪装）中每一种，评估适用性为高 / 中 / 低，附一句话论证。
2. 防御清单。列出已部署的防御措施（哈希固定、静态检测器、网关、签名注册表、MELON、Rule-of-Two 执行）。
3. Rule of Two 审计。对每个工具，分类为不可信 / 敏感 / 后果性并标记单个回合中三者组合的任何情况。
4. 缺失的防御。根据威胁概况指出尚未应用的最高杠杆防御。
5. 操作手册。团队下周应执行的三项操作以改善安全态势。

硬拒绝：
- 任何声称「攻击类别 X 不适用，因为我们信任这个服务器」的威胁模型。假设一个服务器会被入侵。
- 任何使用无声覆盖命名空间解析的部署。
- 任何启用了 Sampling 但没有每会话速率限制器的部署。

拒绝规则：
- 如果部署没有已批准工具描述的文档，拒绝并要求先固定哈希。
- 如果部署使用公共未签名的 MCP 注册表，标记供应链风险并推荐迁移到已验证注册表。
- 如果任何工具组合了不可信输入、敏感数据和后果性操作，拒绝批准并要求拆分。

输出：一页威胁模型，包含攻击适用性表、防御清单、Rule-of-Two 标志列表和三操作手册。以该部署中价值最高的单项安全增强措施结尾。
