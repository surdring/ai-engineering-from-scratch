给定一个仓库、智能体产品及其依赖面，生成项目特定的初始化脚本和 CI 接线。

生成：

1. `tools/init_agent.py`，包含以下探测项：运行时版本、列出的依赖、测试命令可解析性、必需的环境变量、状态文件新鲜度。
2. `init_report.json` 模式在脚本旁文档化。每个探测返回 `(name, status: pass|warn|fail, detail)`。
3. `.github/workflows/agent-init.yml`（或等价物），运行脚本并在任何 fail 严重级别的探测上阻止智能体作业。
4. 一个 `pre-task` 钩子脚本，智能体运行时可以在每次会话开始前调用。
5. `docs/init.md` 文档，列出每个探测、其严重级别以及如何修复失败。

硬性拒绝：

- 没有超时的网络调用探测。初始化必须快速且可离线安全运行。
- 需要 LLM 调用的探测。初始化是确定性的管道工作。
- 被包装器吞没的非零退出码。大声失败是全部意义。
- 没有幂等性就触碰状态的探测。连续两次运行必须产生模时间戳外相同的报告。

拒绝规则：

- 如果项目没有测试命令，拒绝交付脚本。将此缺口添加到工作台审计中。
- 如果环境变量列表包含脚本会打印的密钥，拒绝并强制脱敏。初始化报告绝不应携带密钥。
- 如果探测在干运行中超过三秒，在交付前提出计时发现。长探测会将初始化变成仪式。

输出结构：

```
<repo>/
├── tools/
│   ├── init_agent.py
│   └── pre_task.sh
├── docs/
│   └── init.md
└── .github/
    └── workflows/
        └── agent-init.yml
```
