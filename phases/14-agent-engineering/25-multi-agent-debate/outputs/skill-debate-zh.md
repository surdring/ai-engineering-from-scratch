给定一个问题类别和准确率目标，搭建一个辩论协议。

生成：

1. 具有不同提示（理想情况下不同模型）的 `Debater`，以避免同质化。
2. 轮次运行器：全连接、星形或环形拓扑。
3. 收敛规则：多数投票、按置信度加权、或超多数带回退。
4. 第 1 轮强制分歧：如果可能，每个辩论者返回不同的提案。
5. 成本核算：总评判操作数 + 每个问题的 token 成本。

硬性拒绝：

- 所有辩论者使用相同提示和相同模型。保证群体思维。
- 没有检查成本的全连接且 N >= 6。辩论操作按 O(N*R) 规模增长。
- 没有收敛规则。返回辩论者 0 的第 R 轮答案不是收敛。

拒绝规则：

- 如果产品对延迟敏感（<1s 预算），拒绝辩论。使用 Self-Refine（第 05 课）或并行投票（第 12 课）代替。
- 如果问题类别是简单事实查找（首都、日期、定义），拒绝辩论。查找 + CRITIC（第 05 课）更便宜。
- 如果辩论者在评估集中任何问题的第 1 轮后都没有分歧，拒绝该协议。你需要模型/提示多样性。

输出：`debater.py`、`topology.py`、`convergence.py`、`runner.py`、`README.md`，解释 N/R 选择、拓扑理由和评估集上的成本与准确率测量。结尾的「下一步阅读」指向第 12 课（工作流模式）如果任务更简单，或第 28 课（编排模式）关于将辩论嵌入更大系统。
