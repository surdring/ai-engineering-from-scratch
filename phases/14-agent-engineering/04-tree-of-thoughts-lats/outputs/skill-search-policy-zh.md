给定一个任务形态（单答案 / 多答案 / 开放式）、一个 token 预算和一个可用的评估器（标量测试 / 启发式 / 自评估），生成一个带具体参数的搜索策略推荐。

生成内容：

1. 决策。选择以下之一：线性 ReAct、束搜索 ToT（束宽 k）、BFS ToT（最大深度）、带剪枝的 DFS ToT、MCTS LATS（迭代次数和 UCT c）、进化搜索（仅当评估器是程序化且可检查的）。
2. 参数。每种策略的具体数值默认值：束宽、深度上限、分支因子 K、每层 rollout 数、UCT c（默认 1.4）、超时时间。
3. 价值函数。明确指定评分节点的标准。选项：单元测试通过率、与目标的数值距离、带格式的 LLM 提示评分（sure/likely/impossible 或 1..10 或投票）、或环境奖励。
4. Token 预算估计。最坏情况 token 数 = 分支因子 ^ 深度 * 平均提示 token 数。展示该数值。如果超过用户预算，推荐更便宜的策略。
5. 失败模式。对每种选定策略，列出前两种失败模式及其缓解措施（例如 LATS + 噪声评估器 -> 按 CRITIC 添加工具支撑验证，第 05 课）。

硬性拒绝：

- 当评估器不可靠时推荐搜索（仅自评估，无真实标准）。回退到 ReAct + CRITIC。
- 没有充分理由将分支因子 K 设置高于 5。K=3-5 是论文默认值；K=10 会导致成本爆炸。
- 将 LATS 应用于聊天式任务。搜索对没有程序化目标的对话问答没有帮助。
- 没有机器可检查的适应度函数时使用进化搜索。AlphaEvolve 只有在适应度是程序化的（运行测试、测量速度、验证定理）时才有意义。

拒绝规则：

- 如果 token 预算 < 单轨迹成本的 5 倍，拒绝搜索并推荐 ReAct + Reflexion（第 03 课）。
- 如果墙上时钟延迟预算 < 10 秒，拒绝 LATS 并推荐 ReAct。
- 如果任务是纯粹的信息检索，拒绝搜索并推荐 ReWOO（第 02 课）。

输出：一个推荐块（选定的策略、参数、价值函数、预算估计），加上「下一步阅读」指向第 05 课（CRITIC）关于评估器可靠性，第 11 课（AlphaEvolve）关于进化变体，或第 30 课（评估驱动开发）关于基准级别的验证。
