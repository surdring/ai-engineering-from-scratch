给定一个目标运行时（Python 异步、Python 同步、Node、Rust 异步、Go）和一个工具列表（名称、输入模式、可调用），生成一个一次就能正确的 ReAct 智能体循环。

生成内容：

1. 一个消息缓冲区类型，包含角色 {user, assistant, tool, final} 以及目标服务商期望的模式（Anthropic `tool_use` / `tool_result` 块、OpenAI function-calling 消息、Responses API 推理通道）。绝不在服务商之间静默切换模式。
2. 一个工具注册表，包含名称到可调用分发、输入验证和带类型的返回结果。错误必须被捕获并转换为观察字符串，绝不抛出到循环层。
3. 一个循环，持续运行直到满足以下条件之一：明确的 `finish` 动作、assistant 轮次中没有工具调用、达到最大轮次、达到最大总 token 数、或触发护栏。选择一个主要停止条件；其余作为安全兜底。
4. 一个根据任务类别调整的轮次预算——短任务 10，计算机使用 200，深度研究 400。明确说明所选的预算。
5. 一个追踪记录，记录每一步的思考、动作、观察和停止原因。当运行时存在 OTel SDK 时，发出 OpenTelemetry GenAI span（`invoke_agent`、`tool_call`）。

硬性拒绝：

- 没有轮次上限的循环。这是可靠性问题，不是优化问题。
- 将工具错误吞没为空观察。模型必须看到失败文本才能修正。
- 将检索到的内容视为受信任的指令。所有工具输出都是不可信的输入——只有用户消息携带权限（参见 OpenAI CUA 文档）。
- 在没有模式转换层的情况下混合使用服务商。Anthropic 和 OpenAI 的工具模式和消息形态不同。

拒绝规则：

- 如果目标是「无框架，仅 bash」，拒绝并建议至少使用有类型的消息模式；智能体循环对无类型的 shell 胶水代码来说太容易出错。
- 如果用户要求「工具调用失败时自动重试而不反馈给模型」，拒绝。重试必须要么经过模型（CRITIC/Self-Refine，第 05 课），要么属于工具自身的幂等性约定。
- 如果工具列表中包含没有人工确认环节的破坏性工具，拒绝并指向第 09 课（权限和沙箱）。

输出：每个语言目标一个文件，外加一个 `README.md`，解释停止条件选择、轮次预算的合理性，并展示一个逐步的思考-动作-观察工作追踪。结尾的「下一步阅读」指向第 02 课（ReWOO 规划）如果任务是长时间范围的，第 03 课（Reflexion）如果任务是重复之前的任务，或第 27 课（提示注入）如果工具涉及不可信内容。
