# 物理合理性检查

生成视频的生产部署需要自动化护栏。人工审查无法规模化；物理检查可以捕捉经典的失败模式。

## 使用场景

- 任何从文本或图像提示生成视频的产品。
- 对视频生成 API 端点进行自动化 QA。
- 监控视频模型在微调或基础模型更新后的质量偏移。

## 输入

- `video`: 张量 `(T, H, W, 3)` 或 mp4 文件路径。
- 可选的参考信息：预期的物体数量、初始场景描述。

## 检查项

### 1. 物体恒存性
使用 SAM 3.1 Object Multiplex 跨帧跟踪每个检测结果。当一个稳定跟踪目标消失不超过 3 帧后重新出现时标记 — 模型暂时丢失了该物体。当物体在帧中心附近（而非边缘）消失时判定为硬失败；在边缘消失时判定为软失败。

### 2. 运动平滑性
连续帧之间的光流应当是基本连续的。像素级流场的突然激增表明物体发生了瞬移。使用 RAFT 计算光流；标记第 99 百分位光流幅值超过中位数 10 倍的帧。

### 3. 重力 / 支撑
对于检测为固体的物体（食物、球、工具），检查其垂直位置在没有举起动作的情况下是否不增加。标记向上漂移，除非在物体附近检测到"抓握的手"。

### 4. 身份一致性
对于人物或角色，跨帧使用人脸识别嵌入。对于持续存在的身份，5 帧窗口内的余弦相似度应保持在 > 0.8。低于阈值意味着角色发生了变形。

### 5. 手部和肢体
运行姿态估计器（第 21 课）。标记以下帧：手部可见手指 > 5 或 < 4；手臂长度在两帧之间翻倍；肢体穿透身体表面。

### 6. 文字渲染（如果提示要求生成文字）
如果用户提示包含引号内的字符串，对生成的帧进行 OCR 并计算与所请求字符串的 CER。标记 CER > 20% 的情况。

## 报告

```
[plausibility]
  video frames:           <T>
  permanence violations:  <N>
  smoothness violations:  <N>
  gravity violations:     <N>
  identity drift:         <5帧窗口数N>
  limb anomalies:         <N>
  OCR CER vs requested:   <浮点数>

[verdict]
  ship | hold | reject

[samples for review]
  每个失败发生的帧范围
```

## 规则

- 不要仅凭单一检查就硬性阻止；汇总分数，在总异常数超过阈值时将视频保留供审查。
- 身份偏移和物体恒存性违规的权重最高 — 用户最先注意到这些。
- 随时间记录每个检查的失败率；上升趋势通常意味着基础模型已更新或提示分布发生了变化。
- 绝不删除被标记的视频；保留用于模型调试和事后分析。
- 对于敏感内容（人物、儿童、公众人物），无论分数如何，要求对每个视频进行人工审查。
