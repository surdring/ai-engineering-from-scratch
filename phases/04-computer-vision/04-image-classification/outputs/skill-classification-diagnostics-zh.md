# 分类诊断

一个解读混淆矩阵的透镜。总体准确率告诉你分类器是否有效。混淆矩阵告诉你它**还不知道什么**。

## 使用场景

- 首次查看已训练分类器的验证性能
- 在两次训练运行之间决定下一步调整什么
- 在交付模型前：验证没有关键类别静默地失败
- 调试生产环境中的回归问题，当总体准确率下降一个百分点，你需要知道原因

## 输入

- `cm`：C×C 的混淆矩阵（行 = 真实标签，列 = 预测标签）
- `labels`：C 个类别名称的列表，顺序一致
- 可选 `class_priors`：每个类别的训练频率（默认为 `cm` 的行和）

## 步骤

1. **计算每个类别的指标。**将任何除以零的情况视为该指标对该类别无定义，并报告为 `n/a`；绝不对其静默地替换为 0
   - precision_i = cm[i,i] / sum(cm[:, i])   （当该类别从未被预测时无定义）
   - recall_i    = cm[i,i] / sum(cm[i, :])   （当该类别没有真实样本时无定义）
   - f1_i        = 2 * p * r / (p + r)        （当任一分量为无定义时无定义）

2. **按 F1 分数对最差的三个类别排序。**如果混淆矩阵中类别少于三个，则对实际存在的类别排序。排除所有指标均为无定义的类别

3. **找出每行最大的非对角线单元（Off-diagonal Cell）**——即最常从该类窃走样本的对应类别。报告为 `真实类别 -> 预测类别`

4. **为每个最差的类别分类其失败模式。**使用以下量化阈值，使标签具有可复现性：
   - `歧义（ambiguity）`——与另一类别双向混淆：`cm[i,j] / sum(cm[i, :]) >= 0.15` 且 `cm[j,i] / sum(cm[j, :]) >= 0.15`
   - `不平衡（imbalance）`——该类别的训练样本数不到其最大混淆类别的 0.5 倍
   - `标签噪声（label_noise）`——`|precision_i - recall_i| >= 0.2` 且该类别不在歧义/不平衡路径上
   - `系统性（systematic）`——没有单一混淆类别的错误占比超过 0.2；错误分散在三个或更多类别中

5. **推荐唯一最有影响的下一步行动**：
   - `歧义` -> 收集或合成具有区分度的样本，添加保持区分特征的目标增强
   - `不平衡` -> 过采样少数类或使用类别加权损失
   - `标签噪声` -> 审核该类别的分层抽样样本；在任何其他修改之前先修正标签错误
   - `系统性` -> 增加该类别的数据，或以该类别的损失赋予更高权重进行微调

## 报告

```
[诊断]
  总体准确率: X.XX
  宏观 F1:     X.XX

[最差三类]
  1. 类别 <名称>  F1 = X.XX  精确率 = X.XX  召回率 = X.XX
     主要混淆: <名称> -> <其他名称>  (N 个样本)
     失败模式:  歧义 | 不平衡 | 标签噪声 | 系统性
     行动:      <一句话>

  2. ...
  3. ...

[建议]
  最重要的杠杆: <一句话，指明类别和修复方案>
```

## 规则

- 最多返回三个类别。更多的会掩盖真正的问题信号
- 为每个最差类别指明其主要混淆类别；不要概括为「与多个类别混淆」
- 每条建议都要基于混淆矩阵的证据。不要泛泛地说「增加更多数据」，而不指明是哪个类别
- 当精确率和召回率的差异超过 0.2 时，始终将标签噪声标记为候选原因——训练后的真实类别通常具有对齐的精确率和召回率
