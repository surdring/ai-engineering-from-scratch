# 帧采样器审查器

帧采样是视频流水线中最容易出错的地方。这里的 bug 会传播到所有下游指标。

## 使用场景

- 编写新的视频数据加载器
- 复现论文数据时训练准确率低于报告值
- 调试跨多次运行的评估准确率不稳定的视频模型

## 输入

- `sampler_code`：接受 (num_frames_total, T) 并返回 T 个索引的 Python 函数
- `T`：目标片段长度
- 可选的测试用例：需要覆盖的 `num_frames_total` 值（如 `[3, T-1, T, T+1, 30, 300, 3000]`）

## 检查项

### 1. 短片段处理
传入 `num_frames_total < T`。返回的每个索引必须在 `[0, num_frames_total - 1]` 范围内。标准填充策略是对剩余位置重复最后一帧。

### 2. 边界索引
传入 `num_frames_total == T`。返回的索引应恰好为 `[0, 1, ..., T-1]`。

### 3. 均匀分布
传入 `num_frames_total == 10 * T`。返回的索引应单调递增且大致均匀分布。

### 4. 密集窗口边界
对于密集采样，传入 `num_frames_total == 3 * T`。返回的索引应形成一个连续窗口，绝不跨越片段末尾。

### 5. 确定性
用相同输入和（对于确定性采样器）相同随机数生成器调用采样器两次。索引应匹配。

### 6. 裁剪一致性
如果流水线还为每帧返回空间裁剪，用相同种子对同一片段运行采样器两次，确认每帧使用相同的裁剪框（相同的 `(x, y, w, h)`）。片段内不同帧的不同裁剪会破坏时间连贯性，这是一种经典的静默 bug。可接受的变体：增强以**逐片段**方式应用，在片段内保持一致。

## 报告

```
[采样器审计]
  名称: <函数名>
  T:    <整数>

[短片段处理]
  通过 | 失败（<详情>）

[边界]
  通过 | 失败

[均匀分布]
  通过 | 失败（<间隔标准差>）

[密集窗口]
  通过 | 失败（<详情>）

[确定性]
  通过 | 失败

[裁剪一致性]
  通过 | 失败（<逐帧裁剪变化: 是/否>）

[结论]
  ok | 需要修复
```

## 规则

- 如果短片段处理返回越界索引，绝不标记采样器为「ok」
- 密集采样器绝不返回跨越 `num_frames_total - 1` 的窗口
- 如果采样器是随机的（密集），仅在显式设种子的随机数生成器下测试确定性
- 建议但不静默修复的标准策略：用最后一帧填充、将窗口钳制到末尾、半开区间取整
