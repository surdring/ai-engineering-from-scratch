# 锚框设计器

锚框（Anchor）是基于锚框的检测器中与数据集最相关的超参数。默认的 COCO 锚框在细胞培养图像、卫星瓦片或小目标监控场景下表现不佳。本技能推导出真正匹配目标数据的锚框。

## 使用场景

- 在新数据集上首次训练之前
- 当极小或极大目标的召回率在表现正常的模型上表现较弱时
- 在数据集大幅扩充后，框尺寸分布可能已发生变化

## 输入

- `boxes`：形状为 (N, 4) 的 numpy 数组，格式为 `(cx, cy, w, h)` 或 `(x1, y1, x2, y2)`；建议至少 1000 个正样本框
- `num_anchors_per_level`：通常为 3
- `num_fpn_levels`：通常为 3（P3, P4, P5）或 4
- `input_size`：训练分辨率 H×W
- 可选 `strides`：每个层级的步幅；如省略，则取 `[8, 16, 32, 64]` 的前 `num_fpn_levels` 项。如检测器的 FPN 步幅不同，请显式传入更长或更短的数组

## 步骤

1. **将框归一化**为 `input_size` 下的像素坐标 (w, h) 对。丢弃 w 或 h 小于 2 像素的框

2. **在 (w, h) 对上运行 k-means**，设置 `k = num_anchors_per_level * num_fpn_levels`。使用 `1 - IoU(box, cluster)` 作为距离函数，**不要**用欧氏距离——欧氏距离在 (w, h) 上会把细高框和方形框混为一类。所有框等权重贡献（不加权）；如果你的数据集存在类别不平衡但又需要更大框的召回率，请在输入数组中重复稀有类别框，而不是传递权重向量

3. **按面积升序排列聚类中心。**分为 `num_fpn_levels` 组，每组 `num_anchors_per_level` 个。最小面积的分配给分辨率最高的层级（最小步幅）

4. **按层级计算覆盖率统计**：
   - 每个真实框与其在该层级的最佳锚框的 `中位 IoU`
   - `recall@IoU=0.5`——最佳锚框 IoU >= 0.5 的框的百分比
   - `面积覆盖率`——面积落在该层级 `[anchor_min_area / 4, anchor_max_area * 4]` 范围内的框的比例

5. **报告逐层级锚框**并标记 `recall@IoU=0.5 < 0.9` 的层级；该层级的锚框与数据匹配不佳，应重新调整或增加该层级的锚框数量

## 报告格式

```
[锚框设计器]
  总框数:              <N>
  聚类数:              <k>
  距离度量:            1 - IoU

[层级 P3  步幅=8]
  锚框 (w, h):         [(A, B), (C, D), (E, F)]
  中位 IoU:            <X>
  recall@IoU=0.5:      <X>
  覆盖率:              <X>
  标记:                ok | 需重新调整

[层级 P4  步幅=16]
  ...

[摘要]
  总体 recall@IoU=0.5: <X>
  最小锚框:            <w x h>
  最大锚框:            <w x h>
  建议:                <如有层级被标记，一句话说明>
```

## 规则

- 始终使用基于 IoU 的距离；欧氏 k-means 产生的锚框在视觉上合理但在经验上更差
- 按面积排序聚类中心，然后升序分配到各个层级
- 当 `num_anchors_per_level = 1` 时，完全跳过 k-means：按面积分位数将框分为 `num_fpn_levels` 个区间（如 3 个层级则使用三分位数），并将每个层级的锚框设为该区间的中位 (w, h)。对于小数据集，这比用 `k = num_fpn_levels` 运行 k-means 更稳健
- 绝不输出负的锚框维度；钳制到 1
- 如果数据集的框少于 200 个，警告用户锚框搜索不可靠，建议使用默认的 COCO 锚框并增加训练数据
